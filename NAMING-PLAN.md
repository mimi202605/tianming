## 一、关键事实（决定风险与做法）

1. **全局解耦**：所有文件靠全局 `window.*` 互相调用，文件名与它定义的功能符号无关 → 改名只影响「加载层」（脚本引用），不影响互调。所以「文件越核心 = 改名越危险」不成立，核心文件和叶子文件改名代价一样。
2. **改名真实成本 = 文件名被多少地方引用**。全项目动态按名加载仅 **3 处**：`map-editor-io.js`、`tm-official-scenario-seeder.js`、`tm-semantic-recall.js`。其余全是 `index.html` 里的静态脚本引用（261 段，224 段带 `?v=` 缓存戳）。

## 二、现状真实画像（重要：没想象中乱）

- web/ 共 **351** 个平铺 `.js`，其中 **341 个已合规**（`tm/map/editor-子域-功能` 一致命名）。
- 真正违反规范的只有 **~11 个**，就这一小撮：
  - `phase8-formal-*`（7 个）：`phase8` 是「开发第 8 阶段」代号、无领域含义，实为新版界面。
  - `detect_globals.js`、`scan_globals.js`（2 个）：用了下划线、且无域结构。
  - `editor.js`（1 个）：裸名，未归入 `editor-` 域。
  - `tm-tinyi-v3.js`（1 个）：版本号混进文件名。
- tm 下子域分布（佐证已有结构）：keju 37、faction 21、endturn 21、char 18、ai 10、edict/office/chaoyi/authority/memory 各 4……

## 三、命名规范（目标约定）

1. 全小写、短横线分隔、**禁用下划线**。
2. 三段式：「域-子域-功能」，域前缀走固定白名单。
3. 域前缀白名单：`tm`（核心玩法）、`map`（地图）、`editor`（编辑器）、`formal`（新版界面，替换无意义的 `phase8`）、`dev`（调试/工具类散件）。
4. 文件名不带版本号（版本交给更新日志 + 缓存戳）。

## 四、执行范围

### 必做（本期切割·全部低风险）
规范化这 11 个违例文件，目标名：
- `phase8-formal-*` → `formal-*`（7 个，去掉无意义的 phase8）
- `detect_globals.js` → `dev-detect-globals.js`
- `scan_globals.js` → `dev-scan-globals.js`
- `editor.js` → `editor-core.js`
- `tm-tinyi-v3.js` → `tm-tinyi.js`（改前确认无同名冲突）

> 这 11 个都不在那 3 个动态加载目标里 → 全是标准改名（B 类），风险最低。

### 可选（深度重构·后续单独立项）
- 去掉冗余 `tm-` 前缀 / 落实真实子目录（`web/keju/`、`web/endturn/`…）。涉及 200+ 文件。
- **成本提示**：因为发布走整包（见六），分发成本是固定的、不随改名数量增加 → 深度重构不额外加重玩家下载；唯一成本是内部工作量 + 验证量。
- 建议：本期先不做，确有维护收益时再单独立项。

## 五、A / B 风险分级（按「文件名被引用处数」，非按重要性）

- **A 类（额外小心 / 或保留原名）**：仅在做深度重构、触及以下文件时才相关——
  - 3 个动态加载目标：`map-editor-io.js`、`tm-official-scenario-seeder.js`、`tm-semantic-recall.js`。
  - 生成物 `tm-official-scenario-bundle.js`（由生成脚本产出 + seeder 动态投放，改名需同步改两处）。
  - grep 出「文件名被别处写死」的文件。
  - 处理：改名 + 同步更新所有引用 + 重点验证；生成物可保留原名作为文档化例外。
- **B 类（标准改名）**：其余全部，只被自己那行脚本引用。本期 11 个违例文件全属此类。

## 六、发布策略（切割式）

1. 全量改名 + 内部逐个验证，**攒成一个版本**，不拆成多次热更。
2. 走「热更整包推」：用系统的强制整包开关，把整个前端作为一个完整包下发一次（非增量比对），玩家**不用重装 exe**，一次拿到干净版本。
3. **不发 exe 安装包**（玩家已基本只用热更）。
4. **搭车发布**：改名里程碑与一个有玩家价值的版本（功能/内容/修复）一起发，避免玩家为纯内部改名白下一次全量。
5. **清单自洗**：整包会重新生成文件清单，之前那条畸形条目（绝对路径被压成文件名）自动消失，无需手动修。

## 七、单文件执行流程（每个改名都走一遍）

1. 改文件名为规范名。
2. 改 `index.html` 里对应那行 `<script src>`（含 `?v=` 缓存戳）。
3. 若属 A 类：同步更新动态加载处 / 生成脚本 / 投放脚本里的文件名字符串。
4. 全仓 grep 旧文件名，确认无其它字符串引用残留。
5. 启动验证（进主菜单 + 相关功能 + smoke）。

## 八、风险与回滚

- 热更按「文件名 + 指纹」做增量 → 大批改名若走增量会被当成「删旧 + 下新」、diff 全是噪声难定位 → 必须走整包一次切。
- 逐个改、即时验，避免全改完才暴露问题。
- 回滚：改名是机械操作，保留「旧名 → 新名」映射表，异常可逐条还原。
