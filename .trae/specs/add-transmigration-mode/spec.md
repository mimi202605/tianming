# 穿越·扮演非皇帝角色 Spec

## Why

天命当前默认玩家=皇帝（北极星第4条），但项目核心是「AI 驱动的历史平行宇宙生成器」，玩家身份不应被锁死在皇帝位。已有大量基础设施（`P.playerInfo.playerRole`、`GM.regentState`、`buildRegentSignal`、`_applyRegentDecisions`）支持非皇帝玩家，但 UI 文案、朝议/诏令入口、官制权限判定仍硬编码"皇帝=玩家"。

新增「穿越」模式：玩家在主界面入口选择扮演除皇帝以外的任何人物（朝中重臣/军中将领/一方诸侯/摄政权臣/商贾平民/后宫等），皇帝由 AI 全权接管（自动下旨、朝议、处理事务）。玩家在自己职位上发挥应有作用，UI 和功能随角色定位动态调整。同时配套十四大玩家专属系统——人物互动、赚钱私产、跑商、科技研发（固定路线）、家族与子女（含婚嫁/赘婿/再婚/和离）、私军、自由移动、产业建设、开垦荒地、参加科举、官员年终考核、反叛、玩家自我技能提升（学塾/拜师/游学/历练/江湖习武）——让玩家在自己职位上"有事可做、有成长感、有戏剧性"。这一模式直击命门——把"硬核 × 自由"的核心爽点从"皇帝视角"扩展到"任何角色视角"，每个角色都是一条新的平行宇宙故事线，反叛成功还可翻转回皇帝视角。

## What Changes

### 主界面入口
- 在 `index.html` 主界面（launch 页）新增「穿越」按钮，与「开卷/续卷/著卷/典章」并列
- 点击「穿越」进入角色选择流程：先选剧本 → 再从剧本角色列表中选一位非皇帝角色作为玩家化身
- 角色选择面板按 `playerRole` 分组展示（重臣/将领/诸侯/宗室/外戚/后宫/商贾/隐逸等），并显示该角色档案（姓名、官职、品级、势力、性格、家族、关系网）

### 玩家身份系统
- **BREAKING**：`P.playerInfo.playerRole` 默认值从 `emperor` 改为根据剧本开局判定——若选穿越模式则取所选角色的角色定位
- 新增 `P.playerInfo.transmigrationMode`（boolean）：标记是否为穿越模式存档
- 新增 `P.playerInfo.sovereignName`：当前皇帝名（穿越模式下与 `characterName` 解耦）
- 复用 `c.isPlayer` 作为玩家化身标识，但 `isPlayer=true` 的角色不再隐含 `isEmperor`
- 启动时校验：穿越模式下所选角色 `_offIsSovereign(c)` 必须返回 false，否则拒绝

### 皇帝 AI 自动化
- 新增 `TM.Sovereign.runAITurn(root)`：穿越模式下每回合为皇帝生成决策（下旨/朝议发言/批奏/任免）
- 复用现有派系 NPC LLM 决策管线（`tm-faction-npc-llm-decision.js`），但皇帝作为"超级 NPC"获得最高优先级和独立 prompt 模板
- 皇帝诏令复用 `EDICT_TYPES` 与 `classifyEdict`，落账到 `GM._edictTracker`（带 `source: 'sovereign-ai'` 标记）
- 皇帝朝议发言复用 `_cyInterjectRespond` 的反向：皇帝作为主动发言人触发朝议，而非玩家插言
- 皇帝批奏：奏疏系统（`tm-memorials.js`）新增 AI 自动批答路径，玩家臣子的奏疏会得到皇帝批答

### 玩家角色功能（按 playerRole 分支）
- **朝中重臣（minister）**：
  - 上奏（奏疏面板从"批奏"改为"上奏"）
  - 议政（朝议中发言需先请旨，不能"插言"）
  - 任免建议（不能直接任免，只能廷推/荐举）
  - 处理本职政务（如吏部尚书可拟官制变更建议、户部尚书可拟财政方案）
- **军中将领（general）**：
  - 军事指令（调兵/出征/筑城/补给）需上报请旨
  - 战场指挥全权（沿用 tm-battle-turn）
  - 可上军事奏报
- **一方诸侯（prince）**：
  - 治理封国/封地（独立财政/官制/军事，但需朝贡）
  - 可举兵勤王/叛乱/割据
  - 可上表请旨
- **摄政权臣（regent）**：
  - 复用现有 `regentState` 摄政机制
  - 可代行部分皇权（需标注"代诏"），但有合法性损耗
  - 君主成年后需还政
- **宗室/外戚/后宫**：
  - 后宫：复用 `tm-houguong.js`，玩家可影响皇帝决策（枕边风）
  - 宗室：可就藩/参政/谋夺嫡
  - 外戚：可借势揽权/拱卫皇权
- **商贾/隐逸（merchant/custom）**：
  - 经济活动（经商/购置田产/资助势力）
  - 可上书言事，但需通过臣子转呈
  - 可被征辟出仕

### 玩家专属系统（穿越模式新增）

> 以下系统是穿越模式的核心爽点，让玩家在自己职位上"有事可做、有成长感、有戏剧性"。复用现有引擎底层（角色经济/科举/编年/派系），新增玩家主观视角的参与入口。

- **人物互动系统**：
  - 玩家可与任意 NPC 进行一对一互动：拜访、密谈、请托、结交、馈赠、联姻、寻仇、陷害、笼络、收为门客
  - 每次互动消耗玩家精力/时间，影响 NPC 对玩家的关系值（5 维：师徒/亲友/政敌/同僚/仇敌）
  - 互动结果由 LLM 根据双方性格、关系、利益、当前局势生成（复用 `tm-wendui.js` 一对一对话语料管线）
  - 关键互动写入玩家记忆系统（沿用 `tm-memory-*.js`），形成长期关系弧线
  - 互动可触发"事件钩子"：如笼络禁军将领到一定程度可发动政变、与后宫结盟可影响皇帝继承人

- **赚钱系统**：
  - 玩家个人私产管理（复用 `tm-char-economy-engine.js` 的 14 类角色俸禄/私产/名望）
  - 多元收入源：官俸、赏赐、田租、商税分成、灰色收入（贪墨/受贿/索贿）、经营性收入（自营产业）、收藏性资产（古董/字画/珠宝）
  - 投资渠道：购置田产、置办产业（酒楼/当铺/作坊）、放贷收息、囤货居奇
  - 风险机制：贪墨触发吏治腐败引擎（沿用 `tm-corruption-engine.js`）的"被弹劾"风险；放贷过度触发民怨；囤货触发市舶司调查
  - 与派系勾连：资助某派系换取政治支持，或被派系勒索保护费
  - 玩家个人银钱账本（独立于国库），可被抄家清算（沿用 `tm-char-economy-ui.js` 的抄家路径）

- **跑商系统**：
  - 玩家可派遣商队（或亲自带队）跨区域贸易：买入低价商品→运输到高价区卖出
  - 复用 `tm-economy-engine.js` 的区域商品价格矩阵与市场动态
  - 路线风险：山贼劫掠（沿用 `tm-coastal-raid.js` 模式）、官府盘剥、气候灾害、地方势力索要过路费
  - 商队组建：雇佣护卫（消耗银钱）、购置车马、办理通关文牒（需朝廷许可，可与官场关系挂钩）
  - 大宗贸易可影响区域经济指标（复用 `tm-region-magnate.js`）
  - 跑商可积累商誉，开启新的商业网络与 NPC 关系
  - 跨朝代通用：丝路/茶马/漕运/海贸等都是剧本数据 hook，引擎只提供"商队+路线+风险+利润"通用框架

- **科技系统（固定科技路线）**：
  - 玩家可主导科技研发（不论 playerRole）：农业/水利/军事/工艺/医药/天文/算学/文学等领域
  - **预设固定科技路线**：每个领域有固定科技树，前置科技解锁后才能研发后继
    - 农业线：`农具改良 → 良种选育 → 水利灌溉 → 耕作制度 → 多熟种植`
    - 军事线：`冶铁锻造 → 弩机改良 → 甲胄升级 → 攻城器械 → 火药初探`
    - 工艺线：`纺织改进 → 陶瓷烧制 → 造纸印刷 → 冶铸高炉 → 雕版活字`
    - 医药线：`本草整理 → 方剂编纂 → 针灸推拿 → 疫病防治 → 法医检验`
    - 水利线：`沟渠疏浚 → 陂塘修筑 → 堰坝工程 → 运河开凿 → 海塘修筑`
    - 天文/算学/文学线同理（预设固定 5 级科技链）
  - 玩家出资/出人/出时间投入研发，研发成功解锁对应增益（农业增产、军事强军、工艺增收、医药减疫）
  - 研发进度受玩家学识（`learning` 字段）+ 投入资源 + 现有科技基础 + 时代限制影响
  - 玩家可招揽匠人/学者加速研发（关联人物互动系统）
  - 研发成果可上奏朝廷推广（影响全国），或私藏自用（仅个人/封国受益）
  - 复用 `tm-dynamic-systems.js` 提供科技树通用框架，**默认科技路线内置于引擎**（剧本可覆盖/扩展，但不能空）
  - 跨朝代铁律：固定科技路线取中国古代通用脉络，剧本数据可加朝代专属支线（如明清火器、宋代活字）

- **家族与子女系统**：
  - 玩家拥有完整家族结构：父母/兄弟/姐妹/配偶/子女/宗族
  - 复用 `tm-char-full-schema.js` 的 `family` / `familyMembers` 字段
  - 玩家可结婚（若未婚）：通过人物互动系统联姻，配偶带入嫁妆/陪嫁/家族关系
  - **婚嫁三种模式**：
    - **娶妻/嫁女**（常规）：玩家作为娶方或嫁方家族代表，按常规礼制走"纳采/问名/纳吉/纳征/请期/亲迎"六礼
    - **赘婿**（matrilocal）：玩家若为男性且选择"入赘"，则嫁入女方家族，改为女方姓氏（或保留原姓但身份为赘婿），地位较低但可借妻家势力；赘婿路径常用于贫寒男子借势崛起——可触发"赘婿逆袭"或"被妻家欺压"两条戏剧线
    - **招赘**（若玩家为女性或玩家家主为女性）：玩家可招赘婿入门，赘婿入玩家家族
  - **再婚**：配偶死亡（含战死/病亡/难产/被诛）或正式和离后，玩家可再婚
    - 再婚限制：守制期（父母丧/夫丧/妻丧）内不得再婚，违反触发礼法风险
    - 再婚带子女：鳏夫/寡妇带子女再婚，子女与新配偶关系动态生成（慈/严/慈爱/虐待/敌对）
    - 平妻/继室：再婚配偶身份可为继室（低于原配），特殊情况下可平妻（触发嫡庶之争）
  - 离婚/和离：玩家可主动和离（双方协议）或休妻（男方单方面，需符合七出之条）/休夫（罕见，需妻家强势）；和离后子女归属按礼法判定
  - 玩家可有子女：子女随时间成长（婴幼儿→少年→成年），每个阶段有事件（满月/启蒙/冠礼/及笄）
  - 子女教育：玩家可延师/亲自教导/送书院（关联科举系统），影响子女学识/品性/能力
  - 子女联姻：成年子女可与其他 NPC 家族联姻，扩大势力网络
  - 子女出仕：成年子女可走科举/荫袭/征辟入仕，成为玩家政治资源
  - 子女继承：玩家死亡/罢黜时，子女可继承部分家产/官职/名望
  - 子女叛逃：子女若与玩家关系恶化，可能叛逃敌对势力或自立门户
  - 子嗣危机：无子嗣触发"绝嗣"危机（家族断代、旁支夺嫡）；子嗣过多触发"夺嫡"内斗

- **玩家自我技能提升系统**：
  - 玩家可提升自己的各项技能/属性：`learning`（学识）/ `intelligence`（才智）/ `valor`（勇武）/ `military`（军事）/ `administration`（吏治）/ `management`（治理）/ `charisma`（魅力）/ `diplomacy`（外交）/ `benevolence`（仁德）/ `diction`（辞令）/ `learning` 子项（经史/诗赋/算学/律法/医术/兵法/天文/地理/音律/书法等）
  - **提升场所**：
    - **学塾/书院**：报名入学（交纳学费/通过考核），按期学习提升 `learning` 与子项；不同书院有不同侧重（经史书院/武书院/医书院/律书院等）；与 `tm-keju-school-network.js` 复用基础设施
    - **名师拜师**：通过人物互动拜 NPC 名师，按师徒关系获得定向加成（如拜名将学兵法、拜名医学医术、拜大儒学经史）
    - **自学苦读**：玩家可消耗时间 + 银钱（购书/置办文具）自学，效率较低但灵活
    - **游学**：关联自由移动系统，游历名胜/古战场/名郡，触发游学事件加成属性
    - **历练**：通过实际政务/军事/外交任务积累经验，自动提升对应技能
    - **江湖习武**：若 playerRole 为商贾/隐逸/江湖人，可入武馆/镖局/门派习武，提升 `valor` / `military`
    - **修道/礼佛**：可选宗教路径提升 `benevolence` / `intelligence`，但有副作用（远离世俗、官场关系疏远）
  - 提升机制：每次学习消耗时间 + 银钱 + 精力，按 `当前属性 + 师资 + 投入 + 资质` 计算提升概率与幅度
  - 属性上限：每个属性有上限（如 100），随年龄变化（少年学习快但属性低，老年属性稳定但学习慢）
  - 提升反馈：每次提升写入玩家记忆与编年史，关键突破（如完成一部著作/拜得名师/悟出某理）触发事件
  - 跨朝代通用：学塾/书院/武馆等具体名目由剧本数据 hook，引擎只提供"场所 + 课程 + 投入 → 属性提升"通用框架

- **私军系统**：
  - 玩家可招募并使用私军（独立于朝廷正规军）
  - 私军类型：家丁（轻装护卫）/ 门客剑士（精锐）/ 部曲（重装）/ 死士（特殊任务）
  - 招募渠道：从流民/镖师/江湖人/退役军士中招募；关联人物互动可招揽名将
  - 维护成本：每月消耗银钱/粮草，规模超限触发财政压力
  - 训练度：私军有训练度字段，定期训练可提升战斗力
  - 装备：玩家可购置兵器/甲胄/战马武装私军（关联私产系统）
  - 使用场景：
    - 护卫商队（关联跑商系统，降低山贼劫掠风险）
    - 自卫（反叛筹备期抵御朝廷围剿）
    - 政变/篡位（关联反叛系统，私军是反叛主力）
    - 私斗（与 NPC 家族械斗）
  - 风险：私军规模超阈值触发"僭越"风险（朝廷调查、言官弹劾）；规模达"拥兵自重"触发朝廷问罪
  - 沿用 `tm-military.js` / `tm-army-units.js` 的战斗单位模型，但标记 `kind:'private'` 独立账本

- **自由移动系统**：
  - 玩家可自由移动到地图上任何已发现的地点（城池/乡村/边疆/名胜/属地等）
  - 移动消耗时间（按距离/交通工具）+ 银钱（路费）+ 精力
  - 移动方式：步行/骑马/车驾/舟船/驿站（不同方式速度/成本不同）
  - 复用 `tm-char-full-schema.js` 的 `location` 字段 + `tm-military.js` 的行军路径算法
  - 玩家当前地点决定可执行动作集：
    - 在京城：可上朝/拜访朝臣/参与朝议
    - 在地方：可治理本职辖区/拜访地方士绅
    - 在封国：可治理封国/练兵
    - 在边疆：可巡视边防/招揽边将
    - 在名胜/隐逸地：可游学/招揽名士
    - 在敌国/化外：触发外交事件或被通缉
  - 移动事件：路上可能遇盗匪/天气/偶遇 NPC/发现古迹
  - 玩家可携带家属/私军/商队一起移动（关联其他系统）

- **产业建设系统**：
  - 玩家可在指定地点建设产业（不同于"购置已有产业"，是从无到有建设）
  - 产业类型：
    - 庄园（综合型：含田地/园林/作坊/库房）
    - 农场（粮食产出）
    - 牧场（牲畜产出，关联骑兵/私军战马）
    - 矿场（金属/煤炭/盐/玉石产出）
    - 林场（木材产出）
    - 渔场（水产产出，需在水边）
    - 工坊（手工业产出：纺织/陶瓷/造纸/冶铸）
    - 商号（商业产出：酒楼/当铺/客栈）
  - 建设流程：选址 → 购地（消耗银钱 + 当地官府许可）→ 募工 → 施工（消耗时间）→ 投产
  - 选址限制：不同地形/区位限制可建产业类型（如矿场只能在矿山附近）
  - 产业经营：每月产出货物/银钱，受管理（玩家或雇管家）+ 民夫/匠人数量 + 治安 + 灾害影响
  - 产业升级：可投入资源升级（扩建/改良/特化）
  - 产业风险：火灾/水灾/盗匪/民夫逃亡/官府强征/敌军劫掠
  - 与朝廷关系：大产业可能触发"豪强"标签，朝廷可能强征/抄没
  - 沿用 `tm-building-works.js` 的建筑系统底层

- **开垦荒地系统**：
  - 玩家可在指定区域开垦荒地，增加当地可耕地面积与粮食产出
  - 开垦流程：勘探荒地 → 募集流民/民夫 → 平整土地 → 修水利 → 播种 → 收获
  - 开垦规模：小块（百亩）/ 中块（千亩）/ 大块（万亩），不同规模消耗时间/银钱/人力
  - 开垦前置：需获得当地官府许可（关联人物互动 + 官制权限）；违规开垦触发"占田"风险
  - 开垦产出：新耕地按比例增加当地粮食产量，玩家可享分成（按投入比例）
  - 开垦副作用：可能侵占牧场/林地/猎场，触发当地势力不满；大规模开垦触发生态事件（水患/沙化）
  - 与朝廷互动：开垦有成可上奏请功（升迁/赐爵）；开垦失败触发问责
  - 跨朝代通用：不同朝代开垦政策（屯田/占田/均田）由剧本数据 hook，引擎只提供"开垦→产出→风险"通用框架

- **参加科举考试**：
  - 玩家若 playerRole 为商贾/隐逸/宗室旁支/低级官吏子弟，可选择走科举入仕路径
  - 玩家作为考生参与科举全流程：县试→府试→院试（秀才）→乡试（举人）→会试（贡士）→殿试（进士）
  - 复用 `tm-keju-*.js` 全套（37 文件 ~26k 行），玩家以考生身份接入
  - 考试内容：经义/策论/诗赋（剧本数据决定具体题型），玩家作答后由 LLM 评卷（沿用 `tm-keju-question-ui.js`）
  - 玩家可拜师求学（关联人物互动）、入书院深造（沿用 `tm-keju-school-network.js`）
  - 玩家考中后获官职（影响 `playerRole` 升级为 minister），未中可继续苦读或捐官
  - 玩家可主动卷入科场弊案（关联 `tm-keju-scandal.js`）：行贿考官、请托关节、枪替冒名——风险与收益并存

- **官员年终考核**：
  - 玩家若 playerRole 为 minister/general 等官员身份，每年末参加考核
  - 复用现有官制系统的"考课"机制（如有），新增玩家视角的考核指标：履职情况、政务成绩、廉洁度、人际关系、上级评价、民众口碑
  - 考核结果分九等（上上/上中/上下/中上/中中/中下/下上/下中/下下）
  - 考核影响：升迁（升级 playerRole 或品级）、贬谪、加俸、罚俸、赐物、记过、罢黜
  - 玩家可主动运作考核（关联人物互动）：贿赂考官、托人情、提前布局政绩
  - 考核结果由 LLM 综合 1 年玩家行为生成评语，写入编年史

- **反叛系统**：
  - 玩家若 playerRole 为 prince/regent/general/minister 均可发动反叛
  - 反叛前置：笼络军权（禁军/边军）、积累粮草、联络内外势力、制造舆论（童谣/谶纬）
  - 反叛类型：
    - 宗室夺嫡：趁君主病重/幼年发动政变
    - 藩王起兵：以"清君侧"名义举兵
    - 权臣篡位：废君自立（沿用 `tm-feudal.js` 的篡位机制）
    - 边将叛乱：拥兵自立割据一方
    - 农民起义（merchant/custom 角色可达）：聚众举事（沿用 `tm-class-radical-revolt.js`）
  - 反叛流程：筹备→举事→交战（沿用 `tm-battle-*.js`）→成败→后续（自立/被诛/逃亡/招安）
  - 反叛失败后果：抄家灭族、家属流放、声名扫地（写入编年史永久污点）
  - 反叛成功后果：玩家成为新君主（`playerRole` 转为 emperor，模式从穿越切回皇帝模式）

## ADDED Requirements

### Requirement: 主界面穿越入口

系统 SHALL 在主界面（`web/index.html` 的 `#launch` 区）新增「穿越」按钮，与现有「开卷/续卷/著卷/典章」并列，按钮文案朝代中立（不写死"朕/御极/临朝"等皇帝专属词）。

#### Scenario: 用户点击穿越
- **WHEN** 用户在主界面点击「穿越」按钮
- **THEN** 进入剧本选择界面（复用现有 `showScnSelect`）
- **AND** 选定剧本后进入「角色选择」界面，展示该剧本所有非皇帝角色，按 playerRole 分组

#### Scenario: 角色选择面板展示
- **WHEN** 用户进入角色选择面板
- **THEN** 每个角色条目展示：姓名、字/号、官职、品级、势力、性格摘要、家族层级
- **AND** 皇帝角色（`_offIsSovereign(c) === true`）不出现在列表中或被显式禁用并标注"君主不可选"
- **AND** 提供"角色档案详情"展开入口（显示完整 SCHEMA 字段）

#### Scenario: 取消穿越回到主界面
- **WHEN** 用户在角色选择面板点击返回
- **THEN** 回到主界面，不进入游戏

### Requirement: 穿越模式存档标记

系统 SHALL 在存档元数据中标记 `P.playerInfo.transmigrationMode`，并持久化所选角色 ID 与 `playerRole`。

#### Scenario: 穿越模式存档读档
- **WHEN** 玩家在穿越模式下存档，再读档
- **THEN** `transmigrationMode === true` 被恢复
- **AND** `P.playerInfo.characterName` / `playerRole` / `sovereignName` 全部恢复
- **AND** `c.isPlayer` 正确指向所选角色

#### Scenario: 普通存档不受影响
- **WHEN** 读取传统皇帝模式存档（无 transmigrationMode 字段）
- **THEN** 默认补 `transmigrationMode: false`
- **AND** 行为与改造前完全一致（向后兼容）

### Requirement: 皇帝 AI 自动决策

系统 SHALL 在穿越模式下每回合为皇帝生成 AI 决策，覆盖下旨、朝议发言、批奏、任免四个动作面。

#### Scenario: 皇帝 AI 下旨
- **WHEN** 进入穿越模式的回合结算
- **THEN** 皇帝 AI 根据当前世界状态（国库/民心/边警/吏治/派系等）生成 0-3 道诏令
- **AND** 诏令类型限定在 `EDICT_TYPES` 范围内
- **AND** 诏令落账到 `GM._edictTracker`，标记 `source: 'sovereign-ai'`
- **AND** 诏令经 `estimateResistance` 与 `applyEdictTypedIncidence` 走完整阻力与执行流程

#### Scenario: 皇帝 AI 朝议发言
- **WHEN** 穿越模式下朝议开启
- **THEN** 皇帝作为主动发言人触发朝议议题（而非玩家插言）
- **AND** 皇帝发言经 LLM 生成，符合其性格与时代质感
- **AND** 玩家臣子可请旨发言（非打断式插言）

#### Scenario: 皇帝 AI 批奏
- **WHEN** 玩家臣子上奏或 NPC 上奏
- **THEN** 皇帝 AI 生成批答（准/驳/留中/下议/交部）
- **AND** 批答结果影响奏疏状态与 NPC 忠诚度
- **AND** 玩家上奏的批答以"奉旨"形式反馈给玩家

#### Scenario: 皇帝 AI 任免
- **WHEN** 皇帝 AI 判断需要任免（如廷推结果、罢免罪臣、赏功臣）
- **THEN** 经 `_offAppointPerson` / `_offDismissPerson` 走标准任职树变更
- **AND** 变更经 `tm-ai-change-applier.js` 路径白名单校验
- **AND** 玩家本人可能被任免（升迁/贬谪/罢黜），玩家收到通知

### Requirement: 玩家角色动作集

系统 SHALL 根据 `P.playerInfo.playerRole` 为玩家提供角色定位匹配的动作集，禁用与角色身份不符的动作。

#### Scenario: 朝中重臣上奏
- **WHEN** 玩家 playerRole === 'minister' 进入诏令面板
- **THEN** 面板标题改为"上奏"（非"天子御笔"）
- **AND** 提供 1-3 篇奏疏 textarea
- **AND** 提供"廷推荐举""本职政务"等角色相关动作按钮
- **AND** 不出现"诏令颁行"按钮

#### Scenario: 军中将领请旨出征
- **WHEN** 玩家 playerRole === 'general' 进入军事面板
- **THEN** 调兵/出征按钮改为"请旨出征"
- **AND** 上奏后由皇帝 AI 决定是否准允
- **AND** 准允后玩家在战场拥有完整指挥权（沿用 tm-battle-turn）

#### Scenario: 一方诸侯治理封国
- **WHEN** 玩家 playerRole === 'prince'
- **THEN** 御案显示封国面板（财政/官制/军事，限定于封国范围）
- **AND** 提供"朝贡""举兵""上表"等动作
- **AND** 朝贡不及时会触发朝廷问罪

#### Scenario: 摄政权臣代行皇权
- **WHEN** 玩家 playerRole === 'regent'
- **THEN** 复用现有 `GM.regentState` 摄政机制
- **AND** 玩家可代行部分皇权（如代拟诏令），但诏令标记"代诏"
- **AND** 代诏行为有合法性损耗（影响 `皇威` 或派生合法性指标）
- **AND** 君主成年后系统提示还政，玩家可选择还政或拒还（拒还会触发"权臣架空"危机）

#### Scenario: 后宫玩家影响皇帝
- **WHEN** 玩家 playerRole 属于后宫定位
- **THEN** 御案显示后宫面板（复用 tm-houguong）
- **AND** 玩家可通过"枕边风"机制影响皇帝 AI 决策（增加皇帝 AI prompt 中的"宠爱妃嫔建议"权重）
- **AND** 影响力受位分/宠爱限制

### Requirement: UI 文案朝代中立化

系统 SHALL 将穿越模式涉及的所有 UI 文案改为朝代中立（沿用 CLAUDE.md 跨朝代铁律），不硬编明清专名。

#### Scenario: 朝议发言人动态化
- **WHEN** 朝议渲染气泡或写入起居注
- **THEN** 皇帝发言使用 `P.playerInfo.sovereignName`（穿越模式）或 `P.playerInfo.characterName`（皇帝模式）
- **AND** 不再硬编码 `'皇帝'` 字面量

#### Scenario: 诏令抬头动态化
- **WHEN** 诏令面板渲染
- **THEN** 皇帝模式显示"奉天承运皇帝诏曰"（或剧本配置的朝代特定抬头）
- **AND** 穿越模式下奏疏面板显示朝代中立的"臣某某谨奏"

#### Scenario: 顶栏身份展示
- **WHEN** 顶栏渲染玩家身份
- **THEN** 皇帝模式：显示"年号 · 尊号"（如"崇祯十五年 · 明思宗"）
- **AND** 穿越模式：显示"玩家官职 · 当前皇帝年号"（如"兵部尚书 · 崇祯十五年"）

### Requirement: 玩家人物互动系统

系统 SHALL 在穿越模式下提供玩家与任意 NPC 一对一互动入口，互动消耗精力/时间并影响关系值，结果由 LLM 生成。

#### Scenario: 玩家拜访 NPC
- **WHEN** 玩家在御案选择"拜访"动作并选定 NPC
- **THEN** 玩家精力 -1（沿用 `_spendEnergy`），时间推进对应时长
- **AND** 调用 LLM 生成互动场景描述（基于双方性格、关系、当前局势）
- **AND** NPC 对玩家的关系值（5 维：师徒/亲友/政敌/同僚/仇敌）按结果更新
- **AND** 关键互动写入玩家记忆系统

#### Scenario: 互动触发事件钩子
- **WHEN** 玩家与禁军将领关系达到"死党"级别，玩家选择"密谋"

#### Scenario: 联姻
- **WHEN** 玩家选择"联姻"并与目标 NPC 协商成功
- **THEN** 双方家族建立姻亲关系（写入 `family` 字段）
- **AND** 影响 NPC 对玩家势力的态度

### Requirement: 玩家赚钱与私产系统

#### Scenario: 玩家领取官俸

#### Scenario: 玩家贪墨受贿

- **THEN** 玩家银钱 +对应金额
- **AND** 玩家角色 `corruption` 字段累计
- **AND** 吏治腐败引擎（`tm-corruption-engine.js`）增加"被弹劾风险"，玩家可能被言官弹劾

#### Scenario: 玩家投资产业
- **WHEN** 玩家选择"购置产业"并选择产业类型（酒楼/当铺/作坊）

#### Scenario: 玩家被抄家

- **AND** 玩家银钱归零（或仅留基本生活费）

### Requirement: 玩家跑商系统

#### Scenario: 玩家组建商队
- **WHEN** 玩家选择"组建商队"并配置护卫/车马/通关文牒
- **THEN** 玩家银钱 -组建成本
- **AND** 商队状态记入玩家私产
- **AND** 通关文牒需朝廷许可，许可难度关联玩家官场关系

#### Scenario: 商队跨区贸易
- **WHEN** 玩家派遣商队从 A 区到 B 区贸易
- **THEN** 系统根据 `tm-economy-engine.js` 区域价格矩阵计算预期利润

#### Scenario: 大宗贸易影响区域经济

- **THEN** 调用 `tm-region-magnate.js` 影响区域经济指标
- **AND** 区域商品价格动态调整

#### Scenario: 跨朝代通用贸易路线
- **WHEN** 剧本配置特定贸易路线（如丝路/茶马/漕运/海贸）
- **THEN** 引擎读取剧本 hook 数据，按"商队+路线+风险+利润"通用框架执行

### Requirement: 玩家科技研发系统（固定科技路线）

系统 SHALL 在穿越模式下提供科技研发入口，预设固定科技路线（每领域 5 级科技链），前置科技解锁后才能研发后继。

#### Scenario: 玩家启动研发
- **WHEN** 玩家选择"研发"并选定领域（农业/水利/军事/工艺/医药等）
- **THEN** 玩家银钱 -研发投入
- **AND** 系统读取该领域固定科技路线，定位当前可研发的下一级科技
- **AND** 研发进度按 `玩家学识 + 投入资源 + 现有科技基础 + 时代限制` 计算
- **AND** 进度记入玩家科技账本

#### Scenario: 前置科技解锁
- **WHEN** 玩家完成"农具改良"
- **THEN** 解锁后继"良种选育"研发入口
- **AND** 未完成前置时，后继科技研发按钮禁用并提示"需先研发 X"

#### Scenario: 预设固定科技路线
- **WHEN** 引擎启动
- **THEN** 内置默认科技路线（农业/军事/工艺/医药/水利/天文/算学/文学等线，每线 5 级）
- **AND** 剧本数据可覆盖/扩展路线，但不能为空
- **AND** 路线数据结构与 `tm-dynamic-systems.js` 兼容

#### Scenario: 招揽匠人加速研发
- **WHEN** 玩家通过人物互动招揽到匠人/学者
- **THEN** 研发进度加速（按匠人能力加成）
- **AND** 匠人加入玩家门客清单

#### Scenario: 研发成果上奏推广
- **WHEN** 玩家研发完成并选择"上奏推广"
- **THEN** 上奏到皇帝 AI，皇帝 AI 决定是否采纳
- **AND** 采纳后全国获得对应增益（农业增产/军事强军等）

#### Scenario: 私藏自用
- **WHEN** 玩家研发完成并选择"私藏自用"
- **THEN** 增益仅作用于玩家本人/封国
- **AND** 不影响全国指标

#### Scenario: 跨朝代铁律
- **WHEN** 引擎层处理科技研发
- **THEN** 固定科技路线取中国古代通用脉络
- **AND** 剧本数据可加朝代专属支线（如明清火器、宋代活字）

### Requirement: 玩家家族与子女系统

系统 SHALL 在穿越模式下为玩家建立完整家族结构（父母/兄弟/姐妹/配偶/子女/宗族），子女随时间成长并参与家族延续。系统 SHALL 支持玩家婚嫁三种模式（娶妻/嫁女、赘婿、招赘）、再婚、和离，覆盖中国古代婚姻主要礼制路径。

#### Scenario: 玩家结婚（常规娶妻/嫁女）
- **WHEN** 玩家未婚且通过人物互动成功联姻
- **THEN** 玩家 `family.spouse` 字段写入配偶
- **AND** 配偶带入嫁妆/陪嫁/家族关系
- **AND** 双方家族建立姻亲关系
- **AND** 走"纳采/问名/纳吉/纳征/请期/亲迎"六礼流程

#### Scenario: 玩家入赘为赘婿
- **WHEN** 玩家为男性且选择"入赘"路径联姻
- **THEN** 玩家 `family.name` 改为女方姓氏（或保留原姓但身份标注为赘婿）
- **AND** 玩家加入女方家族，地位低于妻家本族
- **AND** 玩家可借妻家势力崛起，但妻家可随时驱逐/羞辱
- **AND** 触发"赘婿逆袭"或"被妻家欺压"两条戏剧线事件钩子

#### Scenario: 玩家招赘
- **WHEN** 玩家为女性或玩家家主为女性，选择"招赘"路径
- **THEN** 赘婿入门加入玩家家族
- **AND** 赘婿地位低于玩家本族
- **AND** 子女归属玩家家族（不随赘婿姓）

#### Scenario: 玩家再婚
- **WHEN** 玩家原配偶死亡或正式和离
- **AND** 玩家度过守制期（父母丧/夫丧/妻丧）
- **THEN** 玩家可再婚，新配偶写入 `family.spouse`（标记 `status: '继室'` 或 `status: '平妻'`）
- **AND** 若玩家带子女再婚，子女与新配偶关系动态生成（慈/严/慈爱/虐待/敌对）
- **AND** 再婚配偶身份默认为继室，特殊情况下可平妻（触发嫡庶之争风险）

#### Scenario: 守制期违规再婚
- **WHEN** 玩家在守制期内强行再婚
- **THEN** 触发礼法风险（言官弹劾、名声下降）
- **AND** 朝廷官员身份可能被罢免（关联官员考核系统）

#### Scenario: 玩家和离/休妻/休夫
- **WHEN** 玩家主动选择和离（双方协议）或休妻（男方单方面需符合七出之条）/休夫（罕见需妻家强势）
- **THEN** `family.spouse` 字段清空（写入前任记录）
- **AND** 子女归属按礼法判定（默认归父方，特殊情况归母方）
- **AND** 双方家族姻亲关系解除，可能触发敌对

#### Scenario: 玩家生育子女
- **WHEN** 玩家有配偶且时间推进到生育周期
- **THEN** 系统生成子女角色（写入 `family.children`）
- **AND** 子女随时间成长：婴幼儿→少年→成年
- **AND** 每个阶段触发对应事件（满月/启蒙/冠礼/及笄）

#### Scenario: 子女教育
- **WHEN** 玩家为子女选择教育方式（延师/亲自教导/送书院）
- **THEN** 子女 `learning` / `intelligence` / `benevolence` 等字段按教育方式更新
- **AND** 送书院关联科举系统（`tm-keju-school-network.js`）

#### Scenario: 子女联姻
- **WHEN** 子女成年且玩家选择"为其联姻"
- **THEN** 关联人物互动系统
- **AND** 成功后扩大势力网络

#### Scenario: 子女出仕
- **WHEN** 成年子女选择入仕路径（科举/荫袭/征辟）
- **THEN** 走对应入仕流程（科举关联 `tm-keju-runtime.js`）
- **AND** 子女成为 NPC 角色，可担任官职，成为玩家政治资源

#### Scenario: 子女继承
- **WHEN** 玩家死亡或被罢黜
- **THEN** 成年子女可继承部分家产/官职/名望
- **AND** 玩家可切换至继承的子女继续游戏（可选）

#### Scenario: 子嗣危机
- **WHEN** 玩家无子嗣
- **THEN** 触发"绝嗣"危机（家族断代风险、旁支夺嫡）
- **AND** 系统提示过继宗族子弟为嗣

#### Scenario: 子女叛逃
- **WHEN** 子女与玩家关系恶化到"仇敌"级别
- **THEN** 子女可能叛逃至敌对势力
- **AND** 玩家家产/官职/机密可能被泄露

### Requirement: 玩家私军系统

系统 SHALL 在穿越模式下允许玩家招募并使用私军，私军独立于朝廷正规军账本，规模超限触发"僭越"风险。

#### Scenario: 招募私军
- **WHEN** 玩家选择"招募私军"并选定类型（家丁/门客剑士/部曲/死士）
- **THEN** 玩家银钱 -招募成本
- **AND** 私军单位写入玩家 `privateArmy` 字段，标记 `kind:'private'`

#### Scenario: 私军维护
- **WHEN** 每月初结算
- **THEN** 玩家银钱/粮草 -维护成本（按私军规模）

#### Scenario: 私军训练
- **WHEN** 玩家选择"训练私军"
- **THEN** 消耗时间/银钱，私军训练度提升

#### Scenario: 私军装备
- **WHEN** 玩家选择"装备私军"并购置兵器/甲胄/战马
- **THEN** 玩家银钱 -装备成本

#### Scenario: 私军使用 - 护卫商队
- **WHEN** 玩家派遣商队并配置护卫私军
- **THEN** 商队山贼劫掠风险降低（按护卫规模/训练度）
- **AND** 私军可能伤亡（按战斗结果）

#### Scenario: 私军使用 - 政变
- **WHEN** 玩家发动反叛
- **THEN** 私军作为反叛主力参战（沿用 `tm-battle-*.js`）
- **AND** 反叛失败则私军被收编/遣散

#### Scenario: 僭越风险
- **WHEN** 玩家私军规模超阈值
- **THEN** 触发"僭越"风险（朝廷调查、言官弹劾）
- **AND** 规模达"拥兵自重"触发朝廷问罪/讨伐

### Requirement: 玩家自由移动系统

系统 SHALL 在穿越模式下允许玩家自由移动到地图上任何已发现地点，移动消耗时间/银钱/精力，地点决定可执行动作集。

#### Scenario: 玩家发起移动
- **WHEN** 玩家在御案选择"移动"并选定目的地
- **THEN** 系统计算路径距离与耗时（沿用 `tm-military.js` 行军路径算法）

- **AND** 玩家 `location` 字段更新

#### Scenario: 移动方式选择
- **WHEN** 玩家选定移动方式（步行/骑马/车驾/舟船/驿站）
- **THEN** 不同方式速度/成本不同
- **AND** 驿站方式需官场关系（关联人物互动）

#### Scenario: 移动事件
- **WHEN** 玩家在移动途中
- **THEN** 可能触发随机事件：盗匪/天气/偶遇 NPC/发现古迹
- **AND** 事件由 LLM 生成（基于路线 + 当前局势）

#### Scenario: 地点决定动作集
- **WHEN** 玩家到达目的地
- **THEN** 御案动作集按地点更新：
  - [ ] 京城：上朝/拜访朝臣/参与朝议
  - [ ] 地方：治理本职辖区/拜访地方士绅
  - [ ] 封国：治理封国/练兵
  - [ ] 边疆：巡视边防/招揽边将
  - [ ] 名胜/隐逸地：游学/招揽名士
  - [ ] 敌国/化外：触发外交事件或被通缉

#### Scenario: 携带随从
- **WHEN** 玩家选择携带家属/私军/商队一起移动
- **THEN** 移动成本按随从规模加成
- **AND** 随从一并更新位置

### Requirement: 玩家产业建设系统

系统 SHALL 在穿越模式下允许玩家在指定地点从无到有建设产业，产业按月产出货物/银钱并有风险。

#### Scenario: 玩家选址建设
- **WHEN** 玩家选择"建设产业"并选定地点与产业类型

#### Scenario: 施工建设
- **WHEN** 玩家选择产业类型并募工
- **THEN** 玩家银钱 -施工成本
- **AND** 施工消耗时间（按产业规模）

#### Scenario: 产业经营
- **WHEN** 每月初结算

#### Scenario: 产业升级
- **WHEN** 玩家选择"升级产业"

- **AND** 玩家需投入资源修复

#### Scenario: 朝廷干预
- **WHEN** 玩家产业规模过大触发"豪强"标签
- **THEN** 朝廷可能强征/抄没
- **AND** 玩家需通过人物互动运作化解

### Requirement: 玩家开垦荒地系统

系统 SHALL 在穿越模式下允许玩家开垦荒地增加当地粮食产出，开垦有官府许可前置与生态副作用。

#### Scenario: 玩家勘探荒地
- **WHEN** 玩家选择"开垦荒地"并选定区域
- **THEN** 系统校验该区域是否有可开垦荒地
- **AND** 显示可开垦规模（小块/中块/大块）与成本预估

#### Scenario: 开垦前置 - 官府许可
- **WHEN** 玩家选择开垦规模并申请
- **THEN** 需获得当地官府许可（关联人物互动 + 官制权限）
- **AND** 违规开垦触发"占田"风险（言官弹劾/强令退还）

#### Scenario: 开垦施工
- **WHEN** 玩家募集流民/民夫并启动开垦
- **THEN** 玩家银钱/粮草 -开垦成本
- **AND** 流程推进：平整土地 → 修水利 → 播种 → 收获
- **AND** 不同规模消耗不同时间（小块 1 月/中块 3 月/大块 6-12 月）

#### Scenario: 开垦产出
- **WHEN** 开垦完成并投产
- **THEN** 新耕地按比例增加当地粮食产量
- **AND** 玩家按投入比例享受分成

#### Scenario: 开垦副作用
- **WHEN** 开垦规模侵占牧场/林地/猎场
- **THEN** 当地势力（牧民/猎户/原住民）不满
- **AND** 大规模开垦触发生态事件（水患/沙化）

#### Scenario: 与朝廷互动
- **WHEN** 玩家开垦有成
- **THEN** 可上奏请功（升迁/赐爵）
- **AND** 开垦失败触发朝廷问责

#### Scenario: 跨朝代通用
- **WHEN** 不同朝代开垦政策（屯田/占田/均田）
- **THEN** 由剧本数据 hook 政策参数
- **AND** 引擎只提供"开垦→产出→风险"通用框架

### Requirement: 玩家参加科举考试

系统 SHALL 在穿越模式下允许玩家以考生身份参与科举全流程，考中后变更 playerRole。

#### Scenario: 玩家报名县试
- **WHEN** 玩家 playerRole 为商贾/隐逸/宗室旁支等且选择"应试"
- **THEN** 玩家进入科举流程（沿用 `tm-keju-runtime.js`）
- **AND** 玩家状态标记为"考生"

#### Scenario: 玩家作答考题
- **WHEN** 玩家进入考场
- **THEN** 题目由剧本数据决定（经义/策论/诗赋）
- **AND** 玩家通过 `tm-keju-question-ui.js` 作答
- **AND** LLM 评卷生成成绩

#### Scenario: 玩家考中进士
- **WHEN** 玩家通过殿试
- **THEN** 玩家身份变更为进士
- **AND** `playerRole` 升级为 `minister`
- **AND** 系统自动授予官职（沿用 `tm-keju-allocation.js`）

#### Scenario: 玩家卷入科场弊案
- **WHEN** 玩家选择"行贿考官"或"请托关节"
- **THEN** 沿用 `tm-keju-scandal.js` 弊案引擎
- **AND** 玩家可能被检举，触发查办/罢黜后果

### Requirement: 玩家官员年终考核

系统 SHALL 在穿越模式下每年末为玩家官员身份生成考核结果，影响升迁/贬谪/加俸/罢黜。

#### Scenario: 年末触发考核
- **WHEN** 每年末（沿用游戏内时间机制）玩家 playerRole 为 minister/general 等
- **THEN** 系统综合玩家 1 年行为生成考核指标：履职情况、政务成绩、廉洁度、人际关系、上级评价、民众口碑
- **AND** LLM 生成考核评语

#### Scenario: 考核结果分九等
- **WHEN** 考核完成
- **THEN** 结果分九等（上上/上中/上下/中上/中中/中下/下上/下中/下下）
- **AND** 按等级触发对应后果：升迁、贬谪、加俸、罚俸、赐物、记过、罢黜

#### Scenario: 玩家主动运作考核
- **WHEN** 玩家在考核前选择"贿赂考官"或"托人情"
- **THEN** 关联人物互动系统
- **AND** 考核指标向上偏移但有被发现风险

#### Scenario: 考核写入编年史
- **WHEN** 考核结束
- **THEN** 考核结果与评语写入编年史
- **AND** 玩家御案显示考核通知

### Requirement: 玩家反叛系统

系统 SHALL 在穿越模式下允许符合身份的玩家发动反叛，反叛成功可转为皇帝模式。

#### Scenario: 反叛前置筹备
- **WHEN** 玩家选择"密谋反叛"
- **THEN** 进入筹备阶段：笼络军权、积累粮草、联络势力、制造舆论
- **AND** 每项筹备消耗时间/银钱/精力
- **AND** 筹备进度可视化（御案显示筹备度）

#### Scenario: 发动反叛
- **WHEN** 玩家筹备度达到阈值，选择"举事"
- **THEN** 反叛类型按 playerRole 分支（宗室夺嫡/藩王起兵/权臣篡位/边将叛乱/农民起义）
- **AND** 触发交战（沿用 `tm-battle-*.js`）
- **AND** 朝廷（皇帝 AI）发动平叛

#### Scenario: 反叛失败
- **WHEN** 玩家反叛军被击败
- **THEN** 玩家被诛（游戏结束）或逃亡（角色变成通缉犯）
- **AND** 家属流放，私产抄没
- **AND** 编年史写入永久污点

#### Scenario: 反叛成功转皇帝模式
- **WHEN** 玩家反叛军攻破京城或废君自立
- **THEN** `P.playerInfo.playerRole` 转为 `emperor`
- **AND** `P.playerInfo.transmigrationMode` 转为 `false`
- **AND** 模式切回皇帝模式（沿用现有皇帝玩法）
- **AND** 编年史写入"新朝建立"事件

### Requirement: 玩家自我技能提升系统

系统 SHALL 在穿越模式下提供玩家自我技能/属性提升路径，玩家可通过学塾/书院、拜师、自学、游学、历练、江湖习武、修道礼佛等场所与方式提升 `learning` / `intelligence` / `valor` / `military` / `administration` / `management` / `charisma` / `diplomacy` / `benevolence` / `diction` 等属性及 `learning` 子项（经史/诗赋/算学/律法/医术/兵法/天文/地理/音律/书法等）。

#### Scenario: 玩家入读学塾/书院
- **WHEN** 玩家在指定地点（如京城/名郡）选择"入学"并选定学塾/书院
- **THEN** 玩家银钱 -学费，进入"在学"状态（按月/季/年消耗时间）
- **AND** 按期提升 `learning` 与子项（按书院侧重：经史书院/武书院/医书院/律书院等）
- **AND** 与 `tm-keju-school-network.js` 复用基础设施，玩家可与同窗建立关系（关联人物互动）

#### Scenario: 玩家拜名师
- **WHEN** 玩家通过人物互动拜得 NPC 名师
- **THEN** 建立师徒关系（写入玩家记忆与 NPC 关系）
- **AND** 按师徒关系获得定向加成（拜名将学兵法、拜名医学医术、拜大儒学经史）
- **AND** 师父可传授独门技艺（如某兵法/某医方/某学说）

#### Scenario: 玩家自学苦读
- **WHEN** 玩家选择"自学"并投入时间 + 银钱（购书/置办文具）
- **THEN** 按概率提升对应属性（效率低于书院/拜师，但灵活）
- **AND** 自学成果受玩家 `intelligence` / 现有 `learning` 影响

#### Scenario: 玩家游学
- **WHEN** 玩家通过自由移动系统到达名胜/古战场/名郡
- **THEN** 触发游学事件（LLM 生成基于地点的见闻/感悟）
- **AND** 游学事件加成对应属性（如游赤壁加 `military`/`learning`，游泰山加 `benevolence`）
- **AND** 游学见闻写入玩家记忆

#### Scenario: 玩家历练
- **WHEN** 玩家完成实际政务/军事/外交任务
- **THEN** 系统自动积累经验，按任务类型提升对应技能（政务→`administration`/`management`，军事→`military`/`valor`，外交→`diplomacy`/`charisma`）
- **AND** 关键突破（如打赢一场硬仗/成功推行一项政策）触发额外加成事件

#### Scenario: 玩家江湖习武
- **WHEN** 玩家 playerRole 为商贾/隐逸/江湖人，选择"入武馆/镖局/门派习武"
- **THEN** 玩家进入"习武"状态（消耗时间/银钱）
- **AND** 提升 `valor` / `military`，可能习得武艺招式
- **AND** 与江湖人物建立关系（关联人物互动）

#### Scenario: 玩家修道/礼佛
- **WHEN** 玩家选择"修道/礼佛"路径
- **THEN** 提升 `benevolence` / `intelligence`，但有副作用（远离世俗、官场关系疏远、可能被指"不思进取"）
- **AND** 修道/礼佛到一定程度触发"出家"危机选项（玩家身份变更）

#### Scenario: 属性上限与年龄影响
- **WHEN** 玩家尝试提升属性
- **THEN** 系统校验当前属性是否达上限（如 100）
- **AND** 提升效率随年龄变化（少年学习快但属性低，老年属性稳定但学习慢）
- **AND** 提升概率按 `当前属性 + 师资 + 投入 + 资质` 计算

#### Scenario: 关键突破事件
- **WHEN** 玩家在某属性达到里程碑（如 50/80/100）或完成某项成就（完成一部著作/拜得名师/悟出某理）
- **THEN** 触发关键突破事件（LLM 生成对应叙事）
- **AND** 事件写入玩家记忆与编年史
- **AND** 可能解锁特殊能力/称号（如"经学大家"/"兵法宗师"/"医术圣手"）

## MODIFIED Requirements

### Requirement: 玩家身份初始化

原逻辑（`tm-launch.js:945, 950-956`）：剧本加载后强制把首位或匹配 `scn.role` 的角色设为 `isPlayer=true`，默认按皇帝处理。

修改后：
- 皇帝模式（开卷）：保留现有逻辑
- 穿越模式：玩家选定角色后，将该角色 `isPlayer=true`，并根据该角色的 `role`/`officialTitle`/`royalRelation` 推导 `P.playerInfo.playerRole`；同时确保剧本中的皇帝角色保持 `_offIsSovereign===true` 且 `isPlayer===false`，写入 `P.playerInfo.sovereignName`

### Requirement: 官制权限判定

原逻辑（`tm-office-system.js:930-963 canPerformAction`）：硬编码 `role==='皇帝'` 即放行所有动作。

修改后：
- 新增 `playerRole` 参数，按角色定位分支：
  - emperor: 全权
  - regent: 部分皇权（任免三品以下、代诏）
  - minister: 仅可任免本职下属、廷推荐举
  - general: 仅可任免本部军官
  - prince: 仅可任免封国官属
  - 后宫/商贾/隐逸: 无任免权
- 调用方（`tm-office-panel.js` 等）传入 `playerRole`，UI 按权限显隐按钮

### Requirement: 朝议入口

原逻辑（`tm-chaoyi.js:20 openChaoyi`）：玩家直接开启朝议并插言。

修改后：
- 皇帝模式：保留现有"插言"路径
- 穿越模式：
  - 玩家不能主动开朝议（朝议由皇帝 AI 或派系触发）
  - 玩家在朝议中"请旨发言"——通过按钮申请，皇帝 AI 决定准否
  - 玩家旁听时显示朝议气泡但不能插言

### Requirement: 回合结束确认

原逻辑（`tm-office-panel.js:1600 confirmEndTurn`）：玩家点"诏令颁行"结束回合。

修改后：
- 皇帝模式：保留"诏令颁行"路径
- 穿越模式：改为"上奏呈递"或"本回合动作完成"，结算时同时执行皇帝 AI 决策与玩家上奏处理

## REMOVED Requirements

### Requirement: 退位/禅让系统在穿越模式下的使用

**Reason**: 穿越模式下玩家不是皇帝，无权禅让。
