## 一、顶层设计 · 北极星 ⭐

天命是什么,一句话:**AI 驱动的历史模拟游戏**。展开成五句话,这是裁决一切的北极星:

| # | 维度 | 结论 |
|---|------|------|
| 1 | **定位** | AI 驱动的历史模拟游戏 |
| 2 | **路线** | **历史的平行宇宙生成器**——真实历史只是开局,之后让 AI 活人把它带向任何方向;"长歪了"(偏离史实)是 **feature 不是 bug**,正是核心爽点。**不做**"重演真实史实"(那条贵在正确性·成本无底洞·自由度低) |
| 3 | **立身之本** | **硬核历史模拟 × 高度自由度**。传统非 AI 游戏这俩只能二选一(要硬核就靠预写规则→锁死自由;要自由就把系统做浅→丢掉硬核)。**AI 是唯一让两者同时拉满的刀**——玩家做设计者没预想的事,AI 当场模拟出硬核、合理、有时代质感的后果。这是别人抄不走的根 |
| 4 | **玩家** | **皇帝**——这个活世界里最大的扰动源(皇权 / 皇威 / 内帑 / 朝议 / 诏令 / 后宫皆其手段) |
| 5 | **命门** | 玩家**无论多自由地乱来,AI 都能持续给出「够硬核 + 自洽不崩 + 有那个时代质感」的回应**。接住 = 神作;接不住(敷衍 / 前后矛盾 / 穿越出戏)= 塌成四不像 |

> **命门 = 裁决一切的尺子。** 任何功能/需求摆上来,先问一句:**它是在加强"AI 在玩家自由行动下给出硬核可信回应"这个核心,还是只在旁边添砖加瓦?** 是核心就做,是添砖就缓。这把尺子直接解"需求太多 / 无从下手"。

## 七、铁律与原则(动手前必读)

1. **跨朝代通用铁律** —— 每动新功能第一步先判"通用 vs 朝代专属"。东厂 / 司礼监 / 内阁 / 票拟 / 八股等明清专名**绝不进引擎**(`tm-*.js`),只归剧本数据(`scenarios/*.js`);引擎只留朝代中立 hook。(详 memory `tianming-engine-cross-dynasty`)
2. **命门当尺子** —— 见第一节,任何需求先用命门量优先级。
3. **改动边界** —— 用户说改 X 只动 X;大改拆 3–5 个小 slice;preview/mockup 与运行时代码两条 review 路径不混。
4. **发版纪律** —— ship / commit / push 都由 owner 显式触发;进行中的大功能整套完工前不 ship;热更**电脑端 + 安卓端两条管线都要发**。
5. **验证纪律** —— 说"完成 / 没问题"前先跑命令实查,别凭印象。
