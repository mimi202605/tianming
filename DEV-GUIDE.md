# 天命 · 开发规范书 / Agent 协作守则

> 用途：交给任意协作 agent（GPT 等）在改动天命代码前**必读**。读完它应当能安全地改动、不踩隐性依赖的雷。
> 配套：现状与归档见 `PROJECT-STRUCTURE.md`，命名切割计划见 `docs/NAMING-PLAN.md`。本文件管"怎么改才不出事"。
> 基准：2026-06-13 重新实测刷新（上版 2026-05-30；半年来代码扩张明显，下列数字均为当日实测）。

---

## 〇、最小心智模型（进来先懂这 6 条）

1. **零编译、纯原生 JS**。没有构建步骤，改完文件即生效（桌面端 Ctrl+R 重载）。不要引入打包器、TypeScript、框架。
2. **没有模块系统**。全仓 `import` / `export` 静态声明 = **0 处**。模块之间不靠 import，全靠**全局对象**互相调用。
3. **两个数据中枢**：`P`（持久化存档数据）和 `GM`（运行时游戏状态：回合/人物/势力/军队/官制树…）。二者定义在 `tm-data-model.js`，必须**最先加载**。全仓对 `GM.` 的引用约 **1.57 万次**、`P.` 约 **4500 次**——几乎每个文件都读写它俩。（统一门面 `DA.*` 已建但采用率仍低，仅约 205 次；新代码优先走 `DA.*`，别再扩大裸 `GM.` 访问，见 `web/ARCHITECTURE.md` §10。）
4. **310 段顺序脚本**串起整个前端（`index.html` 里的 `<script>` 顺序）。**加载顺序 = 依赖关系**：靠前的先就位，后面的才能用它。
5. **全局命名空间**：约 **1260 个** `window.X = ...` 挂载点。一个模块对外暴露的能力，就是它挂在 window 上的那些名字——这些名字是**对外契约**。
6. **回合结算（endturn）是核心管道**，最长、最敏感，改动风险最高。

---

## 一、依赖与加载顺序铁律（最容易踩的雷，重点看）

因为没有 import，"谁依赖谁"**只能靠加载先后**表达。`index.html` 里已有大量顺序契约注释，例如「**须在 endturn 之前加载**」「记忆增强 helpers 须先于推演」。规则：

1. **新增文件**：必须在 `index.html` 注册一行 `<script>`，且放在它所依赖的全局对象**之后**。漏注册 = 整个游戏白屏。
2. **改名 / 移动文件**：必须同步改 `index.html` 里那一行的路径，否则白屏。
3. **不要随意调脚本顺序**。动之前先读该处注释——很多顺序是硬契约（早加载的不能反过来依赖晚加载的）。
4. `tm-data-model.js`（定义 `GM`/`P`）必须在所有业务脚本之前。

---

## 二、命名规范（细则见 NAMING-PLAN.md）

1. 全小写、短横线分隔、**禁下划线**、不带版本号（`-v3` 之类）。
2. 三段式「域-子域-功能」；域前缀白名单：`tm`（核心玩法）、`map`、`editor`、`formal`（新版界面，替代无意义的 `phase8`）、`dev`（调试/工具散件）。

---

## 三、改动前的强制核查（防踩雷，可逐条卡执行 agent）

1. **改任何全局函数/变量前，先全仓搜它的名字**，看被谁调用、确认波及面。没有依赖图，唯一靠谱的办法就是搜符号名。
2. **不破坏全局挂载契约**：别改/删 `window.X` 的挂载名，别人正靠那个名字调你。别名也别随手删（历史上别名删漏过、当场崩）。
3. **GM/P 字段是契约**：加字段安全；**改名或删字段会连锁崩存档**（旧档里有这个字段，读出来对不上）。要废弃字段走兼容保留。
4. **生成文件不手改**：那个约 4.6MB 的官方剧本包是产出物（文件头标了 GENERATED）。要改改源头剧本再重生成。
5. **3 处动态按名加载**（官方剧本投放、语义检索模型、地图编辑器读写）涉及的文件，改名要同步改代码里写死的文件名字符串。
6. **超大文件**（回合结算两支、新版对话那支）拆分要小步、且保持原有全局挂载名不变。

---

## 四、每次改动的收尾清单（执行 agent 必须全过，我据此核验）

- [ ] 新增/改名文件已在 `index.html` 注册，路径正确
- [ ] 已更新该脚本的版本戳（`?v=...` cache-bust），否则玩家热更拉不到新版
- [ ] 改动的全局符号已**全仓搜过**，波及面确认、无遗漏调用点
- [ ] 未改名/删除任何 `GM`/`P` 既有字段（如必须，已做兼容）
- [ ] 语法自检通过
- [ ] 相关 smoke 测试通过（`scripts/` 下）
- [ ] 起动验证：进主菜单 + 读官方剧本 + 存读档正常
- [ ] 改动前已留带时间戳备份（`原名_YYYYMMDD-HHMMSS.扩展名`）

---

## 五、多 Agent 协作守则

1. **一次只动一个域**（按文件前缀分），改完即验，别跨域大扫荡。
2. **并行先认领不同域**，避免两人同时动共享中枢（`GM`/`P`、endturn 管道）——这是最容易互相踩脚的地方。
3. **每次交付三件套说清**：改了哪些文件 / 波及面搜过没 / 验过没（语法+smoke+起动）。
4. **拿不准就停，问，别猜着改**。这套代码没有类型系统兜底，猜错不会报错、只会悄悄坏。

---

## 六、给执行 Agent 的一句话总纲

> 这套代码的危险不在"看不懂"，而在"改了不报错、却悄悄坏"——因为没有编译器、没有类型、没有依赖图替你兜底。所以铁律只有一条：**动手前先搜清楚谁依赖你要动的东西，动完用真实运行来验证，别信"看起来没问题"。**
