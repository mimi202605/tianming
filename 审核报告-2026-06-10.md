### 发版 — 1.3.3.4 是定时炸弹(发版前必须拦下)

3. **1.3.3.4 服务器清单是「12 文件 delta」形态,与客户端契约冲突——1.2.8.9 假更新事故同形复发**

### 工程卫生 — 打包内容物把内部资产发给玩家

4. **E 盘 1.3.3.4 的 APK 是 1057MB,塞进了 Godot 整个工程 + 内部文档 + 备份**

   - **同类问题三条管线各发不同垃圾**:电脑热更 zip 发了 dev-tools/docs/scripts(已发生,1.3.3.3 在服包实锤);capgo 发了 preview 全量 mockup + dev-tools + .bak-r103;安装包 asar 发了 docs + preview mockup + scripts。

5. **官方剧本多副本漂移,内嵌 bundle 落后磁盘源**

   - **处置:确立官方剧本单一真源 + 补回生成脚本。**

## P1 · 重要风险

### 安全
- **`safeEval` 黑名单沙箱可被绕过 → 恶意社区剧本可执行任意 JS**(`web/tm-utils.js:91-100`)。它用正则黑名单封 `constructor/Function/eval`,但 `GM['cons'+'tructor']['cons'+'tructor']('…')()` 这类拼接可绕过,触达 Function 构造器。调用点求值第三方剧本规则表达式(`tm-endturn-helpers.js:492/825/833/901`、`tm-mechanics.js:1245`)。逃逸后可调 preload IPC、外发玩家 key。**应换成白名单表达式解析器。**
- **工坊远程包 hash 校验可跳过**(`main-impl.js:2179`):sha256 仅在 catalog 带 hash 时生效,不带 hash 的包无校验安装。

### 代码质量
- **IPC `turn` 参数漏消毒**(`main-impl.js:1708/1759`):saveName 走 sanitize,但 `String(turn)` 原样进 path.join,可路径遍历读写任意目录。一行可修。
- **53% 的 catch 是空块**(1954/3721 处):多数是可选集成自卫,但关键写账路径也在静默吞错(如 `tm-integration-bridge.js:725` syncAuthorityPhases 失败被 `catch{}` 吞掉,会让已修过的「段位滞留」bug 无声复发)。
- **`tm-endturn-apply.js` 是 4000 行单函数的上帝文件**(338KB/仅 18 函数),每次改账目落点都在同一函数动刀——正是历史上「并行实例 clobber」事故的土壤。建议沿用已验证的「alias+内联」范式拆解。
- **35/302 个 script 无 `?v=` 缓存戳**(含核心 tm-data-model.js),Electron 无碍但在线版会撞「Chromium 缓存无视 no-store」假更新坑。

### 性能(六项已知修复全部在位,无回归)
- **mammoth CDN 同步标签裸奔**(`index.html:62`):`<head>` 里同步加载 jsDelivr,置于全部 291 个本地脚本之前,无 defer/fallback。jsDelivr 被墙时 HTML 解析卡死数十秒。**改 defer 即可,一行零风险。**
- **手动存档未享 autoSave 瘦身刀**(`tm-save-lifecycle.js:570`):autoSave 已做选择性快照,但手动存档/导出仍全量 deepClone+pretty,是 113MB 时代的病。
- **`_prepareGMForSave` 每分钟做一次注定丢弃的整图深拷**(7MB mapData),纯 CPU+GC 浪费。
- **shijiHistory 无封顶**(`tm-endturn-render.js:1362`):每回合 push 整页 HTML,同账本其余历史都有 cap 唯它漏网。

### 数据与发布
- **自用测试剧本随热更发给全体玩家**:`build-hot-update-package.js:226` 无白名单扫 scenarios,1.3.3.4 manifest 实含 `天启七年…自用测试版V1.json`(14.7MB)+ 111.json、崇祯.json 等。安装包反而只打两个官方剧本。
- **双端文件收集规则不一致 + 安卓打包方法不可复现**:capgo 1.3.3.3 曾错杀 2 个运行时文件(22.9MB 语义模型 + 10.4MB 首页动图)。
- **`publish-all.ps1` 与实际发版脱节**:不传 `--include-preview`、打印的 owner 落位命令缺邸报 changelog 同步 + 缺 manifests 落位 → 会邸报 stale + 增量 404。

### 工程卫生 / 版本控制
- **版本控制状态比「没有」更糟**:`web/.git`(785MB)是真仓库但烂尾(5055 松散对象从未 gc),且 **`main-impl.js`(Electron 主进程,含 2GB 内存修/热更逻辑)git ls-files = 0 条,无任何远端副本**。唯一备份是 3 个 .bak。
- **706 个 .bak / 588MB**,40+ 种后缀族(裸 `.bak` 144 个连标签都没有),同文件多代无法对应功能基线;还跟着源树被 cap sync 扩散进 APK。
- **可回收磁盘 ~30GB**:_archive 9.3GB、release-hot 旧包 5.7GB、_codex_tmp+release 3.6GB、安卓 build/镜像 5.6GB 等。
