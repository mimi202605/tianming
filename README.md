# 天命-穿越篇（可扮演非皇帝角色）

在原作（https://github.com/misfit-user/tianming）的基础上， 新增「穿越」模式：玩家在主界面入口选择扮演除皇帝以外的任何人物（朝中重臣/军中将领/一方诸侯/摄政权臣/商贾平民/后宫/太监/宫女/布衣/盗贼/婴儿/退休官员/僧道/匠人/伶人等），皇帝由 AI 全权接管（自动下旨、朝议、处理事务）。玩家在自己职位上发挥应有作用，UI 和功能随角色定位动态调整。同时配套十五大玩家专属系统——人物互动、赚钱私产、跑商、科技研发（固定路线）、家族与子女（含婚嫁/赘婿/再婚/和离）、私军、自由移动、产业建设、开垦荒地、参加科举、官员年终考核、反叛、玩家自我技能提升（学塾/拜师/游学/历练/江湖习武）、特殊身份玩法路线（太监/宫女/布衣/盗贼/婴儿/退休官员/僧道/匠人/伶人）——让玩家在自己职位上"有事可做、有成长感、有戏剧性"。把"硬核 × 自由"的核心爽点从"皇帝视角"扩展到"任何角色视角"，每个角色都是一条新的平行宇宙故事线，反叛成功还可翻转回皇帝视角。

普通玩家角色：

朝中重臣（minister）：
上奏（奏疏面板从"批奏"改为"上奏"）
议政（朝议中发言需先请旨，不能"插言"）
任免建议（不能直接任免，只能廷推/荐举）
处理本职政务（如吏部尚书可拟官制变更建议、户部尚书可拟财政方案）
军中将领（general）：
军事指令（调兵/出征/筑城/补给）需上报请旨
战场指挥全权（沿用 tm-battle-turn）
可上军事奏报
一方诸侯（prince）：
治理封国/封地（独立财政/官制/军事，但需朝贡）
可举兵勤王/叛乱/割据
可上表请旨
摄政权臣（regent）：
复用现有 regentState 摄政机制
可代行部分皇权（需标注"代诏"），但有合法性损耗
君主成年后需还政
宗室/外戚/后宫：
后宫：复用 tm-houguong.js，玩家可影响皇帝决策（枕边风）
宗室：可就藩/参政/谋夺嫡
外戚：可借势揽权/拱卫皇权
商贾/隐逸（merchant/custom）：
经济活动（经商/购置田产/资助势力）
可上书言事，但需通过臣子转呈
可被征辟出仕

特殊身份玩家路线
以下身份有专属玩法路径，与常规 playerRole（emperor/regent/general/minister/prince/merchant/custom）并列。每条路线都有自己的"成长曲线、专有动作、专有风险、晋升/转线可能"，让玩家选任何历史人物都能玩出独特故事。

太监（eunuch）：

内廷宦官身份，无家族延续（不能婚娶生育），但有"养子/义子"路径（收养 NPC 为义子延续势力）
专有动作：讨好皇帝（影响皇帝 AI 决策权重）、把持内廷机构、与外臣勾结（关联派系）、阉党揽权
专有风险：清流弹劾、皇帝猜忌、被诛（宦官专权必遭反噬）
跨朝代铁律：太监具体职务（司礼监/秉笔/掌印/内书堂等）由剧本 hook，引擎只提供"内廷宦官"通用框架
宫女（maid）：

内廷宫女身份，从底层伺候主位起步
专有动作：伺候后宫主位（关系网）、传递消息（影响前朝）、被宠幸（晋升妃嫔，playerRole 转为后宫）、出宫（年龄到限/被放出）
专有风险：宫廷斗争（被诬陷/被毒杀/被罚）、与外臣私通（死罪）
可晋升路径：宫女→女官→妃嫔（被宠幸后）→太后（罕见）
布衣（commoner）：

平民百姓起点，无官职无势力
专有动作：耕读（关联自我技能提升）、经商（关联赚钱/跑商）、游学（关联自由移动）、应募当兵（转为军汉）、科举入仕（关联科举系统）、上书言事（需通过臣子转呈）
专有风险：被豪强欺压、被征徭役、遇灾荒逃荒
多条出路：科举出仕/经商致富/从军立功/落草为盗/皈依宗教
盗贼（bandit）：

落草为寇起点，占据山头/水寨
专有动作：劫掠商队（关联跑商风险反面）、招揽喽啰（类似私军但身份非法）、与江湖势力结盟、打官府（小型战斗）、收保护费
专有路径：被招安（转为正规军将领）、自立为王（建立割据政权，关联反叛系统）、被剿灭（游戏结束）
专有风险：官兵围剿、黑吃黑、被仇家寻仇
婴儿（infant）：

玩家以婴幼儿身份开局（如皇子/宗室子/名门子），无法主动行动
专有机制：由监护人（父母/乳母/太傅）代为决策，玩家只能"观察 + 选择反应"（如哭闹/乖巧/早慧）
成长阶段：婴幼儿→少年→成年，每阶段有成长事件，玩家随年龄增长逐步获得行动权
专有戏剧线：夺嫡之争（若为皇子）、家产之争（若为名门子）、早慧传闻（影响 NPC 期待）
成年后 playerRole 按家族出身转为对应身份（皇子→宗室，名门子→minister/commoner 等）
退休官员（retired_official）：

致仕/罢黜/告老后的官员身份
专有动作：乡居教学（开馆授徒，关联自我技能提升）、干预地方政务（幕后操纵）、撰写回忆录/史书（提升 learning，写入编年史）、被起复（朝廷重新征召）
专有影响：余威犹存（旧部/门生故吏遍布朝野，可调度资源）、被清算（政敌上门报复）
可被朝廷起复：起复后 playerRole 转为 minister/general
僧道（monk，可选）：

出家身份，远离世俗但有专属影响路径
专有动作：修行（提升 benevolence/intelligence）、收徒（建立宗教势力）、影响信众（包括皇帝）、参与政变（以宗教名义）
专有风险：被指妖言惑众、被朝廷清算（灭佛/毁道历史事件）
匠人（artisan，可选）：

手工业者身份，专精某项工艺（陶瓷/纺织/冶铸/造纸等）
专有动作：精进技艺（关联科技研发）、收徒传艺、为朝廷服役（被征辟为官营作坊匠人）、自办作坊（关联产业建设）
可晋升路径：匠户→匠师→工部匠官（playerRole 转为 minister）
伶人（actor，可选）：

戏曲/音乐/歌舞艺人身份
专有动作：献艺（影响皇帝/重臣）、收徒传艺、结交权贵
专有风险：被指"倡优误国"、被权贵强夺、身份低微受歧视
可晋升路径：被宠幸后 playerRole 转为后宫/宠臣
玩家专属系统（穿越模式新增）
以下系统是穿越模式的核心爽点，让玩家在自己职位上"有事可做、有成长感、有戏剧性"。复用现有引擎底层（角色经济/科举/编年/派系），新增玩家主观视角的参与入口。

人物互动系统：

玩家可与任意 NPC 进行一对一互动：拜访、密谈、请托、结交、馈赠、联姻、寻仇、陷害、笼络、收为门客
每次互动消耗玩家精力/时间，影响 NPC 对玩家的关系值（5 维：师徒/亲友/政敌/同僚/仇敌）
互动结果由 LLM 根据双方性格、关系、利益、当前局势生成（复用 tm-wendui.js 一对一对话语料管线）
关键互动写入玩家记忆系统（沿用 tm-memory-*.js），形成长期关系弧线
互动可触发"事件钩子"：如笼络禁军将领到一定程度可发动政变、与后宫结盟可影响皇帝继承人
赚钱系统：

玩家个人私产管理（复用 tm-char-economy-engine.js 的 14 类角色俸禄/私产/名望）
多元收入源：官俸、赏赐、田租、商税分成、灰色收入（贪墨/受贿/索贿）、经营性收入（自营产业）、收藏性资产（古董/字画/珠宝）
投资渠道：购置田产、置办产业（酒楼/当铺/作坊）、放贷收息、囤货居奇
风险机制：贪墨触发吏治腐败引擎（沿用 tm-corruption-engine.js）的"被弹劾"风险；放贷过度触发民怨；囤货触发市舶司调查
与派系勾连：资助某派系换取政治支持，或被派系勒索保护费
玩家个人银钱账本（独立于国库），可被抄家清算（沿用 tm-char-economy-ui.js 的抄家路径）
跑商系统：

玩家可派遣商队（或亲自带队）跨区域贸易：买入低价商品→运输到高价区卖出
复用 tm-economy-engine.js 的区域商品价格矩阵与市场动态
路线风险：山贼劫掠（沿用 tm-coastal-raid.js 模式）、官府盘剥、气候灾害、地方势力索要过路费
商队组建：雇佣护卫（消耗银钱）、购置车马、办理通关文牒（需朝廷许可，可与官场关系挂钩）
大宗贸易可影响区域经济指标（复用 tm-region-magnate.js）
跑商可积累商誉，开启新的商业网络与 NPC 关系
跨朝代通用：丝路/茶马/漕运/海贸等都是剧本数据 hook，引擎只提供"商队+路线+风险+利润"通用框架
科技系统（固定科技路线）：

玩家可主导科技研发（不论 playerRole）：农业/水利/军事/工艺/医药/天文/算学/文学等领域
预设固定科技路线：每个领域有固定科技树，前置科技解锁后才能研发后继
农业线：农具改良 → 良种选育 → 水利灌溉 → 耕作制度 → 多熟种植
军事线：冶铁锻造 → 弩机改良 → 甲胄升级 → 攻城器械 → 火药初探
工艺线：纺织改进 → 陶瓷烧制 → 造纸印刷 → 冶铸高炉 → 雕版活字
医药线：本草整理 → 方剂编纂 → 针灸推拿 → 疫病防治 → 法医检验
水利线：沟渠疏浚 → 陂塘修筑 → 堰坝工程 → 运河开凿 → 海塘修筑
天文/算学/文学线同理（预设固定 5 级科技链）
玩家出资/出人/出时间投入研发，研发成功解锁对应增益（农业增产、军事强军、工艺增收、医药减疫）
研发进度受玩家学识（learning 字段）+ 投入资源 + 现有科技基础 + 时代限制影响
玩家可招揽匠人/学者加速研发（关联人物互动系统）
研发成果可上奏朝廷推广（影响全国），或私藏自用（仅个人/封国受益）
复用 tm-dynamic-systems.js 提供科技树通用框架，默认科技路线内置于引擎（剧本可覆盖/扩展，但不能空）
跨朝代铁律：固定科技路线取中国古代通用脉络，剧本数据可加朝代专属支线（如明清火器、宋代活字）
家族与子女系统：

玩家拥有完整家族结构：父母/兄弟/姐妹/配偶/子女/宗族
复用 tm-char-full-schema.js 的 family / familyMembers 字段
玩家可结婚（若未婚）：通过人物互动系统联姻，配偶带入嫁妆/陪嫁/家族关系
婚嫁三种模式：
娶妻/嫁女（常规）：玩家作为娶方或嫁方家族代表，按常规礼制走"纳采/问名/纳吉/纳征/请期/亲迎"六礼
赘婿（matrilocal）：玩家若为男性且选择"入赘"，则嫁入女方家族，改为女方姓氏（或保留原姓但身份为赘婿），地位较低但可借妻家势力；赘婿路径常用于贫寒男子借势崛起——可触发"赘婿逆袭"或"被妻家欺压"两条戏剧线
招赘（若玩家为女性或玩家家主为女性）：玩家可招赘婿入门，赘婿入玩家家族
再婚：配偶死亡（含战死/病亡/难产/被诛）或正式和离后，玩家可再婚
再婚限制：守制期（父母丧/夫丧/妻丧）内不得再婚，违反触发礼法风险
再婚带子女：鳏夫/寡妇带子女再婚，子女与新配偶关系动态生成（慈/严/慈爱/虐待/敌对）
平妻/继室：再婚配偶身份可为继室（低于原配），特殊情况下可平妻（触发嫡庶之争）
离婚/和离：玩家可主动和离（双方协议）或休妻（男方单方面，需符合七出之条）/休夫（罕见，需妻家强势）；和离后子女归属按礼法判定
玩家可有子女：子女随时间成长（婴幼儿→少年→成年），每个阶段有事件（满月/启蒙/冠礼/及笄）
子女教育：玩家可延师/亲自教导/送书院（关联科举系统），影响子女学识/品性/能力
子女联姻：成年子女可与其他 NPC 家族联姻，扩大势力网络
子女出仕：成年子女可走科举/荫袭/征辟入仕，成为玩家政治资源
子女继承：玩家死亡/罢黜时，子女可继承部分家产/官职/名望
子女叛逃：子女若与玩家关系恶化，可能叛逃敌对势力或自立门户
子嗣危机：无子嗣触发"绝嗣"危机（家族断代、旁支夺嫡）；子嗣过多触发"夺嫡"内斗
玩家自我技能提升系统：

玩家可提升自己的各项技能/属性：learning（学识）/ intelligence（才智）/ valor（勇武）/ military（军事）/ administration（吏治）/ management（治理）/ charisma（魅力）/ diplomacy（外交）/ benevolence（仁德）/ diction（辞令）/ learning 子项（经史/诗赋/算学/律法/医术/兵法/天文/地理/音律/书法等）
提升场所：
学塾/书院：报名入学（交纳学费/通过考核），按期学习提升 learning 与子项；不同书院有不同侧重（经史书院/武书院/医书院/律书院等）；与 tm-keju-school-network.js 复用基础设施
名师拜师：通过人物互动拜 NPC 名师，按师徒关系获得定向加成（如拜名将学兵法、拜名医学医术、拜大儒学经史）
自学苦读：玩家可消耗时间 + 银钱（购书/置办文具）自学，效率较低但灵活
游学：关联自由移动系统，游历名胜/古战场/名郡，触发游学事件加成属性
历练：通过实际政务/军事/外交任务积累经验，自动提升对应技能
江湖习武：若 playerRole 为商贾/隐逸/江湖人，可入武馆/镖局/门派习武，提升 valor / military
修道/礼佛：可选宗教路径提升 benevolence / intelligence，但有副作用（远离世俗、官场关系疏远）
提升机制：每次学习消耗时间 + 银钱 + 精力，按 当前属性 + 师资 + 投入 + 资质 计算提升概率与幅度
属性上限：每个属性有上限（如 100），随年龄变化（少年学习快但属性低，老年属性稳定但学习慢）
提升反馈：每次提升写入玩家记忆与编年史，关键突破（如完成一部著作/拜得名师/悟出某理）触发事件
跨朝代通用：学塾/书院/武馆等具体名目由剧本数据 hook，引擎只提供"场所 + 课程 + 投入 → 属性提升"通用框架
私军系统：

玩家可招募并使用私军（独立于朝廷正规军）
私军类型：家丁（轻装护卫）/ 门客剑士（精锐）/ 部曲（重装）/ 死士（特殊任务）
招募渠道：从流民/镖师/江湖人/退役军士中招募；关联人物互动可招揽名将
维护成本：每月消耗银钱/粮草，规模超限触发财政压力
训练度：私军有训练度字段，定期训练可提升战斗力
装备：玩家可购置兵器/甲胄/战马武装私军（关联私产系统）
使用场景：
护卫商队（关联跑商系统，降低山贼劫掠风险）
自卫（反叛筹备期抵御朝廷围剿）
政变/篡位（关联反叛系统，私军是反叛主力）
私斗（与 NPC 家族械斗）
风险：私军规模超阈值触发"僭越"风险（朝廷调查、言官弹劾）；规模达"拥兵自重"触发朝廷问罪
沿用 tm-military.js / tm-army-units.js 的战斗单位模型，但标记 kind:'private' 独立账本
自由移动系统：

玩家可自由移动到地图上任何已发现的地点（城池/乡村/边疆/名胜/属地等）
移动消耗时间（按距离/交通工具）+ 银钱（路费）+ 精力
移动方式：步行/骑马/车驾/舟船/驿站（不同方式速度/成本不同）
复用 tm-char-full-schema.js 的 location 字段 + tm-military.js 的行军路径算法
玩家当前地点决定可执行动作集：
在京城：可上朝/拜访朝臣/参与朝议
在地方：可治理本职辖区/拜访地方士绅
在封国：可治理封国/练兵
在边疆：可巡视边防/招揽边将
在名胜/隐逸地：可游学/招揽名士
在敌国/化外：触发外交事件或被通缉
移动事件：路上可能遇盗匪/天气/偶遇 NPC/发现古迹
玩家可携带家属/私军/商队一起移动（关联其他系统）
产业建设系统：

玩家可在指定地点建设产业（不同于"购置已有产业"，是从无到有建设）
产业类型：
庄园（综合型：含田地/园林/作坊/库房）
农场（粮食产出）
牧场（牲畜产出，关联骑兵/私军战马）
矿场（金属/煤炭/盐/玉石产出）
林场（木材产出）
渔场（水产产出，需在水边）
工坊（手工业产出：纺织/陶瓷/造纸/冶铸）
商号（商业产出：酒楼/当铺/客栈）
建设流程：选址 → 购地（消耗银钱 + 当地官府许可）→ 募工 → 施工（消耗时间）→ 投产
选址限制：不同地形/区位限制可建产业类型（如矿场只能在矿山附近）
产业经营：每月产出货物/银钱，受管理（玩家或雇管家）+ 民夫/匠人数量 + 治安 + 灾害影响
产业升级：可投入资源升级（扩建/改良/特化）
产业风险：火灾/水灾/盗匪/民夫逃亡/官府强征/敌军劫掠
与朝廷关系：大产业可能触发"豪强"标签，朝廷可能强征/抄没
沿用 tm-building-works.js 的建筑系统底层
开垦荒地系统：

玩家可在指定区域开垦荒地，增加当地可耕地面积与粮食产出
开垦流程：勘探荒地 → 募集流民/民夫 → 平整土地 → 修水利 → 播种 → 收获
开垦规模：小块（百亩）/ 中块（千亩）/ 大块（万亩），不同规模消耗时间/银钱/人力
开垦前置：需获得当地官府许可（关联人物互动 + 官制权限）；违规开垦触发"占田"风险
开垦产出：新耕地按比例增加当地粮食产量，玩家可享分成（按投入比例）
开垦副作用：可能侵占牧场/林地/猎场，触发当地势力不满；大规模开垦触发生态事件（水患/沙化）
与朝廷互动：开垦有成可上奏请功（升迁/赐爵）；开垦失败触发问责
跨朝代通用：不同朝代开垦政策（屯田/占田/均田）由剧本数据 hook，引擎只提供"开垦→产出→风险"通用框架
参加科举考试：

玩家若 playerRole 为商贾/隐逸/宗室旁支/低级官吏子弟，可选择走科举入仕路径
玩家作为考生参与科举全流程：县试→府试→院试（秀才）→乡试（举人）→会试（贡士）→殿试（进士）
复用 tm-keju-*.js 全套（37 文件 ~26k 行），玩家以考生身份接入
考试内容：经义/策论/诗赋（剧本数据决定具体题型），玩家作答后由 LLM 评卷（沿用 tm-keju-question-ui.js）
玩家可拜师求学（关联人物互动）、入书院深造（沿用 tm-keju-school-network.js）
玩家考中后获官职（影响 playerRole 升级为 minister），未中可继续苦读或捐官
玩家可主动卷入科场弊案（关联 tm-keju-scandal.js）：行贿考官、请托关节、枪替冒名——风险与收益并存
官员年终考核：

玩家若 playerRole 为 minister/general 等官员身份，每年末参加考核
复用现有官制系统的"考课"机制（如有），新增玩家视角的考核指标：履职情况、政务成绩、廉洁度、人际关系、上级评价、民众口碑
考核结果分九等（上上/上中/上下/中上/中中/中下/下上/下中/下下）
考核影响：升迁（升级 playerRole 或品级）、贬谪、加俸、罚俸、赐物、记过、罢黜
玩家可主动运作考核（关联人物互动）：贿赂考官、托人情、提前布局政绩
考核结果由 LLM 综合 1 年玩家行为生成评语，写入编年史
反叛系统：

玩家若 playerRole 为 prince/regent/general/minister 均可发动反叛
反叛前置：笼络军权（禁军/边军）、积累粮草、联络内外势力、制造舆论（童谣/谶纬）
反叛类型：
宗室夺嫡：趁君主病重/幼年发动政变
藩王起兵：以"清君侧"名义举兵
权臣篡位：废君自立（沿用 tm-feudal.js 的篡位机制）
边将叛乱：拥兵自立割据一方
农民起义（merchant/custom 角色可达）：聚众举事（沿用 tm-class-radical-revolt.js）
反叛流程：筹备→举事→交战（沿用 tm-battle-*.js）→成败→后续（自立/被诛/逃亡/招安）
反叛失败后果：抄家灭族、家属流放、声名扫地（写入编年史永久污点）
反叛成功后果：玩家成为新君主（playerRole 转为 emperor，模式从穿越切回皇帝模式）
