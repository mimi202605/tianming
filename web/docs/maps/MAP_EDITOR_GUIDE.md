# 专业地图编辑器使用指南

## 概述

专业地图编辑器基于 Leaflet.js 构建，提供 EU4 级别的地图编辑体验。支持手动绘制省份、精确编辑边界、完整的属性管理和多种导出格式。

## 文件说明

### 新增文件
- `map-editor-pro.html` - 专业地图编辑器界面
- `map-editor-pro.js` - 编辑器核心逻辑（约 900 行）
- `map-converter.js` - 格式转换器（约 400 行）

### 现有文件（保留）
- `map-editor.html` / `map-editor.js` - 旧的 Voronoi 编辑器（仍可使用）
- `map-integration.js` - AI 地图理解系统（兼容新编辑器）
- `map-display.js` - 游戏内地图显示（兼容新编辑器）

## 快速开始

### 1. 打开编辑器

在浏览器中打开 `map-editor-pro.html`

### 2. 加载背景图

**方法 A：加载预设地图**
- 点击"加载预设地图"
- 选择：Asia历史地图（默认）、世界地图1、世界地图2

**方法 B：加载自定义图片**
- 点击"加载背景图"
- 选择任意图片文件
- 调节透明度滑块（0-100%）

### 3. 绘制省份

**绘制多边形省份**
- 点击"绘制省份"按钮或按 `P` 键
- 在地图上点击绘制顶点
- 双击完成绘制

**绘制矩形省份**
- 点击"绘制矩形"按钮或按 `R` 键
- 拖拽绘制矩形

### 4. 编辑省份

**选择省份**
- 点击省份选中
- 右侧面板显示属性

**编辑属性**
- 省份名称
- 所属势力
- 颜色
- 发展度（0-100）
- 驻军数量
- 资源列表

**编辑形状**
- 使用右上角的编辑工具栏
- 拖拽顶点调整形状
- 添加/删除顶点

**设置地形**
- 选中省份后，点击右侧地形图标
- 8种地形：平原、山地、森林、沙漠、草原、丘陵、水域、沼泽

### 5. 高级功能

**计算邻接关系**
- 点击"计算邻接关系"
- 自动检测相邻省份
- 用于游戏逻辑（外交、战争等）

**批量编辑**
- 点击"批量编辑"
- 输入省份ID或名称（逗号分隔）
- 选择操作类型（所属势力、地形、发展度等）
- 输入新值并执行

**搜索省份**
- 点击"搜索省份"或按 `Ctrl+F`
- 输入关键词
- 自动定位到匹配的省份

**验证地图**
- 点击"验证地图"
- 检查：重名、无主省份、孤立省份等
- 显示问题列表

**AI 历史地图生成**
- 点击"AI 历史地图生成"
- 输入朝代、年份、主要势力
- 自动为省份分配所属势力

### 6. 导出地图

**导出到游戏格式**
- 点击"导出到游戏格式"或按 `Ctrl+S`
- 保存为 `地图名_game.json`
- 可直接在游戏中使用

**导出 GeoJSON**
- 点击"导出 GeoJSON"
- 保存为标准 GeoJSON 格式
- 可在其他 GIS 软件中使用

## 键盘快捷键

| 快捷键 | 功能 |
|--------|------|
| `P` | 绘制多边形省份 |
| `R` | 绘制矩形省份 |
| `E` | 编辑模式（提示使用工具栏）|
| `D` | 删除模式（提示使用工具栏）|
| `Delete` | 删除选中的省份 |
| `Ctrl+S` | 导出到游戏格式 |
| `Ctrl+F` | 搜索省份 |

## 与游戏系统集成

### 在剧本编辑器中使用

1. 打开 `editor.html`
2. 进入"地图"面板
3. 点击"打开专业地图编辑器"（新窗口）
4. 在专业编辑器中编辑地图
5. 导出为游戏格式
6. 回到剧本编辑器，点击"从编辑器导入地图"
7. 选择导出的 JSON 文件

### 在游戏运行时使用

地图数据会自动加载到 `P.map`，支持：
- 游戏内地图显示（`showMapInGame()`）
- AI 地图理解（`generateMapContextForAI()`）
- 地图影响游戏逻辑（地形影响战斗、距离影响补给等）
- AI 推演更新地图（所属变化、驻军变化、发展度变化）

## 数据格式

### 游戏地图格式

```json
{
  "name": "地图名称",
  "width": 1200,
  "height": 800,
  "regions": [
    {
      "id": "province_1",
      "name": "江南",
      "type": "poly",
      "coords": [x1, y1, x2, y2, ...],
      "center": [x, y],
      "neighbors": ["province_2", "province_3"],

      "terrain": "plains",
      "resources": ["丝绸", "茶叶"],

      "owner": "吴国",
      "characters": ["孙权"],
      "troops": 50000,
      "development": 85,

      "events": "历史事件记录",
      "color": "#4ECDC4"
    }
  ]
}
```

### GeoJSON 格式

标准 GeoJSON FeatureCollection，可在 QGIS、ArcGIS 等软件中使用。

## 格式转换

`map-converter.js` 提供以下转换函数：

```javascript
// Leaflet 编辑器 -> 游戏格式
convertLeafletToGame(leafletData)

// 游戏格式 -> GeoJSON
convertGameToGeoJSON(gameMap)

// 旧 Voronoi 格式 -> 游戏格式
convertVoronoiToGame(voronoiMap)

// GeoJSON -> 游戏格式
convertGeoJSONToGame(geojson)

// 验证地图数据
validateMapData(mapData)
```

## 与旧编辑器的关系

### 旧编辑器（map-editor.js）
- 基于 Voronoi 图自动生成
- 适合快速生成大量省份
- 自动计算邻接关系
- 支持图片导入（边缘检测）

### 新编辑器（map-editor-pro.js）
- 基于 Leaflet.js 手动绘制
- 完全控制省份形状
- 精确描边历史地图
- 专业级编辑体验

### 选择建议
- **快速原型**：使用旧编辑器自动生成
- **精确地图**：使用新编辑器手动绘制
- **历史地图**：使用新编辑器描边
- **两者结合**：旧编辑器生成 -> 导出 -> 新编辑器精修

## 常见问题

### Q: 如何导入旧编辑器的地图？
A: 旧编辑器导出的地图可以直接在新编辑器中通过"导入 GeoJSON"加载，或使用 `convertVoronoiToGame()` 转换。

### Q: 邻接关系如何计算？
A: 点击"计算邻接关系"按钮，系统会自动检测边界相交的省份。建议在完成所有省份绘制后执行。

### Q: 如何确保地图质量？
A: 使用"验证地图"功能检查常见问题，确保所有省份都有名称、所属势力和邻接关系。

### Q: 可以导入真实地图吗？
A: 可以。加载地图图片作为背景，然后手动描边省份边界。这是最精确的方法。

### Q: 地图数据存储在哪里？
A:
- 剧本编辑器：`scriptData.map`
- 游戏运行时：`P.map`
- 导出文件：JSON 格式

## 技术细节

### 依赖库
- Leaflet.js 1.9.4 - 地图渲染引擎
- Leaflet.draw 1.0.4 - 绘制工具

### 坐标系统
使用 Leaflet CRS.Simple（简单坐标系），不使用地理坐标。坐标单位为像素。

### 性能优化
- 支持数百个省份
- 实时渲染和交互
- 分层渲染（背景、省份、标签）

- Firefox
- Safari
