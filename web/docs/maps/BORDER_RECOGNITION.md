# 边界线识别算法 - 技术文档

## 🎯 核心改进

### 问题
之前的颜色分割算法无法正确识别**由黑色边界线围成的地块**，导致识别结果不准确。

### 解决方案
实现**基于边界线的识别算法**，专门识别黑色边界线围成的封闭区域。

## 🔬 算法原理

### 传统方法 vs 新方法

| 方法 | 原理 | 适用场景 | 问题 |
|------|------|---------|------|
| **颜色分割** | 识别相同颜色的区域 | 填充色明显的地图 | 无法识别边界线 |
| **边界线识别** ✨ | 识别黑线围成的区域 | 有明显边界线的地图 | 需要清晰的边界 |

### 算法流程

```
输入图片
    ↓
1. 检测边界线（黑色像素）
    ↓
2. 填充边界线间隙（形态学闭运算）
    ↓
3. 洪水填充非边界区域
    ↓
4. 提取每个封闭区域
    ↓
5. 提取区域边界和颜色
    ↓
输出地块数据
```

## 📐 详细步骤

### 步骤1：检测边界线

**目标**：识别图片中的黑色或深色线条

**方法**：
```javascript
// 计算像素亮度
brightness = 0.299 * R + 0.587 * G + 0.114 * B

// 如果亮度低于阈值，标记为边界线
if (brightness < borderThreshold) {
    borderMap[pixel] = 1; // 边界线
}
```

**参数**：
- `borderThreshold`（0-255）：边界线阈值
  - **0-50**：只识别纯黑线
  - **50-100**（推荐）：识别深色线
  - **100-150**：识别较浅的线
  - **150-255**：识别所有深色区域

### 步骤2：填充边界线间隙

**目标**：连接断开的边界线，形成完整的封闭区域

**方法**：形态学闭运算（膨胀 + 腐蚀）

```javascript
// 膨胀：扩展边界线
for each pixel near border:
    mark as border

// 腐蚀：收缩回原始大小
for each pixel:
    if all neighbors are border:
        keep as border
```

**效果**：
- 连接断开的线条
- 填充小间隙
- 保持边界线粗细

### 步骤3：识别封闭区域

**目标**：找到所有被边界线围成的区域

**方法**：洪水填充算法

```javascript
for each pixel:
    if not border and not visited:
        // 洪水填充找到整个区域
        region = floodFill(pixel)
        if region.size >= minArea:
            regions.push(region)
```

**特点**：
- 避开边界线
- 只填充封闭区域
- 过滤小区域（噪点）

### 步骤4：提取区域边界

**目标**：提取每个区域的精确边界点

**方法**：
1. 找到边界点（至少有一个邻居不在区域内）
2. 按极角排序（顺时针）
3. 生成边界点序列

### 步骤5：提取区域颜色

**目标**：获取每个区域的代表颜色

**方法**：
- 采样区域内的像素
- 计算平均 RGB 值
- 生成颜色字符串

## 🎛️ 参数调优

### 边界线阈值（Border Threshold）

**作用**：控制什么样的像素被识别为边界线

**推荐值**：

| 地图类型 | 推荐值 | 说明 |
|---------|--------|------|
| 纯黑边界线 | 50 | 只识别纯黑色 |
| 深色边界线 | 100 | 识别深灰色线条（推荐）|
| 浅色边界线 | 150 | 识别较浅的线条 |
| 模糊边界 | 200 | 识别所有深色区域 |

**调试技巧**：
- 识别不到边界 → 增加阈值
- 识别出太多噪点 → 减少阈值

### 最小区域面积（Min Area）

**作用**：过滤掉太小的区域（噪点）

**推荐值**：
- **100**：标准设置
- **500-1000**：只保留大区域
- **10-50**：保留所有小区域

### 填充边界线间隙（Fill Gaps）

**作用**：连接断开的边界线

**建议**：
- ✅ **开启**（推荐）：适合大多数地图
- ❌ 关闭：边界线非常清晰时

## 🗺️ 三种识别模式

### 1. 边界线识别（推荐）✨

**适用**：
- 有明显黑色边界线的地图
- 地图集、历史地图
- 手绘地图

**优点**：
- 精确识别边界
- 不受填充色影响
- 适合复杂地图

**示例**：
- Asia历史地图（有黑色边界）
- 世界地图（国家边界清晰）

### 2. 颜色分割识别

**适用**：
- 没有边界线的地图
- 填充色明显不同
- 简单的区域地图

**优点**：
- 识别速度快
- 适合纯色填充

**缺点**：
- 无法识别边界线
- 需要颜色差异明显

### 3. 自动选择

**原理**：
- 分析图片特征
- 检测暗像素比例
- 自动选择最佳算法

**判断标准**：
```javascript
darkRatio = darkPixels / totalPixels

if (darkRatio > 5% && darkRatio < 30%) {
    // 有边界线 → 使用边界线识别
} else {
    // 无边界线 → 使用颜色分割
}
```

## ❓ 常见问题

### Q: 为什么识别不到地块？

**可能原因**：
1. 边界线太浅 → 增加阈值
2. 边界线断开 → 开启"填充间隙"
3. 图片分辨率太低 → 使用高清图片

### Q: 识别出很多小碎片？

**解决方案**：
1. 增加"最小区域面积"
2. 减少"边界线阈值"（更严格）
3. 关闭"填充间隙"

### Q: 边界不准确？

**解决方案**：
1. 调整"边界线阈值"
2. 使用更高分辨率的图片
3. 手动在专业编辑器中精修

### Q: 三张系统地图应该用什么参数？

## 🔮 技术细节

### 边界线检测算法

```javascript
function detectBorders(pixels, width, height, threshold) {
    const borderMap = new Uint8Array(width * height);

    for (let y = 0; y < height; y++) {
        for (let x = 0; x < width; x++) {
            const idx = (y * width + x) * 4;
            const r = pixels[idx];
            const g = pixels[idx + 1];
            const b = pixels[idx + 2];

            // 亮度公式（ITU-R BT.601）
            const brightness = 0.299 * r + 0.587 * g + 0.114 * b;

            if (brightness < threshold) {
                borderMap[y * width + x] = 1;
            }
        }
    }

    return borderMap;
}
```

### 洪水填充算法

```javascript
function floodFillRegion(startX, startY, width, height, borderMap, visited) {
    const stack = [[startX, startY]];
    const regionPixels = [];

    while (stack.length > 0) {
        const [x, y] = stack.pop();

        if (x < 0 || x >= width || y < 0 || y >= height) continue;

        const idx = y * width + x;

        // 跳过边界线和已访问的像素
        if (borderMap[idx] === 1 || visited[idx] === 1) continue;

        visited[idx] = 1;
        regionPixels.push([x, y]);

        // 四个方向扩展
        stack.push([x + 1, y]);
        stack.push([x - 1, y]);
        stack.push([x, y + 1]);
        stack.push([x, y - 1]);
    }

    return regionPixels;
}
```

### 形态学闭运算

```javascript
function fillBorderGaps(borderMap, width, height) {
    const kernel = 2; // 核大小

    // 1. 膨胀（扩展边界）
    const dilated = dilate(borderMap, width, height, kernel);

    // 2. 腐蚀（收缩回原大小）
    const closed = erode(dilated, width, height, kernel);

    return closed;
}
```

## 📈 算法复杂度

| 操作 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 边界检测 | O(W×H) | O(W×H) |
| 形态学运算 | O(W×H×K²) | O(W×H) |
| 洪水填充 | O(W×H) | O(W×H) |
| 边界提取 | O(N) | O(N) |

其中：
- W, H = 图片宽高
- K = 形态学核大小
- N = 区域像素数
