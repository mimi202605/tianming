# 地图系统架构分析报告

## 一、参考项目地图系统概览

### 1. SlgEditor 地图系统（Unity编辑器工具）

**核心架构：**
- **City（城市）**：地图的基本节点，包含 ID、名称、位置、连接的道路
- **Road（道路）**：连接两个城市的边，用于定义相邻关系
- **Edge（边界）**：城市领地的边界线，由多个点组成
- **EdgePoint（边界点）**：边界的顶点，可以是三角形中心点或中间点
- **PolygonMesh（多边形网格）**：城市领地的可视化区域，支持点击和高亮

**数据结构：**
```csharp
// 城市数据
public class DCity {
    public int Id { get; set; }
    public V3 Pos { get; set; }
    public List<int> Roads { get; set; }        // 连接的城市ID
    public List<string> CityEdges { get; set; } // 边界线的key
}

// 边界数据
public class DEdge {
    public string key { get; set; }             // 边界唯一标识
    public List<int> Points { get; set; }       // 边界点ID列表
    public List<int> Cities { get; set; }       // 相邻城市ID列表
}

// 点数据
public class DPoint {
    public int Id { get; set; }
    public V3 Position { get; set; }
}
```

**核心算法：**
1. **Voronoi图生成**：根据城市位置和道路连接，自动生成领地边界
   - 对每个城市，找到所有相邻城市
   - 计算相邻城市之间的角度，按顺时针排序
   - 为每对相邻城市创建三角形，三角形中心作为边界点
   - 连接边界点形成封闭多边形

2. **多边形三角剖分**：将任意多边形转换为三角形网格用于渲染
   - 凸多边形：简单扇形剖分
   - 凹多边形：递归查找可划分顶点，逐步剖分

3. **边界编辑**：支持在边界上添加/删除中间点，调整领地形状

**优点：**
- 自动化程度高，只需放置城市和连接道路，边界自动生成
- 支持任意形状的多边形领地
- 编辑器友好，可视化编辑

**缺点：**
- 依赖Unity引擎，不能直接用于Web
- 算法复杂，需要理解Voronoi图和三角剖分

### 2. KingOfIreland 地图系统（Unity游戏运行时）

**核心架构：**
- **MapData**：地图数据容器，包含城市、边界、点的字典
- **PolygonMesh**：可交互的多边形网格，支持鼠标事件
- **MapManager**：地图管理器，处理鼠标交互和消息分发
- **MeshController**：网格控制器，管理多边形的颜色和高亮

**交互功能：**
```csharp
// 鼠标悬停
public void OnPointerEnter(PointerEventData eventData) {
    MapManager.Instance.SetMouseMeshId(Id);
}

// 鼠标点击
public void OnPointerClick(PointerEventData eventData) {
    MapManager.Instance.LeftClickMesh(Id);
}

// 颜色控制
public void SetColor(Color color) {
    PolyColor = color;
    ResetMeshColor();
}

public void HighlightMeshColor() {
    GetComponent<MeshRenderer>().material.color =
        PolyColor * SelectMixColor + Color.white * (1 - SelectMixColor);
}
```

**优点：**
- 交互体验好，支持悬停、点击、高亮
- 颜色系统完善，支持势力颜色、选中高亮、混合颜色
- 消息系统解耦，地图事件通过消息分发

**缺点：**
- 依赖Unity的3D渲染和物理系统
- 需要预先生成好地图数据

## 二、天命游戏当前地图系统分析

**当前实现（index.html）：**
```javascript
// 地图数据结构
P.map = {
  regions: [
    {
      id: 'region1',
      name: '关中',
      x: 100, y: 100,
      width: 200, height: 150,
      color: '#8B4513',
      owner: '秦国'
    }
  ]
}

// 简单的SVG渲染
function renderMap() {
  var svg = '<svg width="800" height="600">';
  P.map.regions.forEach(function(region) {
    svg += '<rect x="' + region.x + '" y="' + region.y + '" ' +
           'width="' + region.width + '" height="' + region.height + '" ' +
           'fill="' + region.color + '" stroke="#000" />';
  });
  svg += '</svg>';
}
```

**问题：**
1. **矩形地图**：只支持矩形区域，不能表现真实的领地形状
2. **无交互**：不支持点击、悬停、高亮等交互
3. **无边界**：没有领地边界的概念，相邻关系不明确
4. **无自动布局**：需要手动设置每个区域的位置和大小
5. **无编辑器**：只能通过JSON手动编辑地图数据

## 三、改进方案设计

### 方案一：轻量级多边形地图（推荐）

**核心思路：**
借鉴 SlgEditor 的数据结构，但简化算法，使用 Canvas 2D 或 SVG 渲染。

**数据结构：**
```javascript
// 地图数据
GM.mapData = {
  cities: {
    1: { id: 1, name: '长安', x: 400, y: 300, owner: '秦国', neighbors: [2, 3] },
    2: { id: 2, name: '洛阳', x: 600, y: 300, owner: '魏国', neighbors: [1, 3] },
    3: { id: 3, name: '邯郸', x: 500, y: 200, owner: '赵国', neighbors: [1, 2] }
  },
  polygons: {
    1: { cityId: 1, points: [{x: 300, y: 200}, {x: 500, y: 200}, ...] },
    2: { cityId: 2, points: [{x: 500, y: 200}, {x: 700, y: 200}, ...] },
    3: { cityId: 3, points: [{x: 400, y: 100}, {x: 600, y: 100}, ...] }
  }
};
```

**渲染方案：**
```javascript
// 使用 Canvas 2D 渲染多边形
function renderMapCanvas() {
  var canvas = document.getElementById('mapCanvas');
  var ctx = canvas.getContext('2d');

  // 渲染每个领地
  Object.values(GM.mapData.polygons).forEach(function(polygon) {
    var city = GM.mapData.cities[polygon.cityId];
    var faction = findFacByName(city.owner);

    // 绘制多边形
    ctx.beginPath();
    polygon.points.forEach(function(point, index) {
      if (index === 0) {
        ctx.moveTo(point.x, point.y);
      } else {
        ctx.lineTo(point.x, point.y);
      }
    });
    ctx.closePath();

    // 填充颜色
    ctx.fillStyle = faction ? faction.color : '#cccccc';
    ctx.fill();

    // 绘制边界
    ctx.strokeStyle = '#000000';
    ctx.lineWidth = 2;
    ctx.stroke();

    // 绘制城市名称
    ctx.fillStyle = '#000000';
    ctx.font = '14px Arial';
    ctx.textAlign = 'center';
    ctx.fillText(city.name, city.x, city.y);
  });
}
```

**交互实现：**
```javascript
// 点击检测（点在多边形内算法）
function isPointInPolygon(x, y, polygon) {
  var inside = false;
  var points = polygon.points;

  for (var i = 0, j = points.length - 1; i < points.length; j = i++) {
    var xi = points[i].x, yi = points[i].y;
    var xj = points[j].x, yj = points[j].y;

    var intersect = ((yi > y) !== (yj > y)) &&
                    (x < (xj - xi) * (y - yi) / (yj - yi) + xi);
    if (intersect) inside = !inside;
  }

  return inside;
}

// 鼠标点击事件
canvas.addEventListener('click', function(e) {
  var rect = canvas.getBoundingClientRect();
  var x = e.clientX - rect.left;
  var y = e.clientY - rect.top;

  // 查找点击的领地
  Object.values(GM.mapData.polygons).forEach(function(polygon) {
    if (isPointInPolygon(x, y, polygon)) {
      var city = GM.mapData.cities[polygon.cityId];
      showCityInfo(city);
    }
  });
});

// 鼠标悬停高亮
canvas.addEventListener('mousemove', function(e) {
  var rect = canvas.getBoundingClientRect();
  var x = e.clientX - rect.left;
  var y = e.clientY - rect.top;

  GM.hoveredCityId = null;

  Object.values(GM.mapData.polygons).forEach(function(polygon) {
    if (isPointInPolygon(x, y, polygon)) {
      GM.hoveredCityId = polygon.cityId;
    }
  });

  renderMapCanvas(); // 重新渲染以显示高亮
});
```

**高亮效果：**
```javascript
function renderMapCanvas() {
  // ... 基础渲染 ...

  // 高亮悬停的领地
  if (GM.hoveredCityId) {
    var polygon = GM.mapData.polygons[GM.hoveredCityId];
    ctx.beginPath();
    polygon.points.forEach(function(point, index) {
      if (index === 0) ctx.moveTo(point.x, point.y);
      else ctx.lineTo(point.x, point.y);
    });
    ctx.closePath();

    // 半透明白色覆盖层
    ctx.fillStyle = 'rgba(255, 255, 255, 0.3)';
    ctx.fill();

    // 加粗边界
    ctx.strokeStyle = '#ffffff';
    ctx.lineWidth = 3;
    ctx.stroke();
  }
}
```

**优点：**
- 纯Web技术，无需Unity
- 支持任意形状的多边形领地
- 交互体验好，支持点击、悬停、高亮
- 性能好，Canvas 2D 渲染效率高

**缺点：**
- 需要手动定义多边形顶点（可以用编辑器辅助）
- 不支持自动生成Voronoi图（可以后续添加）

### 方案二：简化矩形地图增强版

**核心思路：**
保持矩形地图，但增加交互和视觉效果。

**改进点：**
1. **添加交互**：点击、悬停、高亮
2. **势力颜色**：根据归属自动设置颜色
3. **边界线**：绘制相邻区域的边界
4. **缩放拖拽**：支持地图缩放和拖拽

**实现示例：**
```javascript
// 增强的矩形地图
function renderEnhancedMap() {
  var canvas = document.getElementById('mapCanvas');
  var ctx = canvas.getContext('2d');

  P.map.regions.forEach(function(region) {
    var faction = findFacByName(region.owner);

    // 绘制矩形
    ctx.fillStyle = faction ? faction.color : '#cccccc';
    ctx.fillRect(region.x, region.y, region.width, region.height);

    // 高亮悬停
    if (GM.hoveredRegionId === region.id) {
      ctx.fillStyle = 'rgba(255, 255, 255, 0.3)';
      ctx.fillRect(region.x, region.y, region.width, region.height);
    }

    // 绘制边界
    ctx.strokeStyle = '#000000';
    ctx.lineWidth = 2;
    ctx.strokeRect(region.x, region.y, region.width, region.height);

    // 绘制名称
    ctx.fillStyle = '#000000';
    ctx.font = '14px Arial';
    ctx.textAlign = 'center';
    ctx.fillText(region.name, region.x + region.width / 2, region.y + region.height / 2);
  });
}
```

**优点：**
- 实现简单，改动小
- 保持现有数据结构

**缺点：**
- 仍然是矩形，不够真实
- 布局不灵活

## 七、技术选型对比

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| Canvas 2D | 性能好、兼容性好、API简单 | 不支持DOM事件、需要手动管理交互 | ⭐⭐⭐⭐⭐ |
| SVG | 支持DOM事件、可缩放、易于调试 | 大量元素时性能差 | ⭐⭐⭐⭐ |
| WebGL | 性能最好、支持3D效果 | 学习曲线陡峭、过度设计 | ⭐⭐ |
| 第三方库（Leaflet/Mapbox） | 功能完善、开箱即用 | 体积大、不够灵活 | ⭐⭐⭐ |

**推荐：Canvas 2D + SVG混合**
- 使用 Canvas 2D 渲染地图主体（性能）
- 使用 SVG 渲染UI元素（交互）
