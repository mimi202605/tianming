# 天命游戏 - 数据监听系统说明

## 概述

数据监听系统借鉴了 KingOfIreland 项目的数据驱动架构，并适配中国古代历史背景。该系统实现了属性变化的自动监听和级联更新，大幅提升了游戏的自动化程度和数据一致性。

## 核心特性

### 1. 响应式属性系统

通过 JavaScript 的 `Object.defineProperty` 实现属性的自动监听：

```javascript
// 创建响应式属性
makeReactive(entity, entityType, propertyName, initialValue);

// 批量创建
makeEntityReactive(entity, entityType, ['loyalty', 'ambition', 'age']);
```

### 2. 监听器注册

为特定实体类型的属性注册监听器：

```javascript
// 注册监听器
registerListener('character', 'loyalty', function(char, prop, oldVal, newVal) {
  if (newVal < 30 && oldVal >= 30) {
    addEventLog('⚠️ ' + char.name + '忠诚度过低');
  }
}, priority);
```

### 3. 变化队列处理

所有属性变化先进入队列，然后批量处理，避免重复计算：

```javascript
// 在 endTurn() 结束时调用
processChangeQueue();
```

## 已实现的监听器

### 角色相关

1. **忠诚度监听** (`character.loyalty`)
   - 忠诚度 < 30：触发警告
   - 忠诚度恢复 ≥ 50：记录恢复

2. **野心监听** (`character.ambition`)
   - 野心 > 70：触发关注提示

3. **官职监听** (`character.position`)
   - 任命/罢免：自动触发权力重分配

### 势力相关

4. **财政监听** (`faction.money`)
   - 财政赤字：降低经济繁荣度
   - 财政充裕：提升经济繁荣度

5. **粮食监听** (`faction.food`)
   - 粮食短缺：降低民心

6. **民心监听** (`faction.popularity`)
   - 民心过低：降低社会稳定度
   - 民心高涨：提升社会稳定度

### 时代状态相关

7. **集权度监听** (`eraState.centralControl`)
   - 集权度变化 > 10%：自动重新计算经济系统

8. **朝代阶段监听** (`eraState.dynastyPhase`)
   - 阶段转变：触发对应历史事件
   - 开国 → 扩张 → 盛世 → 衰落 → 崩溃

### 岗位相关

9. **政绩监听** (`post.performance`)
   - 政绩优秀 ≥ 80：记录表彰
   - 政绩不佳 < 40：提示调整

### 军队相关

10. **士气监听** (`army.morale`)
    - 士气低落 < 30：战斗力下降警告
    - 士气高昂 ≥ 70：记录鼓舞

## 级联更新机制

### 示例 1：财政变化的级联效应

```
财政赤字 (faction.money < 0)
  ↓
触发监听器
  ↓
降低经济繁荣度 (eraState.economicProsperity)
  ↓
触发时代状态更新
  ↓
可能影响贡奉比例
```

### 示例 2：集权度变化的级联效应

```
集权度上升 (eraState.centralControl)
  ↓
触发监听器
  ↓
重新计算经济系统 (recalculateEconomy)
  ↓
调整贡奉比例
  ↓
影响各地区财政
```

### 示例 3：官职变化的级联效应

```
角色就任新职 (character.position)
  ↓
触发监听器
  ↓
重新计算权力结构 (recalculatePowerStructure)
  ↓
更新权力分布
  ↓
可能影响政治稳定度
```

## 中国古代背景适配

### 1. 集权度与贡奉系统

- **低集权时期**（< 0.3）：地方割据，贡奉比例低（0.1-0.3）
- **中等集权**（0.3-0.7）：中央与地方平衡，贡奉比例中等（0.3-0.6）
- **高集权时期**（> 0.7）：中央集权，贡奉比例高（0.6-0.9）

### 2. 朝代阶段事件

- **开国**：开国大典、分封功臣
- **扩张**：开疆拓土、文治武功
- **盛世**：盛世繁华、万国来朝
- **衰落**：国势衰微、内忧外患
- **崩溃**：王朝末路、改朝换代

### 3. 官制联动

- 任命官员：自动更新权力结构
- 罢免官员：触发继承或铨选
- 政绩考核：三年一考，自动评估

### 4. 时代感知

所有监听器都会根据当前时代状态（集权度、正统性来源、王朝阶段）调整行为：

```javascript
// 低集权时期：血统和地方支持更重要
if (centralControl < 0.3) {
  legitimacyWeight = 0.5;
  abilityWeight = 0.2;
}

// 高集权时期：能力和制度更重要
if (centralControl > 0.7) {
  legitimacyWeight = 0.3;
  abilityWeight = 0.5;
}
```

## 使用示例

### 1. 为新角色添加响应式属性

```javascript
// 创建新角色
var newChar = {
  name: '张三',
  loyalty: 80,
  ambition: 50,
  age: 30
};

// 添加到游戏
GM.chars.push(newChar);

// 使其响应式
makeNewEntityReactive(newChar, 'character');

// 现在修改属性会自动触发监听器
newChar.loyalty = 25; // 自动触发警告
```

### 2. 安全更新属性

```javascript
// 使用安全更新函数（兼容响应式和非响应式）
safeUpdateProperty(character, 'loyalty', 60);
```

### 3. 注册自定义监听器

```javascript
// 在 initDataListeners() 中添加
registerListener('character', 'health', function(char, prop, oldVal, newVal) {
  if (newVal < 20 && oldVal >= 20) {
    addEventLog('🏥 ' + char.name + '健康状况堪忧');
  }
}, 2);
```

## 性能优化

### 1. 批量处理

所有变化先进入队列，在回合结束时统一处理，避免重复计算。

### 2. 优先级控制

监听器按优先级排序执行（数字越小优先级越高）：
- 优先级 1：核心系统（集权度、朝代阶段）
- 优先级 2：次要系统（财政、民心、士气）

### 3. 条件触发

只有当属性值真正变化时才触发监听器：

```javascript
if (oldValue === newValue) return; // 跳过
```

## 注意事项

1. **避免循环依赖**：监听器不应该修改触发它的属性
2. **性能考虑**：不要在监听器中执行耗时操作
3. **数据一致性**：确保所有属性修改都通过响应式系统
4. **兼容性**：使用 `safeUpdateProperty` 确保兼容旧代码

✅ **历史感**：深度适配中国古代背景，增强沉浸感
