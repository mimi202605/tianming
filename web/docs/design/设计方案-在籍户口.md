## 八决策结论

| # | 决策 | 选择 |
|---|------|-----|
| Ⅰ | 单位精度 | **户 + 口 + 丁 三元** |
| Ⅱ | 户口分类 | **完整色目户计**（10 类 + 自定义，按朝代启用）|
| Ⅲ | 户籍状态 | **黄籍 / 白籍 / 侨置 / 逃户 / 隐户 五态**（名义 vs 实际）|
| Ⅳ | 户等制 | **按朝代可配**（唐九等 / 宋五等 / 明十等 / 自定义）|
| Ⅴ | 徭役系统 | **完整**（兵役 / 工役 / 漕役 + 折银 + 豁免 + 代役）|
| Ⅵ | 兵役分类 | **五种**（禁军 / 府兵 / 厢军 / 募兵 / 军户）|
| Ⅶ | 人口动态 | **多因素**（出生 / 死亡 / 瘟疫 / 灾荒 / 战乱 / 迁徙 / 逃亡）|
| Ⅷ | 历史预设 | **完整**（各朝代户口典型值 + 9 套制度模板）|

## 四项细则（用户已确认）

1. **丁年龄范围**：剧本可配 — 汉 15-56 / 唐 21-59 / 宋 20-60 / 明 16-60
2. **户籍状态流动阈值**：徭役负担 > 40% 触发黄籍 → 逃户，阈值剧本可调
3. **户等制朝代切换**：保留前朝评级，改革性事件（清丈田亩/造黄册）触发重评
4. **瘟疫/战乱死亡**：计入口减少，独立字段记录便于史记追溯

## Ⅰ 户+口+丁 三元详方

### 概念
- **户** `households`：家庭登记单位
- **口** `mouths`：总人口（男+女+老+幼）
- **丁** `ding`：成年男子（tax/corvee/military 核心基数）

### 丁年龄（剧本可配）
```js
// sc.populationConfig.dingAgeRange
// 朝代默认
DYNASTY_DING_AGE = {
  '汉': [15, 56],
  '唐': [21, 59],
  '宋': [20, 60],
  '明': [16, 60],
  '清': [16, 60],  // 摊丁入亩前
  'default': [16, 60]
}
```

### 派生比例（初始化时）
```js
mouths / households ≈ {
  汉: 5.8, 唐: 5.3, 宋: 5.2, 元: 4.5, 明: 5.0, 清前: 5.2, 清后: 4.3
}
ding / mouths ≈ 0.28-0.35（按户均构成）
```

## Ⅲ 户籍状态五态详方

```
正常流动图：
      黄籍（正式）
       ↓ 徭役重/灾荒/战乱
      逃户（在册未实）
       ↓ 豪强收容
      隐户（不在册）
       ↑ 户口清查/摊丁
      黄籍
       
侨置专线（南北朝）：
      北方黄籍 → 南渡 → 白籍 → 土断 → 侨置黄籍
```

### 状态数值与影响

| 状态 | 纳税 | 服役 | 黄册统计 | 动态 |
|------|------|------|---------|------|
| 黄籍 | ✓ | ✓ | ✓ | 主体 |
| 白籍 | 半 | 半 | ✓但分开 | 南北朝限 |
| 侨置 | ✓（轻）| ✓ | ✓ | 南北朝限 |
| 逃户 | ✗（虚摊）| ✗ | ✓但不可征 | 王朝末期爆发 |
| 隐户 | ✗ | ✗ | ✗（不在册）| 豪强势大时爆发 |

### 转换公式

```js
// 黄籍 → 逃户（每年）
var escapeRate = Math.max(0, corveeBurden - burdenThreshold) * 0.5
               + famineLevel * 0.3
               + warBoost * 0.4
               - benevolenceModifier * 0.2;
huangjiToTaohu = huangji.mouths * clamp(escapeRate, 0, 0.08);

// 逃户 → 隐户（豪强收容）
yinhuAbsorb = taohu.mouths * landlordInfluence * 0.3;

// 隐户 → 黄籍（清查）
// 玩家诏令"大造黄册"/"摊丁入亩" 触发
```

### 实际 vs 名义差

```js
nominalRegistered = huangji + qiaozhi + baiji + taohu  // 在册
actualTaxable     = huangji + qiaozhi + baiji          // 实际可征
realPopulation    = nominalRegistered + yinhu          // 真实人口（包含隐户）

// 两个重要指标
registrationGap = actualTaxable / realPopulation       // 1.0 = 造册完美；<0.6 王朝晚期
taxEfficiency  = actualTaxable / nominalRegistered    // 1.0 = 无虚摊
```

## Ⅳ 户等制详方

### 按朝代预设

**唐九等**（上上/上中/上下/中上/中中/中下/下上/下中/下下）
- 按资产田亩评定
- 上上户承担更多徭役

**宋五等**（上户/中户/下户/贫户/客户）
- 主户五等 + 客户不入等（客户不纳田赋）
- 按田亩资产

**明十等**
- 按田亩分
- 黄册登记户等

**清前期**（弱化）
- 摊丁入亩后户等制度萎缩

### 数据结构
```js
byGrade: {
  tier_1: { label:'上上户', households:120000, taxBurden:1.5, corveeBurden:1.8 },
  tier_2: { label:'上中户', households:180000, taxBurden:1.3, corveeBurden:1.5 },
  ...
}
```

### 切换朝代
保留前朝评级，直到触发：
- "大造黄册"诏令
- 重大改革（两税法/一条鞭）
- 朝代更迭 → 自动重评

```js
    totalDays: sum, fulfilled, commutedRate
  },
  caoyi:  { 
    daysPerDing:5,          // 漕役（漕运）
    totalDays: sum, fulfilled, commutedRate
  }
}
```

### 豁免规则

```js
corvee.exemptions = [
  { group:'官员（九品以上）', multiplier:0 },
  { group:'僧道（有度牒）', multiplier:0 },
  { group:'生员/举人', multiplier:0.5 },
  { group:'军户',   multiplier:0 },      // 免徭役担兵役
  { group:'匠户',   multiplier:0 },      // 免徭役担匠役
  { group:'孝廉/节妇', multiplier:0.3 },
  { group:'皇庄户', multiplier:0.2 }
]
```

### 折银机制

```js
// 唐庸：三日役折绢三尺
// 明一条鞭：全部折银
// 清摊丁入亩：丁役并入田赋

function calculateCorveeCommutation() {
  if (corvee.fullyCommuted) {
    // 一条鞭后
    return { money: totalDing * annualDingTax, grain:0 };
  }
  
  var totalDays = sum of all types;
  var marketLaborPrice = GM.currency.market.laborDayPrice;  // 与经济联动
  var commutedDays = totalDays * corvee.commutationRate;
  
  return {
    money: commutedDays * marketLaborPrice,
    actualServedDays: totalDays * (1 - corvee.commutationRate)
  };
}
```

### 代役

```js
corvee.proxyRate = 0.2  // 20% 徭役由富户雇人代服
// 高代役率：社会阶层分化严重、贫富悬殊
// 代役费市价联动
```

### 接入财政 P.2 百姓负担

`paymentConversionMult` 在 P.2 公式中，徭役折钱部分由此系统动态提供：
```js
peasantBurdenStructure.paymentConversion = 
  corvee.commutationRate * (1 + marketPriceGap);
```

### 可征兵基数（每年）

```js
military.totalPool = 
  Σ(region.ding) 
  - 官员丁 - 僧道丁 - 匠户丁 - 军户丁（已入军）
  - 老病残;
```

### 扩军限制
```js
newRecruits <= military.totalPool * maxExpansionRate;  // 默认 10%/年
// 过度征兵 → 逃户率升
```

### 与将来军事系统接口

兵役数据是未来完整军事系统的输入。本子系统只管"可征兵数"+"5 种兵员库"+"补员逻辑"，不管战斗/后勤/将领。

## Ⅶ 人口动态详方

### 出生率

```js
birthRate = birthRateBase                  // 约 0.035
          × (1 + prosperity * 0.3)        // 富则多子
          × (1 - disruption * 0.5)        // 乱则减
          × (1 + fertilityBonus)          // 政策（鼓励生育）
          × (1 - warBoost * 0.3);         // 战乱
```

### 死亡率

```js
deathRate = deathRateBase                  // 约 0.025
          × (1 + agingFactor * 0.2)       // 老龄化
          × (1 + famineBoost * 2)         // 饥荒可放大 2-5 倍
          × (1 + plagueBoost * 3)         // 瘟疫放大 3-10 倍
          × (1 + warCasualtyRate)         // 战亡
          × (1 - medicineLevel * 0.2);    // 医学水平
```

### 瘟疫单独字段

```js
population.dynamics.plagueEvents = [
  { startTurn, region, severity, deathsThisTurn, totalDeaths, endTurn }
]
// 计入口减少，但独立记录便于史记
```

### 战亡单独字段

```js
population.dynamics.warCasualties = [
  { year, region, battle, deaths, conscriptedCasualties, civilianDeaths }
]
```

### 迁徙

```js
function _calcMigration(regionA, regionB) {
  var push = regionA.disruption * 0.3
           + regionA.corveeBurden * 0.2
           + regionA.famineBoost * 0.4
           + regionA.warBoost * 0.5;
  
  var pull = regionB.prosperity * 0.3
           - distance(A, B) / 1000
           + regionB.asylum * 0.2;  // 某些侨置郡县主动招揽
  
  var flow = regionA.mouths * Math.max(0, push - pull) * 0.02;
  return flow;
}
```

### 年度日志

```js
population.dynamics.yearlyLog.push({
  year, 
  total: { mouths, ding },
  births, naturalDeaths, plagueDeaths, warDeaths, famineDeaths,
  netMigration,
  fugitiveChange, hiddenChange,
  byRegion: {...}
})
```
