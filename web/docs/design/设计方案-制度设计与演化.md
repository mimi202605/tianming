# 天命 · 制度设计与演化系统

> 起草：2026-04-17
> 状态：设计定稿
> 定位：横向增强模块，覆盖已有五大制度域（货币/税制/户籍/徭役/兵制）的"运行时玩家自创"能力
> 核心理念：**玩家自由写诏书（自然语言），AI 负责理解，Help 页面作为格式指南**

---

## 一、设计哲学

### 问题意识
已有各子系统在**编辑器层**支持"自定义"（custom_* 通道），但**游戏运行时**玩家被锁定在历史预设改革库中。这违背了本游戏"剧本仅为开端、玩家主观能动性完全自由"的核心设计原则。

### 解决方案（非侵入式）
**不使用弹窗制度设计器**（会打断沉浸感），而是：

```
① 玩家在主界面用自然语言写诏书（完全自由）
   ↓
② 后台 AI 解析诏书，识别制度类型与参数
   ↓
③ 非侵入式小卡片：AI 摘要解读 + 历代类推 + 预估廷议
   ↓ 玩家确认
④ 入廷议 → 试点 → 推广/失败 → 生效/废除
```

### Help 页面"诏书问对"作为参考

玩家写诏书时可随时打开左侧帮助栏的"诏书问对"主题查阅建议格式。**非强制**——玩家可完全忽略，按自己想法写。格式指南只是帮助玩家写得更清楚，让 AI 解读更准确。

```js
  1. 类型（货币/税种/户籍/徭役/兵制/其他）
  2. 关键参数（尽量提取，按类型字段）
  3. 历代类推（给出 1-3 条历史类似诏书）
  4. 可行性预估（0-100）

返回 JSON：
{
  "type": "currency" | "tax" | "household" | "corvee" | "military" | "other",

  return await _callClaudeAPI(prompt);
```

## 三 · 诏书判定分流（AI 直断 vs 有司复奏/求见）

### 核心判定原则（默认宽松，慎触发复奏）

**默认倾向直断**，只在"**空泛+不急+重要**"三者合流时才触发复奏。AI 不抬杠，能补则补。

```
诏书 → AI 评估三维度：完整度 × 紧急度 × 重要度
    ↓
┌──────────────────────────────────────┐
│ 完整度 > 0.6          → 直断（AI 补缺）│
│ 完整度 < 0.3          ┐               │
│ + 重要度 > 0.7        ├ → 复奏/求见   │
│ + 紧急度 < 0.5        ┘               │
│ 完整度 < 0.1 + 意图模糊 → AI 追问       │
│ 紧急度 > 0.7           → 强制直断      │
│ 其他                    → 直断         │
└──────────────────────────────────────┘
```

### 三维度的计算

```js
// 完整度：关键字段覆盖率
var completeness = (filledCriticalFields) / (requiredCriticalFields);

// 紧急度
function _assessUrgency(decree, context) {
  var u = 0;
  if (context.activeWar)    u += 0.4;
  if (context.activeFamine) u += 0.4;
  if (context.activePlague) u += 0.3;
  if (context.activeRevolt) u += 0.5;
  
  // 回合时间跨度（短=紧张）
  var dpt = sc.timeConfig.daysPerTurn;
  if (dpt <= 3)   u += 0.3;
  if (dpt <= 30)  u += 0.1;
  if (dpt > 180)  u -= 0.2;
  
  // 诏书关键词
  if (decree.text.match(/赈|救|急|速/))  u += 0.3;
  if (decree.text.match(/朕欲.*改.*制/)) u -= 0.2;
  
  return clamp(u, 0, 1);
}

// 重要度：涉面 + 不可逆
function _assessImportance(decree) {
  var i = 0;
  if (decree.scope === 'national')  i += 0.7;
  if (decree.scope === 'province')  i += 0.5;
  if (decree.scope === 'local')     i += 0.2;
  if (decree.changesSystemRules)    i += 0.3;   // 改制度规则
  if (decree.irreversible)           i += 0.2;
  return clamp(i, 0, 1);
}
```

### 通路选择（奏疏 vs 求见）

```js
function _pickPathway(reform, drafter, context) {
  // AI 自定（默认）
  if (reform.sensitivity > 0.7) return 'audience';          // 敏感 → 密奏
  if (reform.isFundamental) return 'memorial';               // 大制度 → 正式奏疏
  if (drafter.intimacyWithPlayer > 0.7) return 'audience';  // 亲近 → 求见
  if (reform.urgency > 0.8) return 'audience';               // 急切 → 求见
  
  // 玩家偏好可锁定
  if (player.preferences.pathway === 'memorial_only') return 'memorial';
  if (player.preferences.pathway === 'audience_only') return 'audience';
  
  return 'memorial';  // 默认正式奏疏
}
```

```js
function _checkObjectionRisk(decree, drafter) {
  var risk = 0;
  
  // 违制 / 背祖训
  if (decree.violatesTradition) risk += 0.35;
  if (decree.violatesZuXun) risk += 0.4;
  
  // 派系对立
  if (decree.opposesFaction(drafter.faction)) risk += 0.3;
  
  // 损害主官/家族利益
  if (decree.harmsOfficialInterest(drafter)) risk += 0.35;
  if (decree.harmsClan(drafter.clan)) risk += 0.3;
  
  // 忠诚度
  if (drafter.loyalty < 30) risk += 0.25;
  
  // 能力底线（清官有道德底线）
  if (drafter.honesty > 80 && decree.vagueOrCorruption) risk += 0.3;
  
  return risk > 0.5;
}
```

### 5.1 阶段转移规则

```

  ↓ 效果差 → failing
  ↓ 灾难性 → abolished

  ↓ 稳定 → national（全国推行）

  ↓ 永久或下一个大改革取代

  ↓ 玩家介入（加强执行/调整/认输）→ 重入 trial 或 abolished
```

## 八、失败与废除

### 8.1 失败条件

| 失败类型 | 触发条件 | 后果 |
|---------|---------|------|
| 廷议否决 | 表决未过 | 诏书作罢，轻微声望损失 |
| 试点效果差 | 3 年内民怨>阈值或财政恶化 | AI 建议废除 |
| 重大事件推翻 | 灾荒/战乱/兵变 | 强制中止 |
| 主持者垮台 | 推动官员被抄家 | 制度失去政治根基 |
| 累积不可持续 | 长期财政黑洞 | 自然消亡 |

### 8.2 废除流程

```
废除诏书 → 廷议（看废除阻力）→ 通过 → 新制度下线 → 旧状态恢复
```

废除代价：
- 推动者 reputation 损失
- 下次立类似制度阻力加倍
- 影响皇权威望（未来子系统）
