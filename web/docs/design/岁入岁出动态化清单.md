# 岁入岁出动态化清单

> **回答用户疑问**：岁入岁出**全部都不是死数值**。下面列出每一项数字依赖什么、玩家如何改变它。

## 岁入（每回合 cascade.collect 重新计算）

| 收入项 | 公式 | 动态依赖（玩家可改的） |
|---|---|---|
| **田赋·粮** | `farmland × 0.3 × 0.13` | 兼并/开垦/清丈 改变 farmland · 灾害降 farmland 临时产能 |
| **田赋折银** | `farmland × 0.012` | 同上 |
| **丁税** | `ding × 0.3` (ding≈mouths×0.26) | 户籍审编/逃户回归 改 mouths/ding · 大徭役 死亡降 ding |
| **庸役折布** | `ding × 0.15` | 同上 |
| **商税** | `commerceVolume × 0.03` | 重商/抑商诏令 改 economyBase.commerceVolume · 战乱/灾害降之 |
| **盐铁专卖** | `mouths × 0.05 × 1.0` | 盐法整顿提 compliance · 走私猖獗增 occupation |
| **辽饷加派** | `farmland × 0.009 法定` | **法定速率可调** · 加派/罢派 影响 fiscalConfig |
| **军屯** | `farmland × 0.04 法定 × (1-87.5%侵占)` | **侵占率可调** · 清丈军屯诏令降侵占 · 反贪降侵占 |
| **茶马司** | `300000 法定 × (1-60%侵占)` | 边贸整顿降侵占 |
| **钞关** | `1200000 法定 × (1-50%侵占)` | 钞关御史强力降侵占 |

每项最终再过 cascade 多重乘子（**可玩家改的**）：
- `(1 - corruption/100 × 0.4)` ← **反贪诏令** 直接降 corruption
- `× compliance` (默认 0.72) ← **整顿吏治** / **税监巡视** 提 compliance
- `× qiyun ratio` ← **央地分账规则** 可在 fiscalConfig 调
- `× (1 - logisticsLoss 15%)` ← 修桥铺路 / 漕运改革 降损耗
- `× (1 - autonomy × 0.8)` ← 削藩 / 改土归流 降 autonomyLevel
- `× (1 - exemption)` ← 灾害减免 / 战乱免赋

## 岁出（每回合 FixedExpense.collect 重新计算）

| 支出项 | 公式 | 动态依赖（玩家可改的） |
|---|---|---|
| **官员俸禄** | `Σ(holder × salary)` + `Σ(established × 0.6 × salary)` 兜底 | **任命/裁撤/兼任** 改 holder · **裁官改革** 改 established · **加俸/减俸** 改 salary 字段 |
| **宗禄** | `royalClanPressure.annualStipendPaid × 比例` | **削宗禄** 诏令直降 annualStipendPaid · **宗藩改革** |
| **军饷** | `Σ(armies[i].soldiers × armyMonthlyPay)` | **扩军/裁军** 改 soldiers · **涨饷** 改 armyMonthlyPay (per-army override 也可) · **募新军** 直接增 |
| **军粮·布** | `soldiers × pay.grain/cloth` | 同上 |
| **宫廷开支** | `imperialMonthly × (0.7 + 皇威/100 × 0.6) × 皇族系数` | **皇威 UP** → 排场大 · **节俭诏令** 降皇威 · 后宫人数加减改皇族系数 |

## 核心数据流（验证它真的动态）

```
玩家诏令 → 改 GM 字段 → 下回合 cascade/FixedExpense 重新读字段算 → 数字变
```

**示例 1：扩军 10 万**
```js
GM.armies.push({ name:'新军', soldiers:100000 })
// 下回合 _calcArmyPay 遍历 → 多扣 100000 × 1.5 = 15 万两/月
```

**示例 2：清丈军屯诏（降侵占率）**
```js
// AI 解析诏令 → fiscal_adjustments
scenario.fiscalConfig.customTaxes.find(t=>t.id==='junhu').occupationRate = 0.5
// cascade _convertCustomTax 重读 nominal × (1-0.5) = 0.02 (而非 0.005)
// 实收提升 4 倍
```

**示例 3：加俸（涨工资 50%）**
```js
GM.officeTree[0].positions.forEach(p => p.salary *= 1.5)
// 下回合 _calcSalary 累加新 salary
```

## UI 透明化

帑廪面板岁入分项每条 customTax 显示三段：
```
┃ 军户屯田                          0.5万 [实收]
┃   ↳ 法定 4 厘/亩 ▸ 侵占 87.5% ▸ 实收 0.5 厘/亩
```

同理茶马司/钞关/海关：法定 ▸ 侵占 ▸ 实收。

## 如何确认数字真在变化（自查清单）

回合 N 与回合 N+1 之间，下面任一操作发生 → 对应数字必动：
1. 任命/裁撤官员 → 俸禄
2. 招募/解散部队 → 军饷
3. 改军饷待遇 → 军饷
4. 兼并/开垦诏令 → 田赋
5. 清丈/反贪诏令 → 各税侵占率
6. 宗禄改革 → 宗禄
7. 灾害事件 → 田赋/承载力
8. 战争爆发 → 战区税收(disruption)

如果上面任一操作后下回合数字没变，那就是 **bug**，请反馈截图我修。
