# 设计方案 · 皇威系统（天威·度之镜）

> 第六官方变量。与皇权并列，构成"乾纲·天威"双柱——"度"哲学的完整呈现。  
> **定位**：不是数值越高越好，而是"**第二段最佳**"。与皇权不同，皇威分**五段**以体现更细的梯度。  
> **核心创新**：**暴君综合症**——当皇威过高，朝堂陷入"无人敢谏"的虚假繁荣，诏书被**过度执行**，玩家在欢呼声中走向灭亡。

---

## 零、核心哲学

### 0.1 三条铁律

1. **第二段最佳，不是最高**  
   威严（70-90）是明主贤君；暴君（90+）是万世笑柄。玩家若追求"数值最大化"，反而走向隋炀帝、朱元璋后期的悲剧。

2. **皇威是情感，皇权是结构**  
   皇权是结构（权力分配），皇威是情感（被敬畏的程度）。二者独立但共振。朱元璋晚年皇权+皇威皆顶峰=活人地狱；汉献帝皇权+皇威双低=傀儡。**最佳是皇权中段 + 皇威威严段**。

3. **过度执行比拒绝执行更危险**  
   失威段诏书被无视只是效率低；暴君段诏书被"过度执行"则可能一道"严查"变成"灭门"，一道"免赋"变成"补赈"。玩家在欢呼中损失帑廪、家破人亡。

### 0.2 与其他系统的关系

- **皇权**：并列"乾纲"。四象限决定帝王原型。
- **腐败**：威严段抑腐；暴君段腐败隐匿（大臣不敢查）；失威段腐败公开化。
- **民心**：军事胜利/德政 → 皇威 + 民心双涨；军败/苛政 → 双跌。
- **异象**：天象 → 皇威负面；祥瑞 → 皇威正面；罪己诏 → 短期降长期升。
- **风闻录事**：暴君段——告状减（畏惧）、耳报增（互相告讦）；失威段——告状增（无忌）、风议增（公开批评）。

### 1.3 真实值 vs 感知值（暴君段关键）

```javascript
function updatePerceivedHuangwei() {
  var t = GM.huangwei.index;
  
  // 暴君段：感知值被颂圣扭曲，显著高估
  if (t > 85) {
    GM.huangwei.perceivedIndex = Math.min(100, t + 8);
  }
  // 威严段：接近真实
  else if (t > 70) {
    GM.huangwei.perceivedIndex = t;
  }
  // 失威段：感知值可能被地方粉饰，低估自己的失威程度
  else if (t < 30) {
    GM.huangwei.perceivedIndex = t + 10;
  } else {
    GM.huangwei.perceivedIndex = t;
  }
}
```

**关键**：暴君段的玩家往往以为自己才是**威严** 80——因为所有奏报都歌颂"圣德巍巍"。

### 1.4 皇权 × 皇威四象限（重述）

| | 皇威高 | 皇威低 |
|---|---|---|
| **皇权高** | 暴君顶点（朱元璋后期/隋炀帝） | 事必躬亲无人听（崇祯末） |
| **皇权低** | 受敬傀儡（极罕见） | 汉献帝式傀儡 |

**最佳**：皇权制衡 + 皇威威严（唐太宗、康熙中期）

### 2.1 军事胜利（militaryVictory）

```javascript
function onMilitaryVictory(battle) {
  var gain = 0;
  if (battle.scale === 'decisive')    gain += 8;
  if (battle.scale === 'major')       gain += 5;
  if (battle.scale === 'minor')       gain += 2;
  
  if (battle.enemy === 'foreign')     gain += 3;   // 对外胜利加倍
  if (battle.enemy === 'rebellion')   gain += 2;   // 平叛
  if (battle.personallyLed)            gain *= 1.5; // 亲征+50%
  
  GM.huangwei.index += gain;
  GM.huangwei.subDims.military.value += gain * 1.5;
  GM.huangwei.sources.militaryVictory += gain;
}
```

## 四、暴君综合症（**第一独特机制**）

### 4.1 激活与维持

```javascript
function checkTyrantSyndrome() {
  var t = GM.huangwei.index;
  
  if (t > 90 && !GM.huangwei.tyrantSyndrome.active) {
    GM.huangwei.tyrantSyndrome.active = true;
    GM.huangwei.tyrantSyndrome.activatedTurn = GM.turn;
    addEB('朝代', '万岁声不绝于朝野——陛下威临天下', { credibility: 'low' });
  }
  
  if (t < 85 && GM.huangwei.tyrantSyndrome.active) {
    deactivateTyrantSyndrome();
  }
}
```

### 4.2 奏疏变质

```javascript
function filterMemorialsInTyrantMode() {
  if (!GM.huangwei.tyrantSyndrome.active) return;
  
  // 进谏类奏疏被地方自审压制
  for (var m of GM.pendingMemorials) {
    if (m.type === 'remonstrance' || m.type === 'critique') {
      if (Math.random() < 0.9) {
        m.suppressedBySelfCensorship = true;
        // 但原作者可能承担风险（被下级揭发）
        if (Math.random() < 0.2) {
          triggerEvent('memorialAuthorInTrouble', m.author);
        }
      }
    }
  }
  
  // 90% 的新奏疏为颂圣类
  GM.huangwei.tyrantSyndrome.flatteryMemorialRatio = 0.9;
}
```

```javascript
  if (!GM.huangwei.tyrantSyndrome.active) return decree;
  
  // 过度执行：量刑加重、范围扩大、力度加码
  var overExecuted = deepClone(decree);
  
  if (decree.contains('惩') || decree.contains('问罪')) {
    overExecuted.severity = 'extreme';  // "惩" → "诛灭三族"
    GM.huangwei.tyrantSyndrome.hiddenDamage.wrongfulDeaths += 50 + Math.random() * 500;
  }
  
  if (decree.contains('严查') || decree.contains('整治')) {
    overExecuted.scope = 'expanded';    // "严查某地" → "株连牵涉"
    GM.huangwei.tyrantSyndrome.overExecutionLog.push({
      original: decree.summary,
      executed: overExecuted.summary,
      turn: GM.turn
    });
  }
  
  if (decree.contains('减赋') || decree.contains('免赋')) {
    overExecuted.magnitude *= 1.5;     // 多减赋 → 帑廪损失
    GM.guoku.balance -= decree.originalCost * 0.5;
  }
  
  if (decree.contains('修') || decree.contains('建')) {
    overExecuted.scale = 'enhanced';   // 扩大工程规模 → 徭役沉重
    GM.minxin.trueIndex -= 3;
  }
  
  // 地方借势勒索（浮收）
  GM.fiscal.floatingCollectionRate += 0.05;
  
  GM.huangwei.tyrantSyndrome.overExecutionLog.push({
    original: decree.summary,
    executed: overExecuted.summary,
    turn: GM.turn,
    damage: calcHiddenDamage(decree, overExecuted)
  });
  
  return overExecuted;
}
```

## 六、执行度公式（与皇权共同构成）

```javascript
// 执行度 = f(皇权, 皇威, 诏书详尽度, 大臣配合, 地方腐败)
function calcExecutionDegree(decree) {
  var h = GM.huangquan.index;
  var w = GM.huangwei.index;
  var detail = decree.detailLevel;
  var support = calcMinisterSupport(decree);
  var localCorr = GM.corruption.subDepts.provincial.true;
  
  // 皇威乘数（过度执行 or 拖延）
  var weiMult;
  var weiError = 0;  // 错误率
  
  if (w > 90) {         // 暴君
    weiMult = 1.3;
    weiError = 0.3;
  } else if (w > 70) {  // 威严（最佳）
    weiMult = 1.0;
    weiError = 0;
  } else if (w > 50) {  // 常望
    weiMult = 0.85;
    weiError = 0.05;
  } else if (w > 30) {  // 衰微
    weiMult = 0.65;
    weiError = 0.15;
  } else {              // 失威
    weiMult = 0.35;
    weiError = 0.25;
  }
  
  // 先按皇权段计算基础执行度
  var base = calcBaseExecutionFromHuangquan(h, detail, support, localCorr);
  
  return {
    actualExecution: base * weiMult,
    errorRate: weiError,
    overExecution: w > 90 ? (base * 0.3) : 0,
    hiddenDamage: w > 90 ? estimateHiddenDamage(decree) : 0
  };
}
```

## 十四、设计特色

1. **"第二段最佳"的反直觉设计**——在多数游戏中"数值最大化"是目标，但皇威系统颠覆此逻辑。追求极高皇威的玩家将滑入暴君深渊。

2. **暴君综合症的虚假繁荣**——玩家看到的是"威临天下"，实际民心、腐败、冤狱悄然崩坏。这是中国历史上反复上演的真实悲剧。

3. **过度执行机制**——诏书不是被打折，而是被**扩大**。"惩工部尚书"变成"工部满门抄没三族"。玩家每一道诏书都可能造成意外的灾难。

4. **隐藏代价 + 觉醒事件**——玩家在暴君段看不见民心、腐败的真实下跌。直到某天忽然叛军破城、忠臣冤死，才知大势已去。这还原了崇祯式"晚知"悲剧。

5. **失威危机的连锁反应**——低皇威不止是"效率低"，还会与皇权联动激活权臣系统、让外邦趁势、令地方观望。

6. **五段色阶的反讽**——一般游戏将最高值标金色（褒义），皇威却在暴君段用深赤（警示）。视觉层面也贯彻"过犹不及"哲学。

7. **与皇权的四象限**——暴君顶点（朱元璋后期）、事必躬亲无人听（崇祯末）、受敬傀儡（罕见）、傀儡（汉献帝）、明主贤君（唐太宗）——五种帝王原型完整覆盖。

8. **14 × 14 的丰富来源**——从军事胜利到外邦朝贡、从天象无回应到失信于臣，皇威的涨落有 28 个可追踪维度，让每个剧情事件都有精确反馈。

9. **亲征的双面刃**——亲征胜利 ×1.5 加成；亲征失败 ×2 扣减，并永久损伤 military 分项。玩家需审慎判断。

10. **12 朝代 × 4 阶段的历史锚点**——从秦始皇到光绪，每个时点有历史真实的皇威值，方便玩家代入史实或自行改写。
