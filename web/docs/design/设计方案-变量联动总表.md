> 起草：2026-04-18

> **定位**：不是设计新机制，而是把已设计的 7 个系统"织"成一张图——查缺补漏，统一交互语言。

## 零、七变量概览与"治体三轴"

### 0.1 七变量归类

```
┌─────────────────────────────────────────────────────────┐
│                      天命 · 七官方变量                     │
├─────────────┬────────────────┬──────────────────────────┤
│  资源轴      │  人口轴          │  政体轴                    │
├─────────────┼────────────────┼──────────────────────────┤
│  帑廪         │  在籍户口        │  腐败（吏治）              │
│  内帑         │                  │  民心                      │
│              │                  │  皇权（权力结构）            │
│              │                  │  皇威（情感敬畏）            │
└─────────────┴────────────────┴──────────────────────────┘
```

- **资源轴（2）**：帑廪（国帑）+ 内帑（私帑）
- **人口轴（1）**：在籍户口
- **政体轴（4）**：腐败 + 民心 + 皇权 + 皇威

### 0.2 "治体三轴"交互

```
      ┌───── 资源轴 ──────┐
      │                        │
      v                        ^
  政体轴 ←────────────→ 人口轴
      ^                        │
      │                        v
      └─────── (反馈) ────────┘
```

- 资源→人口：赋税/徭役/赈济 决定人口生存
- 人口→资源：税基/兵源 决定资源上限
- 政体→资源：腐败漏损、皇权决定执行
- 资源→政体：空库 → 权臣趁势、民心动摇
- 政体→人口：苛政 → 流民、民变 → 户口流失
- 人口→政体：民变 → 皇权动摇、皇威损耗

## 二、42 项联动机制详解

### 2.1 帑廪 → 其他

#### 帑廪 → 内帑
**机制**：帑廪-内帑转运（财政系统 B 决策）  
**方向**：双向 ±  
**强度**：强  
**规则**：
- 帑廪充足时：内帑可"敲边鼓"（借调）1-3%
- 帑廪枯竭时：内帑被迫"垫支"（历史上明末、清末皆如此）
- 长期转运失衡会触发事件（"帑臣欲议损内库以充国用"）

#### 帑廪 → 户口
**机制**：赈灾、救助、徭役规模、农业投资  
**方向**：正  
**强度**：中  
**规则**：`reliefFund > 50% need → population stability +15%`  
**细节**：帑廪专款投农水利 → 长期户口增长率 +0.3%/年

#### 帑廪 → 腐败
**机制**：官员俸禄过低 → 结构性贪腐（见腐败 §2.1 lowSalary）  
**方向**：负（帑廪紧缺→俸禄薄→腐败升）  
**强度**：强  
**公式**：`lowSalaryCorruption = (1 - avgSalary/livingCost) × 30`

#### 帑廪 → 民心
**机制**：三数传导——名义税 / 官府实收 / 民间实缴  
**方向**：双向 ±  
**强度**：强  
**规则**：
- 帑廪紧缺 → 加税 → 民心降
- 帑廪宽裕 → 减税/赈济 → 民心升

#### 帑廪 → 皇权
**机制**：帑廪是皇权的物质基础  
**方向**：正（帑廪丰→皇权稳；帑廪枯→被迫授权地方）  
**强度**：中  
**规则**：`guoku.balance < -annualIncome * 0.5` → `huangquan.drains.militaryDefeat += 5` (被迫让地方自筹军费 → 地方坐大)

#### 帑廪 → 皇威
**机制**：丰库才能办大典、打大仗  
**方向**：正  
**强度**：中  
**规则**：破产期无法封禅/赏赐 → 皇威上升手段缺失

---

### 2.2 内帑 → 其他

#### 内帑 → 帑廪
**机制**：紧急时皇帝捐内帑济国（历史如朱元璋、康熙）  
**方向**：双向 ±  
**强度**：强（罕见但决定性）  
**规则**：危机时可一次性转入帑廪，皇威+5 民心+3，但内帑实力削减

#### 内帑 → 户口
**机制**：皇庄、皇室地产的佃农  
**方向**：正（微小）  
**强度**：弱  
**规则**：皇庄扩张 → 皇庄佃农增加但普通在籍户口减少（皇庄人口独立造册）

#### 内帑 → 腐败
**机制**：内廷腐败——御赐过量、宦官私吞、外戚侵占  
**方向**：条件（内廷管理好则零影响；管理坏则腐败激增）  
**强度**：中  
**规则**：`corruption.subDepts.imperial` 与内帑管理松紧强关联

#### 内帑 → 民心
**机制**：皇帝奢侈（大建园林、宠妃挥霍）  
**方向**：负  
**强度**：中  
**规则**：`neitang.extravagantSpending > threshold` → `minxin -= 5`/年

#### 内帑 → 皇权
**机制**：内帑是皇帝个人财政权  
**方向**：正  
**强度**：强  
**规则**：
- 内帑独立 → 皇权有私资可动（诛权臣可不借国帑）
- 内帑被外戚/宦官侵占 → 皇权受制

#### 内帑 → 皇威
**机制**：大典由内帑出资  
**方向**：正  
**强度**：强  
**规则**：封禅消耗内帑 50 万两 → 皇威 +20；奢华办葬礼 +10

---

### 2.3 户口 → 其他

#### 户口 → 帑廪
**机制**：税基 = 在籍户口  
**方向**：正  
**强度**：强  
**公式**：`nominalTax = registeredHouseholds × taxPerHousehold`

#### 户口 → 内帑
**机制**：皇庄人口  
**方向**：正（微小）  
**强度**：弱  
**规则**：皇庄佃户数直接影响内帑地产收入

#### 户口 → 腐败
**机制**：人口多 → 胥吏多 → 腐败基础大  
**方向**：条件  
**强度**：中  
**规则**：`population / officialCount` 比率决定胥吏密度；过密集的地方胥吏腐败显著

#### 户口 → 民心
**机制**：人口密度→生态压力；流民比例→阶层民心  
**方向**：双向 ±  
**强度**：弱  
**规则**：生态承载力接近上限 → 民心下降

#### 户口 → 皇权
**机制**：人口规模决定皇权行使基础  
**方向**：正  
**强度**：中  
**规则**：小国寡民（人口 < 500 万）→ 皇权天然受限（无足够官僚网络）

#### 户口 → 皇威
**机制**：人口多 = 天命所归  
**方向**：正  
**强度**：弱  
**规则**：登记户口达历史高位（如"盛唐"）→ 皇威轻微加成

---

### 2.4 腐败 → 其他

#### 腐败 → 帑廪
**机制**：实征漏损（见腐败 §3.1）  
**方向**：负  
**强度**：强  
**公式**：`actualTaxRate = 1 - (fiscal + provincial) / 200 * 0.7`  
**举例**：腐败 50 → 实征率 65%；腐败 80 → 44%

#### 腐败 → 内帑
**机制**：内廷腐败（宗室/宦官/外戚）  
**方向**：负  
**强度**：中  
**规则**：`imperial 部门腐败` → 内帑月入被侵吞 50% 比例

#### 腐败 → 户口
**机制**：胥吏勒索 → 百姓逃亡 → 隐户  
**方向**：负  
**强度**：强  
**规则**：`provincial 腐败 > 60` → 隐户率每年 +1%

#### 腐败 → 民心
**机制**：司法不公、胥吏勒索、灾荒截留  
**方向**：负  
**强度**：强  
**规则**：民心来源 14 类中，"司法不公"+"地方官清浊" 两项直接来自腐败

#### 腐败 → 皇权
**机制**：腐败盘根错节 → 皇帝政令难施  
**方向**：负  
**强度**：强  
**规则**：腐败高 → 新政落空 → 皇权威信受损

#### 腐败 → 皇威
**机制**：腐败案揭发 → 朝廷形象受损  
**方向**：负  
**强度**：强  
**规则**：大案曝光 → 皇威 -8~-20；若处置得宜可反升

---

### 2.5 民心 → 其他

#### 民心 → 帑廪
**机制**：顺从度决定实征率  
**方向**：双向 ±  
**强度**：强  
**公式**：`taxCollectionRate = max(0.3, minxin/100 * 0.7 + 0.3)`

#### 民心 → 内帑
**机制**：民心影响皇室地产的佃农服役  
**方向**：负（间接）  
**强度**：弱  
**规则**：民心崩溃时皇庄也受损

#### 民心 → 户口
**机制**：低民心 → 逃亡、民变、户口流失  
**方向**：双向 ±  
**强度**：强  
**规则**：`minxin < 40` → 逃亡率 +0.5%/月；`minxin < 20` → 可能民变导致人口锐减

#### 民心 → 腐败
**机制**：民心高时，士人告举踊跃；民心低时，告状无门反被压制  
**方向**：双向 ±  
**强度**：中  
**规则**：民心 > 70 → 登闻鼓活跃；民心 < 30 → 登闻鼓被禁

#### 民心 → 皇权
**机制**：民心低时，权臣借"清君侧"名义坐大  
**方向**：双向 ±  
**强度**：强  
**规则**：民心 < 30 → 皇权被权臣借机侵蚀

#### 民心 → 皇威
**机制**：民心是皇威的根基（天视自我民视）  
**方向**：双向 ±  
**强度**：强  
**规则**：民心 > 70 → 皇威 +3/月缓升；民心 < 20 → 皇威 -5/月

---

### 2.6 皇权 → 其他

#### 皇权 → 帑廪
**机制**：皇权强时征税/国库调用灵活；弱时地方截留  
**方向**：双向 ±  
**强度**：强  
**规则**：
- 专制段：税调灵活但易征过度扰民
- 权臣段：地方截留严重，帑廪实际可用 50%

#### 皇权 → 内帑
**机制**：皇权决定内帑的独立性  
**方向**：双向 ±  
**强度**：强  
**规则**：
- 皇权强：内帑完整受皇帝支配
- 皇权弱：内帑被外戚/宦官/权臣侵占

#### 皇权 → 户口
**机制**：大造黄册（清查）的执行力  
**方向**：双向 ±  
**强度**：中  
**规则**：皇权强 → 清查到位，在籍户口复升；皇权弱 → 地方敷衍，隐户激增

#### 皇权 → 腐败
**机制**：皇权强有肃贪工具，但酷吏反噬；皇权弱无力整顿  
**方向**：双向 ±  
**强度**：强  
**规则**：
- 专制段：清洗多但易酷吏反弹
- 制衡段：监察体系运转良好
- 权臣段：肃贪沦为党争武器

#### 皇权 → 民心
**机制**：三段均影响民心——过强扰民、过弱失序  
**方向**：双向 ±  
**强度**：中  
**规则**：制衡段民心稳定；专制段短期稳长期躁；权臣段民心流失

#### 皇权 → 皇威
**机制**：皇权极端时皇威也极端化  
**方向**：双向 ±  
**强度**：强  
**规则**：皇权高 → 诛权臣立皇威；皇权低 → 被权臣架空同时失威

---

### 2.7 皇威 → 其他

#### 皇威 → 帑廪
**机制**：暴君段过度执行 → 地方浮收（短期帑廪增但虚账）；失威段地方观望 → 帑廪实征降  
**方向**：双向 ±  
**强度**：中  
**规则**：
- 暴君段：帑廪"报账"增加但实际损耗（冤狱赔偿、浮收激起民变）
- 失威段：实征率 × 0.35

#### 皇威 → 内帑
**机制**：大典支出；失威段被觊觎  
**方向**：负（总体）  
**强度**：中  
**规则**：封禅 / 万寿 / 千叟宴皆消耗内帑；失威段外戚/权臣可能染指

#### 皇威 → 户口
**机制**：暴君段徭役过度 → 户口流失；失威段地方瞒报人口  
**方向**：双向 ±  
**强度**：中  
**规则**：
- 暴君：徭役扩大 → 逃户 + 民变 → 户口减
- 失威：地方瞒报 → 在籍户口虚减（隐实增）

#### 皇威 → 腐败
**机制**：威严段抑腐；暴君段隐匿；失威段公开  
**方向**：双向 ±  
**强度**：强  
**规则**：
- 威严段：肃贪有威（案件揭发率提升）
- 暴君段：腐败隐匿不敢报
- 失威段：腐败公开化（大臣无所顾忌）

#### 皇威 → 民心
**机制**：皇威关乎"天命所归"的情感认同  
**方向**：双向 ±  
**强度**：强  
**规则**：
- 德政+军胜 → 民心 + 皇威双涨
- 暴君段：民心隐性下跌（民众恐惧表面服从实心离）
- 失威段：民心同时崩坏（对朝廷失望）

#### 皇威 → 皇权
**机制**：皇威极端状态放大皇权极端  
**方向**：双向 ±  
**强度**：强  
**规则**：皇威暴君段 + 皇权高 = 暴君顶点（朱元璋后期）；皇威失威段 + 皇权弱 = 权臣系统激活

## 三、多元复合联动

### 3.1 "腐败 × 皇权"四象限

| | 皇权高 | 皇权低 |
|--|--|--|
| **腐败高** | 酷吏式肃贪（武则天式） | 官僚系统性腐败（明末阉党） |
| **腐败低** | 明君盛世（康熙中期） | 分权廉能（宋代台谏） |

### 3.2 "民心 × 皇威"四象限

| | 皇威高（威严/暴君） | 皇威低（衰微/失威） |
|--|--|--|
| **民心高** | 明君贞观 | 被敬傀儡（罕见） |
| **民心低** | 暴君镇压（表面） | 末世（民怨+失威） |

### 3.3 "皇权 × 皇威"五情境（扩展 §皇权 1.4）

| 皇权 | 皇威 | 原型 | 历史 |
|---|---|---|---|
| 高 | 暴君 | 绝对暴君 | 朱元璋晚期、隋炀帝、武则天晚期 |
| 高 | 威严 | 中兴雄主 | 汉武盛年、唐太宗贞观 |
| 中 | 威严 | 盛世明君 | 康熙中期、宣宗宣德 |
| 高 | 失威 | 事必躬亲无人听 | 崇祯末 |
| 低 | 威严 | 受敬傀儡（罕见） | 东汉献帝建安初 |
| 低 | 失威 | 总傀儡 | 汉献帝末年、唐昭宗 |
| 中 | 衰微 | 守成平庸 | 多数中期帝王 |

### 3.4 "帑廪 × 民心 × 户口"三元

**资源-民生-人口闭环**：
- 帑廪→赈灾→民心→户口稳定
- 户口→税基→帑廪
- 民心→顺从→实征→帑廪

**末世崩溃链**：
```
帑廪紧缺
  → 加税 (Δ帑廪 +，民心 -)
    → 民心低 → 逃亡 (户口 -)
      → 户口少 → 税基缩 (帑廪 --)
        → 更加税 (民心 ---)
          → 民变 → 军费 (帑廪 ---)
```
**这是王朝周期律的财政-民生崩溃模型。**

### 3.5 "腐败 × 皇威 × 帑廪"隐性漏损

皇威暴君段 + 腐败高 + 帑廪紧缺：
- 地方官过度执行收税命令
- 浮收严重 → 民间实缴高
- 官府实收被层层克扣
- 朝廷表面税收 OK，实际官僚系统蚕食
- 暴君不允许查账 → 腐败进一步隐匿

**历史**：乾隆后期、明崇祯初年

### 3.6 "皇权 × 户口 × 腐败"的黄册崩坏

- 皇权强 + 腐败低：十年大造黄册，户口清查准确
- 皇权弱 + 腐败高：黄册成废纸，隐户爆炸
- 皇权强 + 腐败高：酷烈清查 → 激起民变

**历史**：明初黄册精密→明中期胡乱→明末完全崩坏

## 四、关键反馈回路

### 4.1 正反馈（强化放大）

#### 明君回路（+）
```
德政 → 民心↑ → 顺从↑ → 税征↑ → 帑廪↑
  → 赈灾/大典 → 民心再↑ → 皇威↑
  → 改革容忍度↑ → 结构优化 → 长期繁荣
```

#### 末世回路（-）
```
腐败↑ → 民心↓ → 逃亡↑ → 户口↓ → 帑廪↓
  → 加税/征役 → 民心更↓ → 民变 → 军费↑
  → 帑廪枯竭 → 地方自筹 → 皇权↓
  → 权臣起 → 皇威↓ → 外邦趁 → 边衅↑
  → 王朝倾覆
```

### 4.2 负反馈（自我平衡）

#### 皇威-皇权自平衡
- 皇威过高 → 暴君综合症 → 隐藏代价 → 觉醒事件 → 皇威回落
- 皇权过低 → 权臣坐大 → 老臣死亡 → 皇权恢复

#### 腐败-民心对冲
- 腐败高 → 民心跌 → 告状上涌 → 暴露腐败 → 整治
- 民心低 → 加重纠察 → 短期严惩 → 表面好转

### 4.3 阈值跳变

| 阈值 | 后果 |
|------|------|
| 皇权 < 35 | 权臣 NPC 激活 |
| 皇威 > 90 | 暴君综合症激活 |
| 皇威 < 30 | 失威危机激活 |
| 民心 < 20 | 揭竿起义高概率 |
| 腐败 > 70 | 腐败集团凝聚 |
| 帑廪 < -年入×0.5 | 财政破产事件 |
| 户口 < 朝代起始×0.5 | 王朝衰落信号 |

## 五、典型王朝兴衰链

### 5.1 开国（理想态）

```
新朝建立
  ↓
开国君主 威严段（80） + 皇权制衡/专制段（50-85）
  ↓
腐败低（15-25）+ 民心高（70-80）
  ↓
户口渐增（战乱恢复）+ 帑廪渐丰（实征率高）
  ↓
内帑稳定，一切正向
```

### 5.2 全盛（最佳态）

```
皇权制衡（55-70）+ 皇威威严（75-85）
  ↓
腐败 25-40（可控）+ 民心 65-75（稳定）
  ↓
帑廪 + 户口达到王朝峰值
  ↓
大典可办（封禅），改革可行，开疆拓土
  ↓
但盛极而衰的暗流已开始（腐败缓升、内廷腐化）
```

### 5.3 中衰（转折点）

```
某次关键事件（军败/权臣坐大/党争爆发）
  ↓
皇权微降，腐败加速（50+），皇威下降
  ↓
帑廪紧缩 → 加税 → 民心松动
  ↓
户口缓降（逃亡+隐户）
  ↓
各项数值同步下滑
  ↓
若无中兴之主，进入末世链路
```

### 5.4 末世（崩溃）

```
权臣/宦官专政
  ↓
皇权 < 35（权臣系统激活）
  ↓
皇威 < 30（失威危机）或 > 90（最后的暴君）
  ↓
腐败 > 70（腐败集团凝聚）
  ↓
民心 < 20（民变四起）
  ↓
帑廪破产、户口锐减
  ↓
禅让 / 内战 / 改朝换代
```

### 5.5 中兴（可能的逆转）

```
末世将临
  ↓
新君登基（或摄政王雄主）
  ↓
1. 诛权臣：皇权+25，皇威+15
2. 肃贪：腐败-15~-30（代价：党争）
3. 整顿户籍：大造黄册，在籍户口复增
4. 慎用兵：帑廪休养
5. 德政收民心
  ↓
若成功，开启"中兴"（汉光武、唐宣宗、明仁宗）
若失败（刚愎自用/刚柔失度），加速灭亡（崇祯）
```

## 八、统一交互语言

为后续实现，约定统一的跨系统调用约定：

### 8.1 数据读取

所有跨系统读取通过 `GM.<varName>.<field>` 统一访问：
```javascript
GM.guoku.balance
GM.neitang.balance
GM.hukou.registeredTotal
GM.corruption.trueIndex
GM.minxin.trueIndex
GM.huangquan.index
GM.huangwei.index
```

### 8.2 跨系统事件

通过事件总线（`GM._listeners`）：
```javascript
// 帑廪破产触发其他系统反应
emit('fiscalCrisis', { severity: 'bankruptcy' });

// 监听：皇威、民心、皇权分别响应
on('fiscalCrisis', (d) => { GM.huangwei.index -= d.severity * 15; });
```

### 8.3 统一增量 API

```javascript
// 标准增量接口
applyDelta(varPath, amount, source, reason);
// 例如：applyDelta('minxin.trueIndex', -5, 'corruption:judicial', '冤狱');

// 统一记录到 GM.turnChanges.variables，便于回合结算展示
```

### 8.4 回合结算顺序（关键）

```
1. 腐败演化（影响实征率）
2. 户口变化（逃亡/生育）
3. 帑廪结算（含腐败漏损）
4. 内帑结算（含侵吞）
5. 民心计算（14 源）
6. 皇权调整（8 上升 + 8 下降）
7. 皇威调整（14 上升 + 14 下降）
8. 暴君综合症 / 失威危机判定
9. 权臣激活 / 起义升级
10. 事件触发
```
