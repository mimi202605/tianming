# 设计方案 · 皇权系统（乾纲·度之枢）

> 第五官方变量。与威望共同构成"帝王之度"——中国政治哲学最独特的**过犹不及**哲学。  
> **定位**：不是执行力数字，是**皇帝 vs 官僚的权力分配轴**。三段哲学——权臣专政 / 制衡 / 专制，**中段最佳**，两端皆危。  
> **核心创新**：**权臣 NPC 诏书干预机制**——皇权弱时，玩家不再是唯一执政者，权臣可驳回、修改、甚至代签诏书，玩家变成 **"**傀儡而不自知"**。

---

## 零、核心哲学

### 0.1 三条铁律

1. **"度"之枢**  
   不是越强越好，不是越弱越好。皇权过强 → 暴君式事必躬亲；皇权过弱 → 权臣架空。**中段最佳**。这违反了许多西方政治模拟的常规（如 CK3 的"越集权越好"），但符合中国历史哲学——"过犹不及"。

2. **玩家体验随段位剧变**  
   不同段位，玩家写诏书的方式完全不同：
   - 专制段：必须事无巨细（朱元璋每日阅奏 200+）
   - 制衡段：可一句话概括（宋代分权）
   - 权臣段：写了也不作数（汉献帝）

3. **皇权 ≠ 执行力**  
   执行度 = f(皇权, 诏书详尽度, 大臣配合, 地方腐败)。**皇权专制**不等于**高效**——反而因事必躬亲、大臣献媚、无人纠偏而**容易出错**。中段的"分权制衡"才是最高效。

### 0.2 与其他系统的关系

- **威望**：二者独立但相互影响。皇权是结构，威望是情感。高皇权 + 高威望 = 暴君；低皇权 + 高威望 = 受敬傀儡（罕见）；低皇权 + 低威望 = 汉献帝式傀儡。
- **腐败**：皇权强时肃贪工具多但易出酷吏；皇权弱时腐败盘根。
- **民心**：皇权过强/过弱都扰民；中段稳定。
- **制度设计**：皇权强 → 诏即法；皇权弱 → 诏需阁议。
- **党争**（已保留概念）：皇权强 → 党争受压转地下；皇权弱 → 党争公开激烈。

## 四、后果传导

### 4.1 执行度计算（核心）

```javascript
// 执行度 = f(皇权, 诏书详尽度, 大臣配合, 地方腐败)
function calcExecutionDegree(decree) {
  var h = GM.huangquan.index;
  var detail = decree.detailLevel;       // 0-1（诏书详尽度）
  var support = calcMinisterSupport(decree);
  var localCorr = GM.corruption.subDepts.provincial.true;
  
  var base = 0;
  
  if (h >= 75) {
    // 专制段：详尽度决定一切；大臣不会补全
    base = 0.4 + detail * 0.5;
    // 若大臣不配合（献媚）可能被表面执行
    base *= 0.8 + support * 0.2;
  } else if (h >= 35) {
    // 制衡段：最高效；大臣会补全
    base = 0.6 + detail * 0.3 + support * 0.2;
    // 地方腐败削减
    base *= 1 - localCorr / 200;
  } else {
    // 权臣段：权臣决定
    var pm = GM.huangquan.powerMinister;
    if (!pm) base = 0.5;
    else {
      var alignment = calcDecreeAlignment(decree, pm);
      // 完全不合权臣意：执行度 0.1；完全合意：0.9
      base = 0.1 + (alignment + 100) / 200 * 0.8;
    }
  }
  
  return clamp(base, 0, 1);
}
```

### 4.2 大臣进谏自由度

```javascript
function calcRemonstranceFreedom() {
  var h = GM.huangquan.index;
  var w = GM.weiwang.index;
  
  // 皇权过强 + 威望过高 → 无人敢谏
  if (h > 80 && w > 85) return 0.05;  // 暴君
  
  // 皇权过弱 → 权臣反过来谏（常为逼宫）
  if (h < 30) return 0.4;  // 但非真进谏
  
  // 中段：自由度 0.6-0.9
  return 0.5 + (Math.min(h, 75) - 30) / 100 + w / 400;
}
```

```javascript
  if (GM.huangquan.powerMinister) return;
  
  var candidates = characters.filter(c => 
    c.role !== 'emperor' &&
    c.influence > 75 &&
    c.loyaltyToEmperor < 60 &&
    (c.abilities.administration + c.abilities.intelligence) > 140 &&
    c.factionStrength > 50
  );
  
  if (candidates.length === 0) {
    // 若无符合条件，AI 生成一名权臣（能力强 NPC 或已有高位官员晋升）
    candidates = [promoteToMinister()];
  }
  
  // 按"潜势"排序
  candidates.sort((a, b) => 
    (b.influence + b.factionStrength + b.abilities.intelligence) -
    (a.influence + a.factionStrength + a.abilities.intelligence));
  
  var chosen = candidates[0];
  GM.huangquan.powerMinister = {
    charId: chosen.id,
    name: chosen.name,
    sinceTurn: GM.turn,
    interceptRate: 0.5,
    rewriteRate:   0.25,
    rejectRate:    0.15,
    selfEdictRate: 0.05,
    factionSupport: chosen.factionStrength,
    endgame: null,
    hiddenAgenda: generateAgenda(chosen)  // '篡位' | '专权' | '护幼主' | '清君侧'
  };
  
  addEB('朝代', '权臣' + chosen.name + '把持朝政', { 
    credibility: 'high', 
    subject: chosen.id 
  });
}
```

### 5.2 诏书拦截流程

玩家颁布诏书时：

```javascript
function processDecreeUnderPowerMinister(decree) {
  var pm = GM.huangquan.powerMinister;
  if (!pm) return { status: 'passed', decree: decree };
  
  // 部分诏书可绕过（仪式性/个人性）
  if (decree.type === 'personal' || decree.type === 'ceremonial') {
    return { status: 'passed', decree: decree };
  }
  
  // 看权臣是否拦截
  if (Math.random() > pm.interceptRate) {
    return { status: 'passed', decree: decree };
  }
  
  // 评估诏书与权臣利益的对齐度
  var alignment = calcDecreeAlignment(decree, pm);  // -100 to +100
  
  if (alignment > 30) {
    return { status: 'approved', decree: decree, 
             note: pm.name + '赞同此诏，奉旨施行' };
  }
  
  if (alignment > -30) {
    var modified = modifyDecreeToFit(decree, pm);
    return { status: 'modified', decree: modified,
             note: pm.name + '谓：陛下此诏当如此拟订——' + modified.summary };
  }
  
  // 强烈反对
  if (pm.factionSupport > 80 && GM.huangquan.index < 20) {
    // 胆大包天：直接驳回或篡改
    if (Math.random() < pm.rejectRate) {
      return { status: 'rejected',
               note: pm.name + '伏请陛下三思，此事不可——臣以身家相抗' };
    }
    
    // 暗改为相反内容
    var perverted = reverseDecree(decree, pm);
    return { status: 'perverted', decree: perverted,
             hint: '（朱批字迹与陛下近日似有不同）' };
  }
  
  return { status: 'rejected',
           note: pm.name + '留中不发（扣压诏书）' };
}
```

## 六、三段诏令体验分化

### 6.1 专制段（75-100）：事必躬亲

**玩家诏书**：必须包含五要素：
- 时间（某日/某月/某年）
- 地点（具体州府县）
- 执行人（点名）
- 经费来源（帑廪/内帑/某税）
- 考核标准（何时查验）

缺失任何一项，AI 大臣奏"**臣不敢擅专，请陛下圣裁**"，要求玩家补全。

```javascript
function checkDecreeCompleteness(decree) {
  if (GM.huangquan.index < 75) return { complete: true };
  
  var missing = [];
  if (!decree.when)      missing.push('时日');
  if (!decree.where)     missing.push('地点');
  if (!decree.who)       missing.push('执行人');
  if (!decree.funding)   missing.push('经费');
  if (!decree.deadline)  missing.push('考核');
  
  if (missing.length > 0) {
    return {
      complete: false,
      response: '臣' + GM.cabinet.head.name + '不敢擅专。陛下此诏，尚缺' 
                + missing.join('、') + '，请陛下圣裁。'
    };
  }
  return { complete: true };
}
```

**大臣行为**：
- 献媚 60%（"陛下圣明"）
- 请示 30%（"臣请陛下圣裁"）
- 纠错 10%（极少，需超高 integrity + 已冒险）

### 6.2 制衡段（35-75）：最佳运行

**玩家诏书**：可一句话概括，AI 大臣自动补全

```
玩家诏书："诏减江南徭役五分之一。"
  ↓ AI 大臣补全（户部+工部奏覆）
  ↓
"臣户部尚书王某谨奏：
  遵陛下诏，江南徭役今岁减五分之一。拟定方案：
  - 苏松常镇减一成五
  - 杭嘉湖减一成
  - 合计减役 30 万夫月
  - 仰赖盐铁余额 3 万两充抵
  - 腊月毕工验收
  请陛下朱批。"
```

玩家简单批"**可**"或"**如议**"即生效。

**大臣行为**：
- 建言 50%
- 补全方案 30%
- 请示 20%

### 6.3 权臣段（0-35）：傀儡

**玩家诏书**：进入权臣手中

四种结果（随权臣意见）：
1. **通过**（~40%）：权臣允许的诏书正常发布
2. **修改**（~25%）：权臣拟订方案后"朱批下发"
3. **驳回**（~15%）：权臣奏"陛下三思"（实为压制）
4. **篡改**（~10%）：暗中改为反向诏书（玩家初期不察）
5. **留中**（~10%）：扣押不发

**玩家视角**：可能不察觉自己已是傀儡，直到多次诏书失效才醒悟。**风闻录事会渐渐透露线索**。

## 十四、设计特色

1. **"度"哲学的完整还原**——两端皆危，中段最佳。这是中国政治哲学的"过犹不及"在游戏系统层面的首次严肃实现。

2. **玩家体验随段位剧变**——专制段玩家写到手酸，制衡段玩家简洁批复，权臣段玩家甚至不知自己是傀儡。这是其他历史模拟游戏从未做到的。

3. **权臣 NPC 诏书干预**——当皇权弱，玩家不再是唯一执政者。权臣会驳回、修改、篡改、自拟诏书。玩家真正体验**汉献帝的处境**。

4. **傀儡而不自知**——权臣的暗中篡改不是立即明显。玩家需通过风闻录事、反常征兆逐渐觉察"朱批字迹似与近日不同"。这是叙事级机制创新。

5. **7 策略反击机制**——密诏 / 分其党羽 / 借兵外臣 / 清议声讨 / 刺杀 / 等其老死 / 忍辱负重。每策略对应历史案例，各有成功率与副作用。

6. **结构改革的永久性**——废丞相、设军机处不是一次性加成，而是**永久改变体制**。朱元璋废相的后果延续到明朝覆亡（懒皇帝即生权臣）。

7. **与威望的四象限**——二者独立但层叠。玩家可以在 4 种帝王原型中演进：暴君 / 事必躬亲无人听 / 受敬傀儡 / 明主贤君。

8. **执行度的多因子公式**——执行度不是皇权直接映射，而是 f(皇权, 详尽度, 配合, 腐败)。专制段高皇权并非高效，因为大臣不补全、自己易出错。

9. **朝代预设贴合历史**——12 朝代 × 4 阶段的皇权值参考实际史实，让玩家从任何时点切入都有"历史真实感"。

10. **诏书历史案例库**——80+ 条历史诏书、权臣、反击案例预设，让玩家的每一步都能找到镜像。
