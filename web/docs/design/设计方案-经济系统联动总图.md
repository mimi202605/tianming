## 二、数据流图（按方向）

### A. 税收流（底→顶）

```
百姓征收（region.farmland × tax.rate × 实际负担系数）
    │
    ├─► 名义应征 [Fiscal P.2]
    │
    ├─► 百姓实际负担 [region.peasantBurdenStructure.peasantActual]
    │     = 名义 × (1 + Σ各级加派)
    │
    ├─► 层层剥夺
    │     → 县级加派 → pocketedByLocal
    │     → 府级加派 → pocketedByLocal
    │     → 路省加派 → pocketedByLocal
    │     → 豪强吞 → pocketedByLandlord
    │     → 折纳价差 → pocketedByConverter
    │     
    ├─► 实际入库部分 [按 region.fiscal.allocation 分账]
    │     ├─► 本级留存 → region.fiscal.ledgers
    │     ├─► 上供下级 → parentRegion.fiscal
    │     └─► 上供中央 × compliance → GM.fiscal.guoku
    │
    └─► 地方留存的使用 [CentralLocal Ⅳ+]
          ├─► fixed 固定扣（俸禄/守军/驿站…）
          │     └─► 发放到 char.resources.privateWealth.cash
          ├─► discretionary AI 自主决策（救灾/水利/教育…）
          │     └─► 产生 region 效果（disasterLevel↓/farmland↑…）
          ├─► imperial 央令指派（诏修工程）
          ├─► illicit 挪用 → char.resources.privateWealth.cash
          └─► downstream 向下拨付
```

### B. 下拨流（顶→底）

```
玩家诏令 / AI 建议："拨银十万赈江南"
    │
    ├─► 创建 transferOrder { central → 江南, 100000 money }
    │
    ├─► Fiscal.spend('guoku.money', 100000, '赈灾')
    │
    ├─► 途中推进（每回合 arrivalTurn-- ）
    │     ├─► logisticsLoss 扣减（漕运损耗）
    │     └─► 沿路事件（流寇/水灾/船坏）可能损失
    │
    └─► 到达 → region.fiscal.ledgers.money.stock += （到达额）
          └─► 触发后续事件（赈济-饥民不起义、民心+）
```

### C. 货币购买力流（贯穿所有账本）

```
货币系统每回合更新：
    ├─► copper.purchasingPowerFactor
    ├─► silver.purchasingPowerFactor
    ├─► paper.creditLevel × reserveRatio
    └─► market.grainPrice / clothPrice
    
所有使用账本数字的地方都要乘 purchasingPowerFactor：
    ├─► Fiscal 账本显示值 × PPF = 实际购买力
    ├─► char.privateWealth.summary × PPF
    ├─► AI prompt "库存 500 万贯（实值 350 万）"
    └─► 赈灾/工程金额换算
```

### D. 角色→系统反馈流

```
char.resources 状态 ──影响──► 
    ├─► 税收效率
    │     └─► 户部尚书能力 → Fiscal.collectTaxes taxEfficiency 系数
    ├─► 贪腐漂没率
    │     └─► 地方官 briberySusceptibility → region.corruption
    ├─► 地方政绩
    │     └─► 知府 贤能 + AI discretionary → region.disasterLevel/farmland
    ├─► 监察有效性
    │     └─► 御史 贤能+名望 → auditSystem 效果
    ├─► 廷议投票权
    │     └─► politicalClout（基于 fame + wealth + virtueMerit）
    └─► 叛乱风险
          └─► rebelRisk → 自立/藩镇坐大 事件
```

## 三、接口清单（四子系统之间的 API 调用）

### Fiscal（财政）暴露的 API

| API | 调用者 | 用途 |
|-----|-------|-----|
| `Fiscal.get(path)` | 任何 | 查库存 |
| `Fiscal.add(path, amount, tag)` | 央地分账、角色俸禄、抄家 | 入账 |
| `Fiscal.spend(path, amount, tag)` | 诏令扣账、地方支出、调拨 | 出账 |
| `Fiscal.transfer(from, to, amount, tag)` | 玩家诏令调拨、内帑↔帑廪 | 账间流动 |
| `Fiscal.collectTaxes(turn)` | endTurn | 每回合税收入 |
| `Fiscal.payFixed(turn)` | endTurn | 俸禄/军饷扣 |
| `Fiscal.processFiscalActions(actions)` | endTurn AI 推演后 | 执行玩家诏令 |
| `Fiscal.settle(turn)` | endTurn 尾 | 回合归档 |
| `Fiscal.annualSettle(year)` | 年末 | 年度归档+弹窗 |

### Currency（货币）暴露的 API

| API | 调用者 | 用途 |
|-----|-------|-----|
| `Currency.getPurchasingPower(coinType)` | Fiscal/Char/UI | 实际购买力系数 |
| `Currency.convert(amount, fromUnit, toUnit)` | 任何换算 | 钱粮布互换 |
| `Currency.mint(agency, quantity)` | 玩家诏令 | 铸币 |
| `Currency.issuePaper(name, amount, reserve)` | 玩家诏令 | 发纸币 |
| `Currency.proposeReform(reformId)` | 玩家诏令 | 货币改革入廷议 |
| `Currency.updateMarketPrices(turn)` | endTurn | 每回合物价 |
| `Currency.processForeignFlow(turn)` | endTurn（明清启用）| 海外银流 |
| `Currency.processIncidentals(turn)` | endTurn | 进奉/羡余 |

### CentralLocal（央地）暴露的 API

| API | 调用者 | 用途 |
|-----|-------|-----|
| `CentralLocal.splitTax(region, tax, amount)` | Fiscal.collectTaxes | 按 allocation 分账 |
| `CentralLocal.executeLocalBudget(region, turn)` | endTurn | 地方 fixed/discretionary |
| `CentralLocal.processTransferOrders(turn)` | endTurn | 调拨在途推进 |
| `CentralLocal.processIncidentals(turn)` | endTurn | 进奉/盈余 |
| `CentralLocal.runAudit(region, censor)` | 玩家诏令/监察 | 审计 |
| `CentralLocal.assessCompliance(region)` | endTurn | 合规度更新 |
| `CentralLocal.checkFrontierEvents(turn)` | endTurn | 藩镇坐大/自立检测 |
| `CentralLocal.proposeReform(reformId)` | 玩家诏令 | 央地改革入廷议 |

### CharacterEconomy（角色经济）暴露的 API

| API | 调用者 | 用途 |
|-----|-------|-----|
| `CharEcon.getResources(char)` | UI/AI/Fiscal | 返回 6 资源 |
| `CharEcon.paySalary(char)` | endTurn Fiscal.payFixed | 发俸禄 |
| `CharEcon.spendPrivate(char, amount, reason)` | AI/玩家 | 私产支出 |
| `CharEcon.embezzle(char, amount)` | AI discretionary | 挪用公库 |
| `CharEcon.computeBehaviorWeights(char)` | AI prompt 注入 | 行为倾向 |
| `CharEcon.confiscate(char, policy)` | 玩家抄家 | 抄家清算 |
| `CharEcon.transferWealth(fromChar, toChar, ...)` | 婚姻/继承 | 财产转移 |
| `CharEcon.checkPromotion(char, pos)` | 升迁判定 | 基于贤能 |
| `CharEcon.updateVirtueFromLocalActions(char, actions)` | endTurn | 政绩→贤能 |
| `CharEcon.updateFameFromEvent(char, event)` | 事件触发 | 名望变动 |
| `CharEcon.aiGenerate(char, sc)` | 编辑器一键生成 | AI 智能生成 |

## 四、每回合执行时序图

```
======== 回合 N 开始 ========

Phase 0. 预处理
    ├─ makeEntitiesReactive()
    ├─ _ensureRegionFiscal()           // 行政区划账本生成
    ├─ _ensureCharResources()          // 角色资源初始化
    └─ _updateVirtueTiers()             // 贤能阶位重算

Phase 1. 玩家输入收集
    ├─ _collectEdicts()                // 诏令文本
    ├─ _collectTingyiDecisions()       // 廷议表决
    └─ _collectXinglu()                // 行止

Phase 2. AI 推演
    ├─ 注入 prompt：
    │   ├─ 【财政现状】Fiscal.guoku/neicang 账本
    │   ├─ 【货币状态】Currency 本位/通胀/市价
    │   ├─ 【央地】各省 compliance/地方支出报告
    │   ├─ 【关键角色 6 资源】高阶/相关 NPC
    │   └─ 【角色行为权重】briberySusceptibility 等
    │
    └─ AI 返回：
        ├─ fiscalActions[]            // 诏令导致的扣账
        ├─ localActions[]              // 地方官自主支出
        ├─ npc_actions[]               // NPC 行为
        ├─ incidentalReceipts[]        // 进奉/盈余事件
        └─ narrative（时政记/正文/奏疏）

Phase 3. 经济处理（核心）
    3.1 征税
        Fiscal.collectTaxes(turn)
            ├─► CentralLocal.splitTax(region, tax, amount)
            │       ├─► region.fiscal.ledgers（本级留存）
            │       ├─► parent.fiscal（上供）
            │       └─► GM.fiscal.guoku（上中央）
            └─► 更新 peasantBurdenStructure

    3.2 俸禄/军饷发放
        Fiscal.payFixed(turn)
            └─► 循环每官员：
                 ├─► 扣 guoku 或 region.fiscal
                 └─► 入 char.resources.privateWealth.cash

    3.3 地方预算执行
        CentralLocal.executeLocalBudget(region, turn)
            ├─► fixed 固定项
            ├─► discretionary（AI localActions 执行）
            │       └─► 产生 region 效果（disasterLevel↓/farmland↑…）
            ├─► imperial 央令
            ├─► illicit 挪用 → char.privateWealth
            └─► downstream 向下拨

    3.4 进奉/羡余/盈余
        Fiscal.processIncidentals(turn)
            └─► 进入 neicang 或 guoku

    3.5 调拨推进
        CentralLocal.processTransferOrders(turn)
            ├─► arrivalTurn-- 
            ├─► 沿途 logisticsLoss 扣
            ├─► 到达则入账
            └─► 可能事件（流寇/水灾）

    3.6 AI 诏令扣账
        Fiscal.processFiscalActions(fiscalActions)

    3.7 自动转账（内帑规则）
        Fiscal.autoTransfer(turn)
            └─► 盐课分成、月例供奉等

Phase 4. 货币更新
    Currency.updateMarketPrices(turn)
        ├─► 粮价/布价/金属料价
        ├─► 货币购买力衰减
        ├─► 纸币状态机转移
        └─► 海外银流（明清）

Phase 5. 角色资源更新
    CharEcon.updatePerTurn()
        ├─► 生活消费扣私产
        ├─► 名望变动（事件累积）
        ├─► 贤能变动（政绩累积）
        ├─► 健康衰减 + 压力变化
        ├─► 债务到期检查
        └─► 阶位重算

Phase 6. 合规与监察
    CentralLocal.assessCompliance()
        ├─► 更新 region.compliance
        ├─► 识别藩镇坐大临界
        └─► 触发央地矛盾事件

    CentralLocal.runAudit() （若有在途审计）
        └─► 发现虚报/贪腐 → 触发调查事件

Phase 7. 事件触发检查
    _checkEconomicEvents()
        ├─► 钱荒/银荒/通胀崩溃
        ├─► 纸币崩溃
        ├─► 地方抗命/自立
        ├─► 破产/抄家
        ├─► 改革进行中阻力
        └─► 外藩贡赋延迟

Phase 8. 结算与归档
    Fiscal.settle(turn)      // 流水归档
    if (isYearBoundary()) {
        Fiscal.annualSettle(year)
        _generateAnnualReport()     // 弹窗
    }

Phase 9. UI 刷新
    ├─► 顶栏（帑廪/内帑/货币/通胀指标）
    ├─► 角色详情面板（6 资源）
    ├─► 地图（region 状态着色）
    └─► 年度决算弹窗（若触发）

======== 回合 N 结束 ========
```

## 五、事件钩子总清单

所有子系统共用 `tm-events.js` 事件总线，事件类型：

### 财政类
- `fiscal.treasury.critical` 国库告急（stock < 阈值）
- `fiscal.peasant_burden_critical` 百姓负担 > 40%
- `fiscal.land_annex_crisis` 兼并 > 50%
- `fiscal.debt_default` 债务违约

### 货币类
- `currency.coin_shortage` 钱荒
- `currency.coin_glut` 钱贱
- `currency.paper_collapse` 纸币崩溃
- `currency.debasement_revolt` 劣钱民变
- `currency.silver_outflow` 白银外流
- `currency.reform_proposed/passed/failed`

### 央地类
- `central_local.region_defiance` 地方抗命
- `central_local.frontier_warlord` 藩镇坐大
- `central_local.self_declaration` 藩镇自立
- `central_local.audit_exposed` 审计揭发贪腐
- `central_local.reform_proposed/passed/failed`
- `central_local.tax_base_lost` 税基丧失（失地）
- `central_local.royal_clan_bankruptcy` 宗室压力爆发（明代）

### 角色经济类
- `char_econ.char_bankrupt` 角色破产
- `char_econ.char_rich_landmark` 角色发家（跨越财富门槛）
- `char_econ.promotion_eligible` 贤能达阶，可升迁
- `char_econ.fame_scandal` 名望丑闻
- `char_econ.marriage_wealth_merge` 婚姻合财
- `char_econ.inheritance` 继承
- `char_econ.confiscation` 抄家执行
- `char_econ.rebellion_risk` 叛乱风险临界
- `char_econ.clan_implication` 株连执行

事件订阅者可跨系统联动（如 `currency.paper_collapse` → 触发 `central_local.region_defiance` 多条）。

## 七、四系统共用数据源

### 7.1 统一存储

```js
GM.fiscal = {
  guoku, neicang,           // Fiscal 子系统
  feudalHoldings,
  transferOrders,
  auditSystem,
  meta: { unitMoney, unitGrain, unitCloth, yearlyArchive, ... }

GM.currency = {             // Currency 子系统
  coins, paper, market, mintAgencies, foreignFlow, standardTimeline, reforms
}

// 行政区划挂载扩展
region.fiscal = {           // CentralLocal 子系统字段
  ledgers, allocation, expenditures, compliance, ...
}

GM.families = [...]         // CharEcon 家族
GM.chars[i].resources = {   // CharEcon 角色资源
  publicTreasury, privateWealth, fame, virtueMerit, ...
}

// 剧本配置
sc.fiscalConfig = {
  unit, initial, taxes, neicangRules, fixed, currencyRules, centralLocalRules,
  charEconomyRules, ...
}
```

### 7.2 存档序列化

所有字段必须在 `_prepareGMForSave` / `_restoreSavedFields` 中处理：

| 字段 | 存 | 读 |
|------|-----|-----|
| `GM.fiscal` | 整块 clone | 整块还原 + `_ensureDefaults` 补全 |
| `GM.currency` | 整块 clone | 同上 |
| `GM.families` | 整块 clone | 同上 |
| `region.fiscal`（递归树）| `_saveAdminTreeFiscal()` | `_restoreAdminTreeFiscal()` |
| `char.resources` | 循环 chars 保存 | 循环 chars 还原 |

## 九、关键风险点与应对

| 风险 | 应对 |
|------|------|
| 四子系统共用 endTurn，执行顺序错乱 | 严格按本总图 Phase 0-9 执行；每 Phase 有明确 pre/post 钩子 |
| 数据一致性（父级≥子级之和）| 每 Phase 末 `_validateFiscalTree()` 审计 |
| AI prompt 长度爆炸 | 分层注入（主推演精炼、sub-call 详细）；关键人物才注入行为权重 |
| 存档体积膨胀 | `history` 定深度 + yearlyArchive 按年归档 + region.fiscal 合并路径 |
| 老存档迁移 | `fullLoadGame` 检测旧字段 `GM.vars.treasury` 等 → 自动迁移 |
| 事件循环无限触发 | 所有事件加 cooldown + 触发计数上限 |
| AI 失控（诏令要花 1 亿）| 硬顶 `single spend ≤ stock`；feasibility='excessive' 反馈 |
