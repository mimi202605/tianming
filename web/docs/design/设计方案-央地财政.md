# 天命 · 央地财政分层系统设计方案

> 起草：2026-04-17
> 状态：八决策定稿 + 地方实际财政仿真扩展定稿
> 归属：经济系统·子系统三（见 `设计方案-经济系统.md`）
> 核心原则：与 `sc.adminHierarchy` 行政区划深度集成；地方留存资金必须被真实花费，不得凭空消失

## 与 `sc.adminHierarchy` 集成（核心原则）

**强制遵循 MEMORY "行政区划设计原则"**：
- 央地财政直接挂 `region` 树，不另建平行结构
- 父级财政数据 ≥ 子级之和（数据约束）
- 领土得失 → region 消失 → fiscal 自动归档 `meta.lostRegions[year]`
- 侨置 → fiscal 字段继承模板但 base=0
- 改革 → allocation 热替换

### 分层自适应

```
剧本 sc.adminHierarchy.depth=2 (仅路+县)：
  中央 → 路/省 → 县

剧本 sc.adminHierarchy.depth=3 (标准)：
  中央 → 路/省 → 府/州 → 县

剧本 sc.adminHierarchy.depth=4 (元清多层)：
  中央 → 行省 → 路 → 府 → 县
```

系统自动适配层级，allocation 比例按层级数分配（如 4 层时上供/留省/留路/留府/留县）。

## Ⅰ 四层分层自适应

### 层级生成

运行时每回合扫描 `sc.adminHierarchy`，为每个 region 建账：

```js
function _ensureRegionFiscal(region) {
  if (!region.fiscal) {
    region.fiscal = {
      ledgers: { money: _newLedger(), grain: _newLedger(), cloth: _newLedger() },
      allocation: _inheritAllocation(region.parent),
      expenditures: { fixed:[], discretionary:[], imperial:[], illicit:[], downstream:[] },
      compliance: 0.9,
      skimmingRate: 0.05,
      overstatement: 0.1,
      autonomyLevel: _defaultAutonomy(region.level),
      governingOfficials: [],
      annualReport: null
    };
  }
  if (region.subs) region.subs.forEach(_ensureRegionFiscal);
}
```

### 父级约束

每回合末检查：
```js
for each region:
  if (region.fiscal.ledgers.money.stock < Σ(子.fiscal.ledgers.money.stock))
    warn('父级 < 子级之和，数据异常')
```

## Ⅱ 分账机制（三种模式）

### B · 唐三分法

```js
region.fiscal.allocation = {
  mode: 'tang_three',
  ratios: {
    toCentral: 1/3,      // 上供户部
    toLevelUp: 0,         // 唐无路级中间
    toSelf: 1/3,          // 留使（节度使/观察使）
    toLevelDown: 1/3      // 留州
  }
}
```

### C · 明清起运存留（按税种）

```js
region.fiscal.allocation = {
  mode: 'qiyun_cunliu',
  perTax: {
    'land_grain': { qiyun: 0.6, cunliu: 0.4 },   // 田赋六成起运
    'head_tax':   { qiyun: 0.8, cunliu: 0.2 },
    'salt':       { qiyun: 0.9, cunliu: 0.1 },
    'commerce':   { qiyun: 0.5, cunliu: 0.5 }
  }
}
```

### D · 自定义

剧本可完全自填 allocation.perTax 或 ratios，或用预设按钮一键切换。

### 分账执行

```js
function _splitTaxByAllocation(region, tax, amount) {
  var alloc = region.fiscal.allocation;
  var splits = {};
  if (alloc.mode === 'qiyun_cunliu') {
    var cfg = alloc.perTax[tax.id] || {qiyun:0.7, cunliu:0.3};
    splits.toCentral = amount * cfg.qiyun;
    splits.toSelf = amount * cfg.cunliu;
  } else if (alloc.mode === 'tang_three') {
    splits = {
      toCentral: amount * alloc.ratios.toCentral,
      toSelf:    amount * alloc.ratios.toSelf,
      toLevelUp: amount * alloc.ratios.toLevelUp,
      toLevelDown: amount * alloc.ratios.toLevelDown
    };
  }
  
  // compliance 扣款
  splits.toCentral *= region.fiscal.compliance;
  splits.skimmedByLocal = amount - Σ(splits) - splits.toCentral * (1 - region.fiscal.compliance);
  
  _executeSplits(region, splits);
}
```

## Ⅳ 地方实际财政仿真（核心扩展）

### 核心理念

**地方留存的钱不是消失了，而是被真实花费**——地方官员根据本地情势自主决策支出，产生实际游戏效果；中央通过监察系统核查。

### 五大支出去向

```js
region.fiscal.expenditures = {
  fixed: [],          // 自动每回合扣
  discretionary: [],  // 地方官自主决策（AI 生成）
  imperial: [],        // 中央指派（玩家诏令）
  illicit: [],         // 挪用贪污（隐蔽）
  downstream: []       // 向下级拨付
}
```

### 1. Fixed 固定扣项

```js
fixed: [
  { category:'local_salary',      amount, cadence:'monthly', priority:1 },
  { category:'local_garrison',    amount, cadence:'monthly', priority:1 },
  { category:'post_stations',     amount, cadence:'monthly', priority:2 },
  { category:'yamen_upkeep',      amount, cadence:'monthly', priority:2 },
  { category:'ritual_sacrifice',  amount, cadence:'annual',  priority:3 }
]
```

资金不足时按 priority 顺序扣，低优先级被欠薪。

### 2. Discretionary 自主决策（AI 产出）

每回合 AI 为每个 region 产出 `localActions`：

```js
{
  regionId: '江南',
  proposer: '知府 王某',
  actions: [
    { type:'disaster_relief',       amount:15000, reason:'本府水灾' },
    { type:'public_works_water',    amount:8000,  reason:'修东堤' },
    { type:'charity_local',         amount:3000,  reason:'族赏穷户' }
  ]
}
```

### 3. Imperial 央令临时指派

玩家诏令"诏修长城" → 中央命令地方出资：
```js
imperial: [
  { category:'funded_project', amount:50000, reason:'诏修长城', orderedBy:'central' }
]
```

### 4. Illicit 挪用贪污

```js
illicit: [
  { category:'embezzlement', amount:4000, officials:['赵某'], concealed:true }
]
```

隐蔽性由官员能力决定，被御史发觉则查办。

### 5. Downstream 向下拨付

```js
downstream: [
  { targetRegion: '苏州府', column:'grain', amount:5000, reason:'府内歉收' }
]
```

### AI 决策频率与规模

| 情况 | 频率 | 规模 |
|------|------|------|
| 正常回合 | 每回合产出 1-2 条 | 小（<3% 本地留存）|
| 本地灾荒/战乱 | 每回合产出 3-5 条 | 大（10-30% 本地留存）|
| 太平丰年留存多 | 偶尔产出大额项 | 中（如修水利、建书院）|
| 邻省灾荒 | 可能产出跨省援助 | 小（5-15%）|
| 本地豪强活跃 | 可能产出 charity_local | 小 |

AI 通过 `region.fiscal` + `region.disasterLevel` + `governingOfficials` 能力性格决定。

### 支出→效果映射表（14 项）

| 支出类别 | 游戏效果 |
|---------|---------|
| disaster_relief | region.disasterLevel ↓ / unrest ↓ / 民心 ↑ |
| public_works_water | region.farmland ↑ / logisticsLoss ↓ |
| public_works_road | region.commerceVolume ↑ / tradeSpeed ↑ |
| public_works_wall | region.garrison ↑ / defense ↑ |
| education | region.educationLevel ↑ / 科举本地占比 ↑ |
| military_prep | region.troopsQuality ↑ / defense ↑ |
| post_stations | region.logisticsLoss ↓ / 中央调度效率 ↑ |
| ritual_sacrifice | 民心 ↑ / legitimacy ↑ |
| charity_local | 豪强乡绅好感 ↑ / 民心中性 |
| embezzlement | 官员 wealth ↑ / 若被察觉 region unrest ↑ |
| courtship_capital | 官员 reputationCentral ↑（讨好中央）|
| **supernatural_disaster_relief** | **unrest ↓（心理安慰）；若无实效，民心反而 ↓** |
| **local_amnesty** | **unrest ↓；但法治威信 ↓、后续犯罪 ↑** |
| **grain_price_stabilization** | **粮价 ↓ / 市场信任 ↑；消耗 financial** |

```js
char.governance = {

```

影响任期评价、升迁概率、身后定论。

### 玩家事后追究

玩家可在"央地"Tab 中看到 `region.fiscal.annualReport` + `localActions` 历史，对可疑官员：

- 诏令"查 X 知府" → 触发审计（需监察资源）
- 诏令"撤换 X" → 换将
- 诏令"抄 X 家" → 追赃（从 char.wealth 追回部分给中央）
- 诏令"刑责 X" → 杖责/流放/处决（影响威信）

**重点：不影响本回合已扣的支出。** 追究只能基于事后报告，本回合地方官已花的钱是花出去了。

### 年度报告

```js
region.fiscal.annualReport = {
  year,
  revenueClaimed: 0,        // 报告收入
  revenueActual: 0,          // 实际收入
  expenditureClaimed: 0,     // 报告支出
  expenditureActual: 0,      // 实际支出
  surplusClaimed: 0,
  surplusActual: 0,
  majorProjects: ['修东堤', '建书院'],
  auditResult: 'unchecked' | 'passed' | 'flagged' | 'investigated'
}
```

### 虚报机制

```js
// 每年生成报告时
revenueClaimed = revenueActual * region.fiscal.selfReportAccuracy
                * (1 - officialHonestyBias)
expenditureClaimed = expenditureActual * (1 + paddingForPockets)

// 虚报差额 = |claimed - actual|
```

御史审计发现差额 > 阈值 → `auditResult = 'flagged'` → 触发腐败调查事件。

## Ⅴ 中央下拨（双向+强征更严）

### 下拨动作

```js
GM.fiscal.transferOrders = [
  {
    id, type:'central_to_local'|'local_to_local'|'central_to_feudal',
    from, to, column, amount, reason,
    issueTurn, arrivalTurn,
    logisticsLoss: 0.15,
    escort: null,
    status: 'en_route'|'arrived'|'lost',
    events: []
  }
]
```

### 触发

| 诏令 | 动作 |
|------|------|
| 拨银十万赈江南 | central → 江南（途中损耗）|
| 江南调粮陕西 | 江南 → 陕西 |
| 借调湖广米粟北运 | 湖广 → 京师 |
| 诏封燕王岁赐十万 | central → feudal |

### 强征（更严）

玩家"括湖广米三十万石" →
- `region.compliance -= 0.2`（原 -0.1）
- `region.unrest += 15`
- 连续强征（2 回合内）→ 双倍惩罚（compliance -0.4，unrest +30）
- 强征 > 本地留存 50% → 立即触发"地方暴动"事件
- 强征 > 本地留存 80% → 触发"地方自立"临界

## Ⅶ 央地博弈（完整）

### 三博弈状态

```js
region.fiscal.compliance = 0.9
region.fiscal.skimmingRate = 0.05
region.fiscal.overstatement = 0.1
```

### 动态

```js
compliance -= 
  centralPressureOver + autonomyRise +
  localOfficialDisloyalty + centralFailurePerception

compliance +=
  centralInspection + centralRewards +
  localOfficialLoyalty + centralAuthorityShow
```

### 矛盾事件链

| 条件 | 事件 | 效果 |
|------|------|------|
| compliance < 0.5（2 年）| 地方抗命 | 实际上缴暴跌 |
| compliance < 0.3 + autonomy > 0.7 | 藩镇坐大 | 准割据 |
| compliance < 0.1 + 军队忠地方 | 藩镇自立 | 升格 feudal 或脱离 |
| 玩家削藩 + 阻力大 | 兵变反抗 | 武装冲突 |

### 削藩机制（走判定分流 + 廷议 2.0）

## 监察系统建模

**监察是有成本的**——派御史/巡按/按察使消耗中央财政和人力。

### CensorSystem 结构

```js
GM.fiscal.auditSystem = {
  annualBudget: 50000,           // 中央年度监察预算（贯）
  activeCensors: [
    { char_name, position:'御史', assignedRegion, salary, tenureStart }
  ],
  standingPositions: [
    { title:'按察使', region, char_name, annualCost }
  ],
  ongoingInspections: [
    { id, inspector, target, startTurn, expectedEnd, cost, findings:[] }
  ],
  coverageRatio: 0.3,            // 被覆盖的 region 比例
  lastAuditedByRegion: { region_id: turn }
}
```

### 监察动作

| 动作 | 成本 | 效果 |
|------|------|------|
| 巡按 X 省（短期）| 3000 贯 + 御史 1 名 × 2 回合 | 临时审计，compliance +0.1 |
| 按察使驻 X 省（长期）| 年 10000 贯 + 御史 1 名常驻 | 持续监察，compliance +0.2，虚报被即时揭 |
| 突击查 X 官 | 1500 贯 | 单官针对性审计 |
| 密探制度 | 年 30000 贯总预算 | 全国暗查网，发现隐蔽腐败 |

### 监察资源约束

```js
// 中央监察预算不够 → 覆盖不足 → 地方钻空
if (auditSystem.annualBudget < demandedAuditCost) {
  coverageRatio = annualBudget / demandedAuditCost
  // 未覆盖的 region：skimming 无人查，地方虚报泛滥
}
```

玩家诏令"充监察预算" → 从 Fiscal.guoku 扣。

### 监察 NPC

御史/按察使本身是 NPC 角色，他们的能力、清廉度、派系影响审计效果：
- 清廉强御史：发现大额差额
- 贪腐御史：反成被查对象的同谋（"沆瀣一气"）
- 政治派御史：只查敌派官员

### 8.1 地方截留率 × 皇权 × 皇威

```javascript
function calcLocalRetentionRate(region) {

  var h = GM.huangquan.index;
  var w = GM.huangwei.index;

  if (h < 35) baseRate += 0.15;      // 权臣段：地方截留多，实际上缴 ~ 45%

  if (w < 30) baseRate += 0.10;      // 失威段：地方观望，截留增

}
```

### 8.2 削藩成功条件

```javascript
function evaluateXueFanFeasibility() {
  var h = GM.huangquan.index;
  var w = GM.huangwei.index;
  var m = GM.minxin.trueIndex;
  var treasury = GM.guoku.balance;
  
  // 必要条件
  if (h < 50) return { feasible: false, reason: '皇权不足，强行削藩恐反叛' };
  if (w < 50) return { feasible: false, reason: '皇威不足，藩镇不服' };

  // 成功率
  var successRate = 0.4 
    + (h - 50) / 100 * 0.3 
    + (w - 50) / 100 * 0.2 
    + (m - 40) / 100 * 0.1;
  
  return { feasible: true, successRate: clamp(successRate, 0.1, 0.9) };
}
```
