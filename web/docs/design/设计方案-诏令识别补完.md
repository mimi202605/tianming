# 设计方案 · 诏令识别补完（虚空渠道回收）

> **目的**：回收户口深化、环境承载力、央地财政等方案中承诺但未挂钩 UI 的"虚空渠道"。  
> **原则**：**不新增独立标签页**。所有虚空渠道统一归入现有四大入口——**诏令**（主）/ **问对**（辅）/ **朝议**（重大）/ **奏疏**（反向触发）。  
> **核心思路**：强化 AI 诏令解析器的识别能力与预设模板库；奏疏系统主动提示可行动作；问对朝议承接争议决策。

---

## 一、设计原则

### 1.1 四大入口分工

| 入口 | 适用场景 | 特点 |
|------|---------|------|
| **诏令** | 玩家主动发起的一切政令 | 自由文本 AI 解析，支持预设模板 |
| **问对** | 玩家向具体官员咨询具体事务 | 一对一，AI 扮演该官员角色答复 |
| **朝议** | 涉及百官或重大分歧的决策 | 多方陈议，可投票或皇帝独断 |
| **奏疏** | 官员主动上报、请示、反馈 | 反向触发——系统识别条件后由 AI 生成奏疏提示玩家 |

### 1.2 四大入口互相衔接

```
玩家诏令 ──→ AI 解析 ──┬─→ 直断实施（诏令意图明确）
                      ├─→ 奏疏复奏（需方案细化）
                      └─→ 问对/朝议（存疑或涉重大）

系统条件 ──→ AI 奏疏 ──→ 玩家批复（内附多选项）
                                ↓
                          可升级为诏令/朝议
```

### 1.3 预设模板的角色

诏令编辑框**右下角**（或抽屉）提供"**参考**"按钮，展开历代典型诏令模板（25+ 大徭役、13 环保政策、8 大卫所案例等）。  
- 点击模板 → 文本填入诏令编辑框
- 玩家可直接发布或修改后发布
- **不是新标签页**，仅是诏令输入框的增强

### 2.1 大徭役工程发起（户口深化 F3.2）

#### 入口：诏令 + 预设模板库

**识别关键词**（AI 解析器）：
```
兴工 / 修 / 筑 / 凿 / 开 / 建 / 营造 / 大役 / 征发 + 规模量词（万/十万/百万）
```

#### 解析流程

```javascript
// AI 诏令解析器识别
function parseCorveeDecree(text) {
  var match = {
    project: extractProjectName(text),      // "潼关大堤"、"长城北段"
    type: classifyProject(text),            // 'dike'|'wall'|'canal'|'palace'|'road'
    scale: extractScale(text),              // 丁数（万夫/十万/百万）
    duration: extractDuration(text),        // "三年"/"毕工乃止"
    region: extractRegion(text),            // 征发省份
    budget: estimateBudget(text) || 'AI-estimate'  // AI 估算

  // 进入二阶段流程（制度设计器已有机制）

  return match;
```

### 2.2 招抚流民（户口深化 F3.4）

#### 入口：诏令 + 奏疏反向触发 + 问对

**识别关键词**：
```
招抚 / 安插 / 归耕 / 赐田 / 劝农 / 招徕 / 复业 / 还乡
```

#### 奏疏反向触发（**最重要的入口**）

当系统检测到 `regionState.refugees > threshold` 时，户部或地方巡抚自动奏疏：

```
臣江南巡抚王某谨奏：
  自去岁水旱，两浙流民约十五万户。今聚于苏常、游食市井，
  恐生窃盗民变。查湖广、河南仍有荒田可耕，伏请陛下圣断：

  [诏招抚安插湖广，赐田二十亩免赋三年]  ← 一键生成诏令
  [令地方赈粥以度岁]
  [遣使巡查，详陈情形]
  [留中不发]
```

勾选第一项 → 自动生成诏令文本填入编辑框 → 玩家可直发或改写。

#### 问对入口

玩家可直接召户部尚书问对："**近来流民几何？安插之策若何？**"  
→ AI 扮演户部尚书，陈述现状 + 建议 + 提供可执行诏令。

#### 诏令 AI 解析

```javascript
function parseRefugeeDecree(text) {
  return {
    action: 'resettle-refugees',
    source: extractSourceRegion(text),     // "江南"
    target: extractTargetRegion(text),     // "湖广"
    incentive: {
      landGrant: extractLandGrant(text),   // "二十亩"
      taxExempt: extractTaxExempt(text),   // "三年免赋"
      toolsGrant: extractToolsGrant(text), // "给耕牛种子"
      militaryRelocation: isMilitary(text) // 是否军事性迁徙
    },
    volume: extractVolume(text),           // "十万户"或 AI 估算
    coerced: /强徙|迁徙/.test(text)        // 强制性
  };
}
```

### 2.4 环保政策触发（环境承载力 Ⅶ）

#### 入口：诏令 + 奏疏反向触发 + 朝议（重大危机）

**13 类环保政策关键词表**：

| PolicyId | 识别关键词 |
|---------|----------|
| `forestClose` | 封山 / 禁樵 / 育林 |
| `dredging` | 疏浚 / 浚河 / 清淤 / 治河 |
| `reclamation` | 开荒 / 垦田 / 屯垦（⚠ 反环保） |
| `desertControl` | 治沙 / 沙防 / 植树 |
| `seaBan` | 禁海 / 海禁 / 闭关 |
| `fishBan` | 禁捕 / 休渔 / 护泽 |
| `waterAllocation` | 分水 / 渠规 / 闸制 |
| `manureUse` | 粪田 / 沤肥 / 地力 |
| `rotationFarming` | 休耕 / 轮作 / 代田法 |
| `terrace` | 筑梯田 / 劝山耕 |
| `floodRelief` | 蓄洪 / 泄洪 / 滞蓄 |
| `woodlandPlant` | 植树 / 广种桑柘 |
| `animalProtect` | 禁猎 / 祝网 / 护生 |

#### 奏疏反向触发（**最关键**）

当生态疤痕（`ecologicalScars`）累积到阈值，工部或地方奏疏：

```
臣陕西巡抚张某谨奏：
  陇右连年水土流失，渭水泥沙猛增，恐成大患。
  臣谨议，请陛下圣断：

  [诏封陇西诸山三年，禁樵采]  ← 一键诏令（forestClose）
  [诏疏浚泾渭三年]              ← dredging
  [诏植桑柘于荒坡]              ← woodlandPlant
  [令地方自为之，不必烦朝议]
  [留中]
```

#### 朝议触发

若同时有**多项生态危机叠加**（如沙暴+饥荒+河患）→ 自动升级为朝议，百官各陈己见（六部+地方代表+清流）→ 皇帝独断或投票。

#### 诏令 AI 解析

```javascript
function parseEnvironmentalDecree(text) {
  var policies = [];
  for (var pid in POLICY_KEYWORDS) {
    if (POLICY_KEYWORDS[pid].test(text)) {
      policies.push({
        policyId: pid,
        scope: extractScope(text),        // 区域范围
        duration: extractDuration(text),  // 年限
        strength: extractStrength(text)   // 严/宽
      });
    }
  }
  return { action: 'env-policy', policies: policies };
}
```

### 3.2 迁都（户口深化 F3）

#### 入口：朝议（**必**）+ 诏令（最终定案）

迁都是王朝级决策，**强制朝议**：

```
玩家意图 → 朝议三轮
  第一轮：百官各陈主张（留/迁/缓议）
  第二轮：若主迁，议"迁何处"
  第三轮：确定迁地后议"迁都方案"（时间表、随迁人口、财政）
  ↓
确定后由皇帝下诏："诏迁都于X，限XX年毕"
  ↓
多年大徭役 + 大规模人口迁移 + 财政巨耗
```

#### 诏令 AI 解析
识别"迁都X"→ 若尚未朝议 → 弹出提示"此事关国本，请先开朝议"。

## 四、诏令通用识别流程（强化版）

### 4.1 AI 诏令分发矩阵

玩家输入任何自由文本诏令，AI 解析器按下列流程分发：

```
诏令文本 → Tokenize → 关键词扫描 → 意图识别
                                    ↓
        ┌─────────────────┬─────────┬──────────┬──────────┐
        ↓                 ↓         ↓          ↓          ↓
     直断实施        奏疏复奏    问对咨询   朝议升级     拒绝
     （明确单一）    （需细化）  （存疑）   （重大争议） （违制）
                       ↓
              ┌────────┴─────────┐
              ↓                  ↓
         户部/工部/兵部…      甲乙丙丁方案
         按内容分配           展示给玩家批复
```

### 4.2 识别优先级表

| 关键词族 | 主责部门 | 触发系统 | 复杂度 |
|---------|---------|---------|--------|
| 兴工/修筑/凿/开/建 | 工部 | 大徭役 + 财政 | 高（二阶段） |
| 招抚/安插/赐田 | 户部 | 流民迁徙 + 户口 | 中 |
| 置卫/设所/屯田 | 兵部 | 军屯 + 边防 | 高 |
| 封山/疏浚/禁捕/植… | 工部 | 环保 13 类 | 低-中 |
| 加役/减役/免差 | 户部 | 徭役分配 | 低 |
| 改土归流 | 理藩 | 民族政策 | 高（朝议） |
| 迁都 | 百官 | 王朝级 | 极高（三轮朝议） |
| 开边/徙实 | 户部+兵部 | 边疆开发 | 中 |
| 大造黄册/清查 | 户部 | 户口清理 | 中 |

### 4.3 异常/歧义处理

- **多意图**：诏令含多个关键词族（如"疏浚大运河兼招江南流民"）→ 拆分为多个子诏令，分别进入各自流程
- **模糊**：未能明确识别 → **降级到"复奏"**，由 AI 奏疏问询细节（"陛下所谓XX，是欲X还是Y？"）
- **违制**：明显违反祖宗之法（如皇权极弱时下达高压诏令） → 大臣"抗疏"（已有机制）
- **资源不足**：财政或人力不足→ 大臣奏"臣请陛下三思"→ 玩家可强推或撤回

## 五、奏疏反向触发清单

**设计原则**：玩家不必知道"可以做什么"，系统条件成立时会主动提醒。

| 触发条件 | 奏疏来源 | 附议动作 |
|---------|---------|---------|
| 某地流民 > 阈值 | 户部/地方巡抚 | 招抚诏令、赈粥、遣使 |
| 生态疤痕累积 | 工部/地方 | 13 类环保政策 |
| 边患风险升高 | 兵部 | 设卫、调兵、增戍 |
| 某工程完工 | 工部 | 验收/追责/记功 |
| 大案浮现 | 御史台/都察院 | 钦差、三法司、留中 |
| 每年春末 | 户部 | 赋役总纲（徭役分配） |
| 每十年开春 | 户部 | 大造黄册 |
| 饥荒/水旱 | 地方 | 赈济、减税、招抚 |
| 卫所糜烂 | 兵部 | 整顿、裁撤、重置 |
| 土司反叛 | 兵部/理藩 | 征剿、羁縻、改流 |
| 人口激增/锐减 | 户部 | 安置、募民、徙实 |

## 十一、设计总结

### 回收的虚空渠道

| 虚空 | 归入入口 | 预设模板 | 反向触发 |
|------|---------|---------|---------|
| 大徭役工程（F3.2） | 诏令 + 二阶段奏疏 | ✓ 25 条 | — |
| 招抚流民（F3.4） | 诏令 + 奏疏触发 + 问对 | ✓ 10 条 | ✓ 流民阈值 |
| 卫所军屯（F3.5） | 诏令 + 问对 | ✓ 8 条 | ✓ 边患 |
| 环保 13 策（环境 Ⅶ） | 诏令 + 奏疏触发 + 朝议 | ✓ 13 条 | ✓ 生态疤痕 |
| 徭役分配（F3.3） | 诏令 + 年度赋役奏疏 | ✓ 6 条 | ✓ 年春末 |
| 改土归流（F3.5） | 诏令 + 朝议 | ✓ 5 条 | ✓ 土司反叛 |
| 迁都（F3） | 三轮朝议 + 诏令 | — | — |
| 边疆开发（F3） | 诏令 | ✓ 8 条 | — |
| 户口清查（F3） | 诏令 + 十年奏疏 | ✓ | ✓ 十年节点 |

### 核心设计哲学

> **"虚空"的根源不是缺少标签页，而是缺少"何时、从哪里入口"的说明。**  
> 本补完通过三招回收：  
> 1. **诏令 AI 解析矩阵** ← 玩家主动时能被识别  
> 2. **奏疏反向触发** ← 系统条件成立时主动提醒  
> 3. **预设模板库** ← 降低玩家"不知道可以做什么"的心理门槛  
> 共同保证：**玩家不会错过任何机制，也不会被迫学习一套新 UI**。
